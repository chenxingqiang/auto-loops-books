Teams hit \textbf{eager mode} when Three Kernel Execution Modes looks correct in isolation yet decode telemetry disagrees---usually because eager was planned without PyTorch residency on the SKU you ship \citep{patel2025llminference,dlbook}.

Hardware truth: GPUs bound eager with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on eager/PyTorch legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper may legalize eager with TMA double-buffer; XDNA requires static tile assignment; CPU needs cache-blocked PyTorch---identical math, different bytes/token \citep{semianalysis2024tma,amd2024xdna,dlbook}.

The production surprise at \textbf{graph mode} is not peak FLOPs; it is illegal tier placement: CUDA Graphs spilling to HBM/DDR while CUDA Graphs still pays launch or DMA tax every token \citep{taxbreak2026,nvidia2024cudagraphs}.

Trace residency: CUDA Graphs producers must feed CUDA Graphs consumers without a DDR round trip \citep{dao2022flashattention,yirage2026}. Silent spill reads as ``mysterious'' p99 regression \citep{patel2025llminference,semianalysis2024tma}.

Operational fix: diff IR or CUDA for new allocs; count launches/token (GPU) or DMA edges (NPU); require graph capture legality on GPU and static legality on XDNA \citep{nvidia2024cudagraphs,amd2024aie}.

\paragraph{Review gate.} Attach roofline or buffer accounting for CUDA Graphs on target SKU before merge \citep{memoryfloor2026,yirage2026}.

\section{MegaKernel mode}
\label{sec:ch12_megakernel_mode}

\textbf{MegaKernel mode} is where Three Kernel Execution Modes meets Chapter~1 counters: fix MegaKernel and MegaKernel together or bytes/token flatlines while kernel microbenchs celebrate \citep{taxbreak2026,nvidia2024cudagraphs}.

Hardware truth: GPUs bound MegaKernel with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

\section{Chapter Summary}

\paragraph{Review gate.} Before merging compiler changes affecting eager mode, attach roofline or NPU buffer accounting showing eager residency fits on-chip for the target SKU \citep{patel2025llminference,dlbook}.

\paragraph{Worked contrast.} A Hopper decode cell may legalize eager with TMA-backed double buffering while an XDNA cell requires the same math expressed as static tile assignments---identical FLOPs, different bytes/token because DDR traffic differs by an order of magnitude \citep{patel2025llminference,dlbook}.

The usual Three Kernel Execution Modes failure at \textbf{graph mode} is importing a CUDA mental model and expecting runtime scheduling to hide illegal buffer lifetimes.
