\chapter{XLA: Production-Grade AI Compiler}
\label{chap:ch15}
\typeout{START_CHAPTER "ch15" \theabspage}

XLA (Accelerated Linear Algebra) is the production compiler spine behind JAX, TensorFlow, and many TPU/GPU serving stacks---yet teams still treat it as ``just fusion'' until decode telemetry proves otherwise \citep{chen2020compilerbasedhardw,yirage2026,taxbreak2026,patel2025llminference}. Part~VI opens with XLA because its HLO IR, target-specific lowering, and layout passes are the reference pattern for \emph{multi-hardware} AI compilation: same graph, different bytes/token on GPU, CPU, and NPU \citep{dlbook,amd2024xdna,amd2024aie,semianalysis2024tma,kwon2023vllm,liu2024deepseekv3}.

TaxBreak dense Llama-3.2-1B averages 847.5 kernel launches per output token on H100; OLMoE-1B/7B averages 9,305 \citep{taxbreak2026}. Memory-Floor CUDA Graphs on Qwen-2.5-7B (BS${=}1$, ctx${=}2048$) yields $1.259\times$ on H100 by removing host overhead---XLA can shrink launch counts when fusion and layout passes are legal, but cannot fix illegal HBM residency \citep{memoryfloor2026,nvidia2024cudagraphs,dao2023flashdecoding}. The production bar: every XLA pass must declare which decode counter it moves and which tier it may occupy \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,dao2022flashattention,nvidia2024cudagraphs,nvidia2022hopper,amd2024aie,amd2024xdna,semianalysis2024tma}.

Five surfaces anchor this chapter: \textbf{hardware matrix}, \textbf{architecture passes}, \textbf{codegen}, \textbf{tuning}, and a \textbf{decode case study} \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Later Part~VI chapters (TVM, Triton, IREE) reuse the same decode-first compiler methodology with different IR dialects \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Anti-pattern: tuning XLA on prefill GEMM shapes while batch-1 decode still launches dozens of thin elementwise kernels per token \citep{taxbreak2026,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Fix: profile decode cells first, then let HLO passes chase launches/token and bytes/token \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\textbf{XLA hardware matrix} maps HLO operators to per-target capabilities---CUDA cores and shared memory on GPU, AVX/AMX vector paths on CPU, static tile SRAM and DMA on NPU \citep{chen2020compilerbasedhardw,yirage2026,amd2024xdna,amd2024aie,dlbook,semianalysis2024tma,nvidia2022hopper,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. The matrix is not a benchmark table; it is a legality oracle: which fusions, layouts, and custom calls each backend may emit at batch-1 decode \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\paragraph{Multi-hardware delta.} GPU matrix entries emphasize TMA-capable fusion and CUDA graph capture; CPU entries emphasize cache-blocking and oneDNN layout; NPU entries emphasize fixed tile shapes and DMA depth---identical HLO, incomparable residency \citep{semianalysis2024tma,amd2024xdna,amd2024aie,dlbook,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\begin{example}
Llama decode attention block: GPU matrix may legalize flash-style fused softmax+matmul; XDNA matrix may require split DMA stages---same math, different bytes/token \citep{dao2022flashattention,dao2023flashdecoding,amd2024xdna,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper}.
\end{example}

\textbf{XLA architecture passes}---algebraic simplification, fusion, layout assignment, SPMD partitioning---determine whether decode stays on-chip or spills every token \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Fusion pass merges elementwise chains into fewer kernels---directly attacking launches/token \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Layout pass chooses tiled vs default memory order; wrong layout can double bytes/token without changing FLOPs \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\paragraph{Review gate.} Each pass merge attaches buffer accounting on the decode cell SKU---new allocs or layout changes must show bytes/token delta \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Silent spill after a ``successful'' fusion pass is the classic XLA production surprise \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\section{XLA codegen}
\label{sec:ch15_xla_codegen}

\textbf{XLA codegen} lowers optimized HLO to LLVM IR, PTX, or backend-specific emitters---where platform legality becomes concrete launch boundaries \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Custom-call boundaries split fused regions; each boundary is a potential launches/token regression at batch-1 \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

GPU codegen must cooperate with CUDA graph capture: dynamic shapes or illegal sync points break capture even when microkernels are fast \citep{nvidia2024cudagraphs,memoryfloor2026,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. CPU codegen through oneDNN favors static shapes; NPU codegen emits DMA schedules with zero runtime flexibility \citep{amd2024xdna,amd2024aie,dlbook,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\section{XLA tuning}
\label{sec:ch15_xla_tuning}

\textbf{XLA tuning}---autotune flags, layout search, and profile-guided recompilation---must optimize decode buckets, not training microbenches \citep{patel2025llminference,chen2020compilerbasedhardw,yirage2026,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Profile inputs should match production ctx lengths and BS${=}1$ decode, or autotune picks layouts that win on prefill and lose on token generation \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Operational pitfall: enabling aggressive fusion without re-running launches/token counters---TaxBreak shows launch explosions are often pass-order bugs, not model bugs \citep{taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Tune with fingerprinted configs so fleet rollbacks replay the same HLO snapshot \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

XLA autotune search space grows quickly on MoE models---DeepSeek-class graphs need expert-aware layout constraints or autotune explores illegal paging patterns \citep{liu2024deepseekv3,patel2025llminference,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Production teams cap autotune wall time and require decode-cell replay before promoting any winning config \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\section{XLA case study}
\label{sec:ch15_xla_case_study}

\textbf{Case study: Llama-class decode on GPU via XLA.} Start from eager baseline launches/token; compile with fusion enabled; diff HLO before/after layout pass; validate CUDA graph capture on the fused decode step \citep{taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Success: fewer launches/token \emph{and} stable bytes/token across ctx buckets; failure: fused prefill kernel that cannot replay at BS${=}1$ decode \citep{patel2025llminference,dao2023flashdecoding,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,nvidia2024cudagraphs,nvidia2022hopper}.

Cross-hardware extension: rerun the same HLO export on CPU (oneDNN layout) and NPU (static schedule) with shared decode-cell methodology---compare counters, not FLOPs \citep{amd2024xdna,amd2024aie,dlbook,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. YiRage iron rules (Chapter~2) foreshadow what XLA passes should encode as checkable metadata \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Document the case study as a reproducible artifact: HLO snapshots, pass flags, driver matrix, and decode-cell CSV---the same bundle Chapter~26 packages for fleet rollout \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Without artifacts, ``XLA is faster'' claims do not survive the next driver upgrade \citep{patel2025llminference,memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\section{Key Takeaways}
\label{sec:ch15_key_takeaways}

\begin{itemize}
\item \textbf{HW matrix.} Per-target legality oracle before fusion---not a FLOPs leaderboard \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Arch passes.} Fusion and layout move launches/token and bytes/token; gate on buffer accounting \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Codegen.} Backend emit must respect graph capture (GPU) and static DMA (NPU) \citep{nvidia2024cudagraphs,amd2024xdna,amd2024aie,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Tuning.} Autotune on decode buckets; fingerprint configs for rollback \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Case study.} Llama GPU decode: launches/token + bytes/token + graph legality \citep{taxbreak2026,memoryfloor2026,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\end{itemize}

\section{Conclusion}
\label{sec:ch15_conclusion}

XLA is a residency contract under decode metrics Chapter~1 established \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Chapter~16 (TVM) and Chapter~17 (Triton) apply the same decode-first bar with different stack surfaces \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Gate merges on buffer accounting and reproducible decode cells---not microbench FLOPs alone \citep{yirage2026,semianalysis2024tma,amd2024aie,patel2025llminference,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
