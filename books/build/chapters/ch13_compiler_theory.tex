Teams hit \textbf{tiling theory} when Core Theory looks correct in isolation yet decode telemetry disagrees---usually because tiling was planned without tiling residency on the SKU you ship \citep{nvidia2022hopper,amd2024aie}.

Hardware truth: GPUs bound tiling with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on tiling/tiling legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper may legalize tiling with TMA double-buffer; XDNA requires static tile assignment; CPU needs cache-blocked tiling---identical math, different bytes/token \citep{semianalysis2024tma,amd2024xdna,dlbook}.

\begin{example}
Decode cell (BS${=}1$, ctx${=}2048$): unfused tiling raises bytes/token; fused schedule must keep tiling on-chip across GPU, CPU, and NPU baselines \citep{taxbreak2026,memoryfloor2026,yirage2026}.
\end{example}

\paragraph{Scope.} This section binds tiling to decode residency and orchestration metrics on GPU, CPU, and NPU targets \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference}.

The production surprise at \textbf{fusion theory} is not peak FLOPs; it is illegal tier placement: fusion spilling to HBM/DDR while fusion still pays launch or DMA tax every token \citep{dao2022flashattention,yirage2026}.

Trace residency: fusion producers must feed fusion consumers without a DDR round trip \citep{dao2022flashattention,yirage2026}. Silent spill reads as ``mysterious'' p99 regression \citep{patel2025llminference,semianalysis2024tma}.

Operational fix: diff IR or CUDA for new allocs; count launches/token (GPU) or DMA edges (NPU); require graph capture legality on GPU and static legality on XDNA \citep{nvidia2024cudagraphs,amd2024aie}.

\paragraph{Review gate.} Attach roofline or buffer accounting for fusion on target SKU before merge \citep{memoryfloor2026,yirage2026}.

\section{Chapter Summary}
\label{sec:ch13_summary}

Core Theory succeeds when hardware constraints drive schedules and decode metrics prove relief \citep{yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw}. Carry bytes/token and launches/token forward; later chapters assume these baselines exist. Key threads: Section~\ref{sec:ch13_tiling_theory}, Section~\ref{sec:ch13_fusion_theory}, Section~\ref{sec:ch13_layout_theory} \citep{dao2022flashattention,amd2024xdna,nvidia2024cudagraphs,liu2024deepseekv3,kwon2023vllm,dlbook}.

\textbf{tiling theory} exposes a cross-hardware fracture: the same fused subgraph legal on Hopper can be rejected outright on XDNA or CPU cache budgets.

Binding software to silicon: compiler passes must encode tiling/tiling legality before codegen, not as comments in hand-written kernels \citep{nvidia2022hopper,amd2024aie}. YiRage runs target-aware checks against chip models from Chapter~3; manual stacks should treat failed legality as a release blocker, not a backend TODO.

When tiling theory disagrees with microbenchmarks, trust fleet telemetry: fix orchestration and tier placement first, then revisit MMA tile shapes \citep{nvidia2022hopper,amd2024aie}.

\paragraph{Review gate.} Before merging compiler changes affecting tiling theory, attach roofline or NPU buffer accounting showing tiling residency fits on-chip for the target SKU \citep{nvidia2022hopper,amd2024aie}.

\paragraph{Worked contrast.} A Hopper decode cell may legalize tiling with TMA-backed double buffering while an XDNA cell requires the same math expressed as static tile assignments---identical FLOPs, different bytes/token because DDR traffic differs by an order of magnitude \citep{nvidia2022hopper,amd2024aie}.

The usual Core Theory failure at \textbf{fusion theory} is importing a CUDA mental model and expecting runtime scheduling to hide illegal buffer lifetimes.

Hardware constraint: fusion and fusion must be co-designed. On GPUs that means shared-memory/TMA budgets and warp-grain barriers; on XDNA/AIE it means static tile buffers, DMA chains, and carrier slots with compile-time lifetimes; on CPUs it means L2/L3 blocking and NUMA-local KV placement \citep{dao2022flashattention,yirage2026}. Decode at batch-1 keeps arithmetic intensity on the memory-bandwidth slope---micro-optimizing isolated GEMMs rarely moves p99 latency.

Compiler takeaway for fusion theory: lower the same decoder IR, but predicate passes on target residency tables---never ship a GPU-only schedule to NPU fleets \citep{dao2022flashattention,yirage2026}. Success metrics: bytes/token, launches/token, and p99 decode latency; compare GPU, CPU, and NPU cells on identical context lengths.
