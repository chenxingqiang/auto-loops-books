Chapter~28 placed inference \emph{frameworks} above the decode step; this chapter covers YiRage \emph{below} exported graphs---the C++ runtime, Persistent Kernel launcher, and backend registry that turn \texttt{superoptimize()} results into repeatable decode cells \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dlbook}. Source tree: git submodule \texttt{deps/YiRage} (upstream \citep{yirage2026}); init via \texttt{git submodule update --init --recursive deps/YiRage} (see \texttt{deps/README.md}).

TaxBreak counts launches/token inside the decode box; Memory-Floor counts bytes/token and graph replay relief \citep{taxbreak2026,memoryfloor2026}. Compiler passes from Part~VI--VII emit fused plans; \textbf{Layer~5 runtime} owns device memory discipline, launch templates, and backend selection so those plans survive fleet load \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. Without runtime contracts, \texttt{superoptimize()} wins evaporate at the framework plugin boundary \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dlbook}.

Readers should map each Layer~5 API to Chapter~1 counters: \texttt{PersistentKernel} collapses launches/token; runtime residency metadata cuts bytes/token; \texttt{HardwareRegistry} constrains search to the SKU you ship \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,amd2024xdna,amd2024aie,semianalysis2024tma,dlbook}. Chapter~30 stitches framework schedulers to this runtime; here we stay inside \texttt{deps/YiRage} \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dlbook}.

Fregly mechanical sympathy at runtime means treating device memory like registers with lifetimes: allocations inside the decode step are bugs unless proven on-chip \citep{yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,patel2025llminference,dao2022flashattention,dlbook,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Layer~5 code paths should log residency class transitions the way Chapter~6 planners annotate IR \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. When runtime logs show unexpected HBM allocs per token, return to Layer~3 fusion legality before tuning scheduler batch windows \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,dao2022flashattention,dlbook}.

This chapter is the mirror of Chapter~28: frameworks own everything outside the decode box; runtime owns everything inside once the framework plugin invokes PK \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Integration meetings should draw that box first, then assign Layer~1--5 owners \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

Runtime engineering begins where compiler IR ends. The fused graph is no longer a data structure in Python; it is a device resident program with allocation pools, synchronization points, and launch templates that must survive millions of decode steps \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Teams new to YiRage often stop measuring after superoptimize reports a winning tile config. Production goodput is decided in Layer~5 when paging pointers arrive from vLLM, when CUDA graphs replay, and when softmax carriers stay in registers instead of spilling to HBM \citep{kwon2023vllm,memoryfloor2026,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,nvidia2024cudagraphs,dlbook}. Treat runtime as a first class performance surface with its own counters, owners, and release train \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

The submodule layout under deps mirrors this responsibility. Python examples in the book intentionally point at pinned upstream APIs, but Layer~5 behavior lives in C++ under libyirage runtime \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. When documentation disagrees with profiler traces, trust traces and file issues against runtime launch paths rather than re tuning scheduler batch sizes alone \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Chapter~30 will show how framework schedulers call into this stack; here we keep the camera inside Layer~5 \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dlbook}.

Three questions gate every runtime review. First, does warmed PersistentKernel beat eager launches/token on the fleet SKU at batch one \citep{taxbreak2026,yirage2026,patel2025llminference,dlbook}. Second, does bytes/token beat the pre fusion baseline at context 512, 2048, and 8192 \citep{memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Third, does integration preserve paging ABI and softmax carriers through ten thousand cancel and resume cycles \citep{kwon2023vllm,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. If any answer is no, stop feature work and return to superoptimize or descriptor design \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Runtime tuning knobs cannot fix illegal fusion or missing manifest fields \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Runtime ownership should appear on the org chart, not only in architecture slides. Layer five engineers maintain launch descriptor schemas, pool allocator invariants, and graph capture bucket tables that framework teams consume as stable ABI. When paging block size changes in vLLM, runtime must publish a semver bump before framework plugins ship; when superoptimize emits a new softmax carrier layout, runtime validates manifests at load time rather than discovering divergence during golden token tests. On-call runbooks should list the three counter worksheets from Chapter one beside PK warmup steps so incidents do not devolve into guessing whether regression lives in search, runtime, or scheduler batching.

Production fleets rarely fail because a single kernel is slow; they fail because two replicas run different artifact digests, because cancel paths leak pool slots under continuous batching, or because a canary node reports the wrong chip id to the registry. Layer five observability therefore emphasizes per-step JSON logs with fingerprint, registry id, descriptor version, launches count, and byte estimate aggregated into dashboards that framework SREs can read without reading C++. Postmortems should replay ten thousand decode steps from a single request id and verify pool high-water marks return to baseline after cancel. Treat runtime regressions as release train defects: roll back the artifact bundle, not only the framework container image.

\textbf{Layer boundaries and debugging.} Production incidents often misattribute Layer~3 search regressions to Layer~5 runtime or vice versa \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook}. When bytes/token regress, diff the fused $\mu$Graph fingerprint first (Layer~3 output); when launches/token regress, inspect PK launch descriptors and graph capture buckets (Layer~5) \citep{yirage2026,nvidia2024cudagraphs,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,kwon2023vllm,dao2022flashattention,dlbook}. Layer~2 must reject cross-backend artifacts silently loaded from cache---a Hopper-tuned plan must not execute on XDNA without re-search \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,patel2025llminference,dlbook}.

Industrial teams should staff Layer~5 with engineers who read nsys and who understand paging ABIs from Chapter~28 \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Compiler teams own Layers~1--4 artifacts; runtime teams own launch stability, pool lifetimes, and graph replay \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. When both groups share one on-call rotation, counter worksheets prevent thrash: if launches/token improved but bytes/token worsened, rollback Layer~3; if both stalled, inspect descriptor versions at the framework boundary \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Layer~3 logs should include enough context to reproduce PK configs without re-running full search \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Threadblock lowering is the last human-readable checkpoint before opaque device code \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,patel2025llminference,dlbook}. When numerics diverge, compare threadblock dumps against eager reference at the same paging layout \citep{kwon2023vllm,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Do not tune softmax epsilons in runtime unless threadblock IR confirms carrier layout \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. This discipline keeps runtime hotfixes from masking compiler legality bugs \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{Python API surface.} Layer~1 exposes \texttt{yirage as yr}, \texttt{yr.new\_graph()}, tensor helpers, and \texttt{graph.superoptimize(backend=...)} entry points documented in upstream \texttt{docs/API\_REFERENCE.md} \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Application code should treat graphs as immutable after superoptimize: runtime consumes frozen configs \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,dlbook}. Mixing eager edits post-search breaks fingerprint caches and framework plugin contracts from Chapter~28 \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.

\textbf{Backend Manager routing.} Layer~2 selects installed backends (CUDA, CPU, NPU) and validates build flags from native extensions \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,patel2025llminference,dlbook,semianalysis2024tma,liu2024deepseekv3,taxbreak2026,memoryfloor2026,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,nvidia2024cudagraphs,kwon2023vllm,dlbook}. Import failures here surface as missing \texttt{yirage.core} or \texttt{libyirage\_runtime}---not as silent CPU fallbacks in production \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. CI should compile each target backend separately; triplet tests compare counters, not import success alone \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024xdna,amd2024aie,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{Search versus runtime split.} Layer~3 explores fusion and tiling; Layer~5 assumes a fixed plan \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,dao2022flashattention,semianalysis2024tma,dlbook,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Autotune during serving is an anti-pattern---it destroys graph capture stability and p99 latency \citep{memoryfloor2026,nvidia2024cudagraphs,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,dlbook}. Offline superoptimize plus pinned artifacts is the production pattern \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Layer~4 threadblock ops are the last IR-like surface before opaque device binaries---debug dumps here when PK numerics diverge from eager reference \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,patel2025llminference,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}.

\textbf{Observability per layer.} Layer~1 logs Python call sites; Layer~5 emits device timestamps and byte counters \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Correlate layer logs with framework traces from Chapter~28 using shared request ids \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}. Without correlation, runtime teams optimize kernels while queue time dominates p99 \citep{patel2025llminference,memoryfloor2026,kwon2023vllm,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook}.

Export a single JSON line per decode step containing layer id, registry id, artifact fingerprint, launches count, and byte estimate \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Aggregates belong in fleet dashboards; per step JSON enables postmortems \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Teach on-call engineers to filter on fingerprint first, registry id second, and framework version third \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Most regressions are mismatched artifacts, not mysterious hardware noise \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Layer handoffs deserve explicit interface tests in CI, not tribal knowledge. After superoptimize completes, assert that Layer four threadblock dumps include every carrier field the Layer five loader expects. After PersistentKernel construction, assert that warmup completes within a bounded step count and that steady decode reuses the same pool handles across one thousand tokens. When Layer two rejects a backend, the error message should name the missing native extension so developers do not confuse import success with PK readiness. Stack diagrams in documentation are useful, but automated boundary tests prevent silent drift when upstream YiRage refactors internal module names while preserving Python API spelling.

\paragraph{Multi-hardware delta.} Layer~4--5 differ sharply: CUDA PK uses graph capture and TMA-aware pools; XDNA uses static AIE binaries; CPU uses vectorized host loops \citep{amd2024xdna,amd2024aie,nvidia2022hopper,semianalysis2024tma,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Same Python API must not imply identical launch semantics \citep{yirage2026,chen2020compilerbasedhardw,amd2024aie,amd2024xdna,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook}.

\begin{example}
Search on laptop CPU then deploy CUDA PK without re-superoptimize: numerics may match while launches/token and bytes/token diverge---violates same-backend rule \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,amd2024xdna,amd2024aie,dlbook}.
\end{example}

\textbf{Worked layer walkthrough.} Start from a minimal decode graph in Layer~1 and execute superoptimize on the fleet GPU \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Inspect Layer~3 logs for chosen fusion boundaries and tile sizes, then dump Layer~4 threadblock lowering for the attention seam \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,patel2025llminference,dlbook}. Instantiate PersistentKernel in decode mode and run ten tokens at context 512, 2048, and 8192 \citep{yirage2026,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,dlbook}. Record launches per token, HBM bytes per token, and wall time per token separately for each bucket \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Only after Layer~5 counters flatten should you integrate the framework plugin from Chapter~28 \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Skipping the isolated PK microbench hides descriptor bugs that become production incidents under continuous batching \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}.

\textbf{Persistent kernel decode} uses \texttt{yr.PersistentKernel(mode="decode", backend=..., fallback\_backends=[...])} to select backend execution with graceful CPU/NPU fallbacks \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,liu2024deepseekv3,nvidia2024cudagraphs,amd2024xdna,amd2024aie,dlbook,dao2022flashattention,semianalysis2024tma,dao2023flashdecoding,nvidia2022hopper}. The \texttt{persistentkernel} object wraps a long-lived device program: internal loops over decoder layers replace per-op host launches \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,nvidia2024cudagraphs,dao2022flashattention,dlbook}. \texttt{mode="decode"} selects shape buckets and numerics carriers tuned for BS${=}1$ token steps rather than prefill-only tiles \citep{patel2025llminference,dao2022flashattention,dao2023flashdecoding,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,dlbook}.

\textbf{Same-backend rule.} Search and execute on the installed target---do not profile on CPU and ship CUDA kernels without re-search \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,amd2024xdna,amd2024aie,semianalysis2024tma,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. Fleet labels (GPU SKU, driver, CUDA version) must match \texttt{HardwareRegistry} entries used during \texttt{superoptimize} \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Warmup passes prime PK device memory pools and graph capture---cold first token latency is a runtime concern distinct from compiler fusion quality \citep{memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,dao2022flashattention,dlbook}.

Decode PK differs from training kernels in occupancy goals \citep{patel2025llminference,dao2022flashattention,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Training chases large batch matmul utilization; decode chases minimal bytes per token at batch one \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Persistent loops amortize layer weights across tokens inside one launch, but only if softmax and norm carriers remain on chip between layers \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,patel2025llminference,dlbook}. Profile carrier lifetimes explicitly when PK shows unexpected HBM traffic \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Cancellation and mid-sequence abort must release PK pool slots without leaking device memory \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Frameworks free paging blocks on cancel; runtime must mirror that lifecycle in pool accounting \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Long-running servers accumulate leaks if cancel paths skip pool teardown \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Stress tests should randomize cancel timing under continuous batching before fleet promotion \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\textbf{Fallback backends.} \texttt{fallback\_backends} enable dev laptops and CI smoke tests without forking model code \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024aie,amd2024xdna,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,dao2023flashdecoding,nvidia2022hopper,dlbook}. Production must pin primary backend to fleet SKU; fallbacks are for correctness checks, not performance baselines \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Accidentally serving through CPU fallback while GPU PK is available reads as mysterious fleet cost overrun \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{Device memory pools.} Layer~5 allocates persistent buffers for weights, KV views, and softmax carriers across decode steps \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,memoryfloor2026,taxbreak2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Framework paging passes block table pointers into PK launch descriptors---runtime must not copy KV to contiguous staging each token \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. Silent staging copies show up as bytes/token regressions with unchanged fusion plan \citep{memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}.

\textbf{Launch descriptor contract.} Each decode step receives slot mask, page table handle, step id, and sampling flags from the framework plugin \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. PK runtime validates descriptor version before launch---version skew must hard-fail, not produce wrong tokens \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Golden-token tests compare eager reference against PK for each descriptor version bump \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,dlbook}.

\textbf{Prefill versus decode PK modes.} \texttt{mode="decode"} selects persistent loops tuned for single-token advance; prefill may use separate superoptimize profiles \citep{patel2025llminference,dao2022flashattention,dao2023flashdecoding,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Mixing prefill tiles in decode PK wastes bandwidth---tag modes explicitly in artifact manifests \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Framework schedulers call the correct PK mode per engine iteration \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,dlbook}.

Shape buckets for decode should align with framework max batch width and max context policies \citep{kwon2023vllm,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Each bucket needs a warmed PK instance and a graph capture template \citep{memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Missing buckets fall back to slow paths that show up as tail latency spikes rather than mean regression \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Document bucket coverage in runbooks beside paging block size \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

Persistent kernel tuning is a memory lifetime problem disguised as a launch problem. Each decoder layer expects weight tensors, norm statistics, and attention softmax carriers to arrive at fixed addresses across steps so the device loop can branch minimally. When runtime allocates fresh staging buffers per token, bytes per token rises even though TaxBreak launch counts look excellent. Profile with memory transaction counters and verify that page table pointers from the framework pass straight into gather kernels without intermediate copies. When carriers spill to high bandwidth memory between layers, return to Layer three fusion legality before widening graph capture buckets.

Stress decode under adversarial scheduler patterns: alternating prefill and decode on the same slot, random cancels at ten percent of steps, and context growth from five hundred twelve to eight thousand one hundred ninety two within one session. PK pools must survive these patterns without monotonic growth in reserved bytes. Framework plugins should not assume PK warmup happens once per process lifetime if workers restart frequently in Kubernetes; document whether warmup is idempotent and how long it blocks readiness probes. Serving teams often tune liveness thresholds without accounting for first capture latency, then blame runtime for pod churn that is actually missing warmup in the probe design.

\paragraph{Multi-hardware delta.} CUDA PK pairs with CUDA Graph buckets; XDNA PK loads AIE binaries with static DMA chains; CPU PK uses OpenMP-blocked loops \citep{nvidia2024cudagraphs,amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,semianalysis2024tma,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. Identical Python \texttt{persistentkernel} constructor arguments may map to different launch paths \citep{yirage2026,chen2020compilerbasedhardw,amd2024aie,amd2024xdna,patel2025llminference,dlbook}.

\begin{example}
Decode cell (BS${=}1$, ctx${=}2048$): PK launch drops launches/token from TaxBreak hundreds to single digits while framework paging unchanged---runtime owns launch collapse, not batching \citep{taxbreak2026,yirage2026,kwon2023vllm,patel2025llminference,memoryfloor2026,nvidia2024cudagraphs,chen2020compilerbasedhardw,dao2022flashattention,dlbook}.
\end{example}

\textbf{PK versus CUDA graphs alone.} Memory-Floor style graph replay removes host launch storms but cannot fix illegal fusion \citep{memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. PersistentKernel combines graph stability with compiler chosen fusion inside the replayed region \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,taxbreak2026,memoryfloor2026,patel2025llminference,dlbook}. Teams seeing graph wins without byte wins should escalate to superoptimize rather than adding more graph buckets \citep{memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. PK is not a replacement for paging or batching; it replaces per op launches inside the decode cell \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{MoE and expert parallel hooks.} Large MoE models add expert parallel communication below the PK entry point \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Open DeepEP components illustrate all to all patterns that must compose with PK memory pools rather than allocate per token \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Profile EP separately from PK fusion when bytes/token regress on MoE checkpoints \citep{liu2024deepseekv3,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\section{superoptimize handoff}
\label{sec:ch29_superoptimize_handoff}

\textbf{Superoptimize handoff} bridges \texttt{graph.superoptimize(backend=...)} outputs to runtime launch: fused $\mu$Graphs, fingerprint verification, and cached kernel configs \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,nvidia2024cudagraphs,dao2023flashdecoding,dlbook}. Layer~3 search emits a serialized plan; Layer~5 loads it into PK launch templates \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Framework hooks (Chapter~28) must not strip metadata required for capture or MegaKernel legality \citep{yirage2026,nvidia2024cudagraphs,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook,kwon2023vllm,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}.

Handoff is where many projects stall: search finishes, a demo looks fast, but no artifact pipeline reaches serving \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Treat superoptimize output like a container image: digest, manifest, signature, and rollback tag \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Framework plugins should load by digest at process start, not by mutable local paths \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Canary deploys swap digest on a fraction of replicas while comparing counter dashboards \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Without digest discipline, two replicas silently run different PK programs under the same endpoint \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}.

\textbf{Fingerprints and cache keys.} Superoptimize caches results keyed by IR fingerprint plus backend id and hardware registry hash \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Framework upgrades that change page-table ABI must bump cache salts or risk loading incompatible PK binaries \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. CI should verify fingerprint mismatch fails closed---never silently fall back to unfused eager \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,dlbook}.

\textbf{Graph to kernel packaging.} Threadblock lowering produces device-specific binaries or CUDA C++ sources consumed by PK runtime \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,amd2024xdna,amd2024aie,patel2025llminference,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,dlbook}. Handoff artifacts include tile sizes, double-buffer depth, and softmax carrier layout \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,patel2025llminference,dlbook}. Missing carrier metadata passes unit tests on square tensors and fails on paged decode \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dao2022flashattention,dao2023flashdecoding,dlbook}.

\textbf{Warmup and steady state.} First PK launch may compile, allocate pools, and capture graphs; steady decode reuses them \citep{memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Fleet benchmarks must separate warmup from sustained tokens/s \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Production SLOs often ignore first-token compile; decode p99 must use warmed PK state \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,nvidia2024cudagraphs,dlbook}.

\textbf{Artifact storage layout.} Store superoptimize outputs as versioned bundles: fingerprint, registry id, backend, blob, and manifest JSON \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Framework plugins fetch bundles by hash at startup---avoid embedding blobs inside Python wheels without hash checks \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Rollback is redeploy prior bundle, not revert framework scheduler alone \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dlbook}.

\textbf{Legality metadata handoff.} Export softmax carriers, page-table layouts, and graph-capture flags as manifest fields \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. Runtime refuses launch if manifest missing required fields---fail closed \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. This prevents partial exports from Chapter~28 plugins \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}.

Manifest evolution requires semver: breaking carrier layout bumps major version; additive fields bump minor \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Framework plugins declare supported manifest major before loading artifacts \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Mixed major versions across replicas is a deployment error, not a runtime tuning opportunity \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Chapter~30 runbooks formalize this three party release train \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dlbook}.

Artifact promotion should resemble a binary signing pipeline. Search cluster produces a bundle; staging loads it read only; golden token harness compares hashes against eager reference on three context buckets; production receives a digest pin in the framework config. Never promote because a demo video looked fast on one laptop. Store the promotion checklist JSON beside the bundle so auditors can reconstruct who signed off on launches per token and bytes per token deltas. When a framework upgrade changes only scheduler policy but not paging ABI, artifacts may carry forward; when block table layout changes, invalidate caches even if IR fingerprint unchanged. Handoff meetings should read manifest fields aloud until boring repetition makes missing fields impossible.

Offline search and online runtime must share a single hardware registry revision checked into version control. Drift between search cluster registry and serving cluster labels produces plans that are legal yet slow, which is harder to detect than hard failures because tokens still look plausible. Automate a nightly job that compares detect current chip output on every node class against expected registry ids and pages when mismatches exceed zero. Superoptimize caches keyed only on IR miss these failures until bytes per token creep upward over weeks.

\paragraph{Multi-hardware delta.} Cache entries are not portable across backends---re-superoptimize per target even when IR matches \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook,semianalysis2024tma,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}.

\begin{example}
Fingerprint hit loads cached PK plan; framework block size change without re-search yields wrong KV gathers---cache must invalidate on paging ABI change \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}.
\end{example}

\textbf{Handoff checklist before fleet deploy.} Verify manifest lists backend id, registry id, fingerprint, paging ABI version, and softmax carrier schema \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,patel2025llminference,dlbook}. Run golden tokens on three context buckets with warmed PK \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}. Compare launches/token and bytes/token against prior artifact in the same dashboard row \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Sign off only when both counters move or regressions are explained with profiler evidence \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dlbook}. Store the checklist JSON beside the artifact bundle for audit \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\section{hw registry runtime}
\label{sec:ch29_hw_registry_runtime}

\textbf{Hw registry runtime} (\texttt{HardwareRegistry}, \texttt{detect\_current\_chip()}) supplies chip models for search configs---Hopper vs XDNA vs M3 Max tile candidates \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,patel2025llminference,semianalysis2024tma,dlbook,kwon2023vllm,liu2024deepseekv3,taxbreak2026,memoryfloor2026,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,nvidia2024cudagraphs}. The \texttt{hardwareregistry} maps detected devices to search spaces: SM count, TMA availability, L2 size, NPU tile SRAM, CPU vector width \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,nvidia2022hopper,semianalysis2024tma,patel2025llminference,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,dlbook}. \texttt{detect\_current\_chip()} runs at import or first superoptimize---runtime detection must align with fleet labels used in framework schedulers \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,amd2024xdna,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,dao2023flashdecoding,nvidia2022hopper,dlbook}.

\textbf{Search space pruning.} Registry entries prune illegal tile shapes before autotune explosion \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,amd2024xdna,amd2024aie,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook,kwon2023vllm,liu2024deepseekv3,nvidia2022hopper,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,dlbook}. Hopper profiles enable TMA double-buffer candidates; XDNA profiles restrict to static DMA chains \citep{nvidia2022hopper,semianalysis2024tma,amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Wrong chip detection yields legal-but-slow plans---profile bytes/token on representative ctx buckets \citep{memoryfloor2026,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Registry entries should be versioned documents checked into the same repo as artifact manifests \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. When hardware vendors ship new SKUs, add entries before marketing promises fleet migration \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,dlbook}. Each entry lists peak bandwidth, on-chip capacities, and forbidden fusion patterns for that target \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,nvidia2022hopper,semianalysis2024tma,patel2025llminference,dlbook}. Search reads these as hard constraints, not hints \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Detect current chip at process start and log the resolved registry id in every performance trace \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Post-incident review should filter traces by registry id to spot partial rollouts \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Mixed ids behind one load balancer explain bimodal latency histograms \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Registry maintenance is a hardware catalog discipline, not a one time bootstrap script. Each entry documents peak bandwidth, on chip SRAM, vector width, tensor engine availability, and forbidden fusion patterns for that SKU. When vendors ship new accelerators, add entries before marketing announces fleet migration dates. Search reads entries as hard constraints; runtime logs the resolved id on every decode step so capacity planners can correlate artifact lines with instance types. Retire entries only after all pinned artifacts migrate and dashboards show zero traffic on the old id for two release cycles.

Virtualized and partitioned GPUs distort detection unless runbooks override generic ids with fleet labels from node metadata. Multi instance GPU slices may report parent SKU names that mislead tile search toward shapes that fit the parent but not the slice memory budget. Time shared virtual machines may hide NPU presence entirely until first superoptimize call. Document override procedures and require platform teams to validate overrides quarterly against vendor tools output.

\textbf{Fleet inventory sync.} Kubernetes node labels, cloud instance types, and driver versions should map to registry ids \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. \textbf{Registry-driven autotune budgets.} Cap search trials per chip model to keep CI bounded \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook,semianalysis2024tma,amd2024xdna,amd2024aie,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. Registry metadata feeds roofline estimates that prune hopeless tile shapes before device trials \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,semianalysis2024tma,dlbook,taxbreak2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. \label{sec:ch29_deps_build_native}

\textbf{Deps build native}: \texttt{import yirage} requires \texttt{yirage.core} + \texttt{libyirage\_runtime}---no Python-only native build path \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Submodule init: \texttt{git submodule update --init --recursive deps/YiRage}; build via \texttt{YIRAGE\_BACKEND=cuda pip install -e deps/YiRage} (see upstream \texttt{docs/INSTALLATION.md}) \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,amd2024xdna,amd2024aie,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,nvidia2024cudagraphs,patel2025llminference,dlbook}.

\textbf{Native extension layout.} Python package wraps C++ runtime under \texttt{deps/YiRage/yirage/} with compiled \texttt{yirage.core} module linking \texttt{libyirage\_runtime} \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,amd2024xdna,amd2024aie,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,nvidia2024cudagraphs,dlbook}. Missing .so files import as pure Python stubs that fail at first PK call---smoke tests must invoke \texttt{PersistentKernel}, not only \texttt{import yirage} \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Building from source ties book examples to reproducible hashes \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Pin submodule commit in release notes whenever prose references API symbols \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Nested submodules may require recursive init; document network and disk expectations in deps README \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Agents editing prose should not hand modify generated native sources; changes belong upstream in YiRage \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Wheel-based installs are convenient for demos but hide compiler flags \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Production should prefer editable installs from pinned submodule with recorded nvcc and driver versions \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. When wheels and submodule builds diverge, counter worksheets become incomparable across environments \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\textbf{Backend build flags.} \texttt{YIRAGE\_BACKEND=cuda|cpu|npu} selects compile targets; nested submodules may pull vendor SDKs \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,patel2025llminference,dlbook,semianalysis2024tma,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. Docker images for serving should bake the same backend flags as search CI \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Drift between build and runtime containers is a common PK load failure \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{Version pin with book repo.} Pin submodule SHA in \texttt{.gitmodules} or recorded commit; Chapter~30 runbooks require matching \texttt{deps/YiRage} revision with framework plugin ABI \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Book examples reference APIs at pinned SHA---update prose when bumping submodule \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{Developer workflow.} Typical loop: edit IR or search config $\rightarrow$ \texttt{superoptimize} on target SKU $\rightarrow$ run PK decode microbench $\rightarrow$ export artifact hash for framework plugin \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Skipping native rebuild after C++ runtime change produces stale .so mismatches \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{CI smoke template.} Minimal pipeline: submodule init, backend-specific compile, import yirage, superoptimize tiny graph, PersistentKernel decode ten tokens, assert golden hash \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Extend with BS${=}1$ counter worksheet from Chapter~1 before promoting artifacts to staging \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Docker and reproducibility.} Lock CUDA driver, toolkit, and YiRage SHA in serving images \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Non-reproducible PK builds frustrate bisect when fleet regressions appear \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}. Record compiler flags in artifact manifest \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}.

Serving images should compile YiRage once in the build stage and copy artifacts into the runtime stage \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Runtime stage installs matching driver user space libraries only; never compile on production pods \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Init containers can verify artifact digest against manifest before accepting traffic \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. This pattern prevents silent drift between search cluster and serving cluster \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Native build failures often masquerade as Python packaging issues. Missing nvcc, wrong driver user space libraries, or stale build directories produce import success followed by opaque failures at first PersistentKernel call. CI must execute the full smoke template on the same backend flag as production containers, including ten token decode and golden hash comparison. Developers on laptops without GPUs may compile CPU backend for unit tests but must never promote CPU searched artifacts to CUDA serving. Record compiler flags, submodule SHA, and backend enum in artifact manifests so bisect can distinguish runtime code changes from toolchain drift.

Book readers reproducing examples should init the submodule recursively, install editable with explicit backend flag, and run PK decode microbench before reporting API mismatches. Agents editing prose should not hand modify generated native sources; changes belong upstream in YiRage with submodule bump and manifest semver bump in this repository. When documentation and profiler traces disagree, trust traces and file issues against runtime launch paths.

\paragraph{Multi-hardware delta.} CUDA builds need compatible nvcc and driver; XDNA builds need Ryzen AI SDK paths documented upstream \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,semianalysis2024tma,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. CI matrices should include at least one compile plus PK smoke per supported backend \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024aie,amd2024xdna,dlbook}.

\begin{example}
\texttt{pip install -e deps/YiRage} on dev laptop without CUDA toolkit: import succeeds, PK decode fails at runtime---gate merges on PK smoke, not import \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,dlbook}.
\end{example}

\textbf{Local dev versus fleet parity.} Developers may compile CPU backend locally for unit tests while fleet runs CUDA PK \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024aie,amd2024xdna,dlbook}. Never promote CPU searched artifacts to GPU serving \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Provide a small GPU CI runner that executes the same smoke template as production \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Document driver and toolkit versions in deps README when bumping submodule SHA \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Book readers should clone submodule recursively before reporting API mismatches \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\section{Key Takeaways}
\label{sec:ch29_key_takeaways}

\begin{itemize}
\item \textbf{Runtime is Layer~5.} Compilers emit; Persistent Kernels execute with device memory discipline \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.
\item \textbf{Same-backend search.} Profile and ship on the target SKU; \texttt{hardwareregistry} must match fleet labels \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,amd2024xdna,amd2024aie}.
\item \textbf{Handoff is contractual.} Fingerprints, page-table ABI, and softmax carriers must survive \texttt{superoptimize} to PK launch \citep{yirage2026,dao2022flashattention,kwon2023vllm,memoryfloor2026,taxbreak2026}.
\item \textbf{Submodule is source of truth.} \texttt{deps/YiRage} native build required; smoke-test PK decode in CI \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.
\item \textbf{Counters at runtime.} Measure launches/token and bytes/token after PK warmup, not only import or search logs \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dlbook}.
\end{itemize}

\section{Conclusion}
\label{sec:ch29_conclusion}

YiRage runtime closes the gap between fused IR and fleet decode: Layer~5 \texttt{persistentkernel} programs collapse launches/token; device memory pools and graph hooks address bytes/token and orchestration tax \citep{yirage2026,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,chen2020compilerbasedhardw,liu2024deepseekv3,dao2022flashattention,dlbook,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper}. Treat \texttt{superoptimize} artifacts, registry ids, and native builds as release-train inputs alongside framework paging policy \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Before Chapter~30, complete one Layer~5 worksheet: warmed PK, three context buckets, launches and bytes per token logged, manifest hash pinned \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dlbook}. Chapter~30 integrates this layer with vLLM-class frameworks and Chapter~12 execution modes into one production workflow \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,dlbook,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}.
