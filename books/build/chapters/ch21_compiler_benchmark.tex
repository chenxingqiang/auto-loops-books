The production surprise at \textbf{compile overhead} is not peak FLOPs; it is illegal tier placement: compile spilling to HBM/DDR while overhead still pays launch or DMA tax every token \citep{patel2025llminference,dlbook}.

Trace residency: compile producers must feed overhead consumers without a DDR round trip \citep{dao2022flashattention,yirage2026}. Silent spill reads as ``mysterious'' p99 regression \citep{patel2025llminference,semianalysis2024tma}.

Operational fix: diff IR or CUDA for new allocs; count launches/token (GPU) or DMA edges (NPU); require graph capture legality on GPU and static legality on XDNA \citep{nvidia2024cudagraphs,amd2024aie}.

\paragraph{Review gate.} Attach roofline or buffer accounting for compile on target SKU before merge \citep{memoryfloor2026,yirage2026}.

\section{Chapter Summary}
\label{sec:ch21_summary}

Unified AI Compiler Analysis and Cross-Hardware Benchmark succeeds when hardware constraints drive schedules and decode metrics prove relief \citep{yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw}. \textbf{compiler ratings} exposes a cross-hardware fracture: the same fused subgraph legal on Hopper can be rejected outright on XDNA or CPU cache budgets.

Binding software to silicon: compiler passes must encode compiler/rating legality before codegen, not as comments in hand-written kernels \citep{patel2025llminference,dlbook}. YiRage runs target-aware checks against chip models from Chapter~3; manual stacks should treat failed legality as a release blocker, not a backend TODO.

When compiler ratings disagrees with microbenchmarks, trust fleet telemetry: fix orchestration and tier placement first, then revisit MMA tile shapes \citep{patel2025llminference,dlbook}.

\paragraph{Review gate.} Before merging compiler changes affecting compiler ratings, attach roofline or NPU buffer accounting showing compiler residency fits on-chip for the target SKU \citep{patel2025llminference,dlbook}.

\paragraph{Worked contrast.} A Hopper decode cell may legalize compiler with TMA-backed double buffering while an XDNA cell requires the same math expressed as static tile assignments---identical FLOPs, different bytes/token because DDR traffic differs by an order of magnitude \citep{patel2025llminference,dlbook}.

The usual Unified AI Compiler Analysis and Cross-Hardware Benchmark failure at \textbf{compile overhead} is importing a CUDA mental model and expecting runtime scheduling to hide illegal buffer lifetimes.

Hardware constraint: compile and overhead must be co-designed. On GPUs that means shared-memory/TMA budgets and warp-grain barriers; on XDNA/AIE it means static tile buffers, DMA chains, and carrier slots with compile-time lifetimes; on CPUs it means L2/L3 blocking and NUMA-local KV placement \citep{patel2025llminference,dlbook}. Decode at batch-1 keeps arithmetic intensity on the memory-bandwidth slope---micro-optimizing isolated GEMMs rarely moves p99 latency.

Compiler takeaway for compile overhead: lower the same decoder IR, but predicate passes on target residency tables---never ship a GPU-only schedule to NPU fleets \citep{patel2025llminference,dlbook}. Success metrics: bytes/token, launches/token, and p99 decode latency; compare GPU, CPU, and NPU cells on identical context lengths.
