\label{chap:ch30}

Production decode is a \emph{three-way} contract: the inference \textbf{framework} schedules requests and owns KV paging; the \textbf{compiler} (YiRage middle-end + \texttt{superoptimize}) owns intra-step fusion and layout; the \textbf{runtime} (Persistent Kernel Layer~5) owns launches, device memory, and backend fallbacks \citep{yirage2026,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dlbook,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper}. When p99 latency stalls, teams that optimize only one leg recreate Chapter~1's framework fragmentation at warehouse scale \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Part~VIII closes by making that contract explicit: every integration meeting draws the decode step \textbf{boundary} first, then assigns owners and counters \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Chapter~28 mapped framework buckets and scheduler policy; Chapter~29 descended into YiRage Layer~5 Persistent Kernel runtime \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dlbook}. This chapter stitches the three parties into one production workflow: who owns prefill versus decode handoff, how vLLM paging composes with YiRage fused cores, which execution mode applies per subgraph on a heterogeneous fleet, and how regression gates prevent silent drift across submodule bumps \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,dlbook}. The book's industrial arc ends here---not with another isolated kernel trick, but with deployable co-design anchored on decode goodput counters \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

Readers should leave with a release train mental model. Framework teams ship scheduler and paging policy; compiler teams ship fused plans and manifest schemas; runtime teams ship PK launch descriptors and pool invariants \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. No party optimizes in isolation: a framework upgrade that changes block table layout without bumping artifact cache salts breaks compiler handoff; a compiler win that ignores paging ABI wastes bytes per token; a runtime regression that leaks pool slots masquerades as scheduler overload \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Co-design means shared worksheets, shared request ids in traces, and shared rollback pins---not shared blame in postmortems \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Fregly mechanical sympathy at the three-way boundary treats each handoff as an API with semver, golden tests, and profiler evidence \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dao2022flashattention,nvidia2024cudagraphs,dlbook}. Chapter~12 execution modes reappear here as fleet policy: eager hulls for sampling branches, CUDA Graph buckets for launch-bound cells, MegaKernel and Persistent Kernel paths for byte-bound decoder cores \citep{nvidia2024cudagraphs,yirage2026,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,dlbook}. DeepSeek-class disaggregation adds a fourth axis---prefill and decode pool routing---but the same boundary discipline applies inside each pool's decode step \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Three questions gate every co-design review before fleet promotion. First, does the responsible party for the moving counter actually own the lever being tuned \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Second, do handoff manifests survive framework, compiler, and runtime version bumps without silent fallback \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Third, does triplet regression on GPU, CPU, and NPU backends show bytes per token and launches per token moving together or explained separately \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,dlbook}. If any answer is no, pause feature work and fix the contract before tuning batch windows \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

This closing chapter is deliberately operational. Earlier parts taught rooflines, fusion legality, dialect lowering, and autotune search \citep{chen2020compilerbasedhardw,yirage2026,semianalysis2024tma,patel2025llminference,dlbook}. Part~VIII asked where those ideas attach in servers humans operate \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. The answer is never a wholesale framework replacement---it is a negotiated boundary with versioned handoffs and regression gates \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Teams that skip co-design and chase isolated kernel records on leaderboard shapes reproduce the fragmentation Chapter~1 diagnosed \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Co-design is the antidote measured in the same counters from prompt to token \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Warehouse-scale serving adds constraints single-GPU labs hide \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,dlbook}. Continuous batching interleaves unrelated clients; cancel paths free slots unpredictably; disaggregated pools move KV across networks \citep{kwon2023vllm,liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Compiler and runtime optimizations must tolerate that chaos without leaking memory or violating paging ABI \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Framework optimizations must expose enough telemetry for compiler teams to see in-step counters, not only fleet aggregates \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Co-design is the discipline of making those telemetry paths first-class product requirements \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

Leadership reviews should ask which party moved which counter last quarter, not which kernel microbenchmark improved \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Roadmaps that fund framework features without compiler handoff metadata or runtime descriptor tests recreate integration debt \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Conversely, compiler teams that ship artifacts without framework plugin support plan fail at the boundary \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Co-design funding follows counter movement with named owners \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. This chapter gives vocabulary for those staffing and budget conversations \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Part~VIII therefore completes the narrative arc from mindset and hardware bounds through compiler stacks to deployable serving \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Readers who implement only one chapter should still implement this one if they operate fleets \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Theory without co-design contracts becomes shelfware; serving without counters becomes guesswork \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. The combined worksheet---boundary diagram, integration digest, mode map, triplet rows---is the minimum viable artifact set for production decode \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dlbook}.

The \textbf{framework} owns everything outside the decode step box: continuous batching, paged KV block tables, speculative decoding policy, disaggregated routing between prefill and decode pools, and client-facing SLO dashboards \citep{kwon2023vllm,patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,dlbook}. Framework engineers optimize queue depth, slot assignment, and memory capacity---not softmax carrier layout inside a fused attention kernel \citep{patel2025llminference,kwon2023vllm,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,dlbook}. When fleet tokens per second rise but bytes per token inside the step is flat, the framework did its job; the compiler and runtime did not \citep{memoryfloor2026,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

The \textbf{compiler} owns intra-step fusion and residency: which operators merge, which tensors stay on chip between layers, which layouts match paging gathers, and which plans are legal for graph capture \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Compiler output is not PyTorch source---it is versioned artifacts with fingerprints, manifest fields, and hardware registry ids from Chapter~29 \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Framework plugins consume those artifacts; they must not silently strip softmax carrier metadata or paging ABI version fields \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.

The \textbf{runtime} executes compiler plans on device: Persistent Kernel loops, memory pool lifetimes, CUDA Graph capture buckets, backend fallbacks, and descriptor validation at each decode step \citep{yirage2026,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,patel2025llminference,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Runtime engineers read nsys and pool accounting; they do not rewrite scheduler batching policy to mask launch storms \citep{taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. When launches per token regress after a runtime bump, bisect artifact digest and descriptor version before touching framework code \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Draw the decode \textbf{boundary} on every architecture diagram: from slot mask and block table handle ingress through final hidden state before sampling \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,dlbook}. Inside the box, compiler and runtime own bytes per token and launches per token; outside, framework owns queue time and paging efficiency \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Integration meetings that skip the boundary redraw recreate Chapter~1 fragmentation: compiler teams tune GEMMs while scheduler queue time dominates p99 \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}.

\textbf{On-call triage by party.} Queue wait up, in-step ms flat: framework scheduling or capacity \citep{patel2025llminference,kwon2023vllm,dlbook}. In-step bytes up, launches flat: compiler fusion or layout regression \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Launches up, bytes flat: runtime graph capture or PK descriptor regression \citep{taxbreak2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. All three up: likely boundary mismatch---paging ABI changed without artifact invalidation \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.

\textbf{Org design implications.} Staff framework, compiler, and runtime teams with separate counter dashboards but shared release train \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Shared on-call without shared worksheets devolves into thrash: each party tunes its leg while p99 stalls \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Co-design standups read one row per request id: queue ms, in-step ms, launches, bytes, artifact digest \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\begin{example}
Regression after framework block size change: compiler cache hit loads stale PK plan---boundary contract requires cache salt bump on paging ABI change, not runtime knob tuning \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.
\end{example}

Industrial teams document the split in runbooks linked from fleet config repos \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. New hires should trace one decode token through framework slot assignment, compiler artifact load, runtime PK launch, and sampling egress before proposing optimizations \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. That trace is the co-design literacy test for Part~VIII \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Release trains should version the responsibility table alongside framework, compiler, and runtime artifacts \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. When a new feature spans two parties---for example speculative decoding in the framework plus extra PK launches in runtime---file a joint design doc with counter acceptance criteria before implementation \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,dlbook}. Escalation paths should name DRI per party, not a generic platform team that owns none of the levers \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Quarterly audits replay archived traces from peak traffic week and verify each party still matches the published table \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

Contract tests between parties catch boundary drift early \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Framework publishes paging ABI fixtures; compiler asserts superoptimize output loads against fixtures; runtime asserts PK descriptors accept fixture block tables \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Breaking any contract test blocks merge until semver bump and coordinated release notes ship \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. This is how co-design scales beyond a single heroic integration engineer \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Worked boundary exercise.} Pick one production model and one fleet SKU \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. List ten operations in decode order from scheduler slot assignment through sampling \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Mark each operation framework, compiler, or runtime owned \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Attach one counter per party \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Run one request through traces and verify marks match reality \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Mismatches indicate undocumented boundary creep---fix the runbook before optimizing \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Repeat after every major framework or submodule bump \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dlbook}.

Vendor and open-source upgrades flow through the same table \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. CUDA driver updates may change graph capture behavior; vLLM releases may change block tables; YiRage bumps may change manifest majors \citep{nvidia2024cudagraphs,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Assign each upgrade type a default owner and regression template so on-call does not debate responsibility during incidents \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook}. The responsibility split is living documentation, not a one-time slide \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\textbf{Decode pipeline handoff} sequences three phases with different owners: \textbf{prefill} ingestion, per-token \textbf{decode} compute, and \textbf{sampling} egress \citep{patel2025llminference,dao2022flashattention,dao2023flashdecoding,kwon2023vllm,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,nvidia2022hopper}. Prefill often stays framework-orchestrated because shapes vary widely and batching policy dominates \citep{kwon2023vllm,patel2025llminference,dao2022flashattention,dao2023flashdecoding,yirage2026,chen2020compilerbasedhardw,dlbook}. Decode steps enter compiler-owned cores when bytes per token and launches per token dominate at batch one \citep{memoryfloor2026,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,dlbook}. Sampling and speculative branches remain \emph{eager hulls} outside CUDA Graph capture per Chapter~12 legality rules \citep{nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{Prefill handoff.} Framework schedulers batch prompt tokens, allocate paging blocks, and may route long prompts to disaggregated prefill pools \citep{kwon2023vllm,liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Compiler involvement during prefill varies: some stacks superoptimize prefill profiles separately from decode; others reuse decode tiles inefficiently \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,dao2023flashdecoding,dlbook}. Document which PK mode the framework invokes at prefill-to-decode transition---mixing modes without manifest tags causes silent bandwidth waste \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,memoryfloor2026,taxbreak2026,dlbook}. Prefill completion must commit KV layout metadata the decode artifact expects: block table dimensions, head layout, and rope cache handles \citep{kwon2023vllm,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{Decode handoff.} Each decode iteration passes slot mask, page table pointer, step id, and hidden state buffers into the compiler-owned cell \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. Runtime validates descriptor version, launches PK or graph replay, and returns logits or hidden states to the framework \citep{yirage2026,nvidia2024cudagraphs,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Handoff latency includes framework Python overhead, plugin marshalling, and device sync---profile each separately before blaming fusion quality \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Continuous batching may swap slots mid-flight; decode artifacts must tolerate dynamic slot masks without re-capture on every step when graph mode is enabled \citep{kwon2023vllm,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,dlbook}.

\textbf{Sampling handoff.} Top-k, top-p, temperature, and stop criteria execute outside captured regions \citep{nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Speculative decoding adds draft and verify loops that re-enter decode cores multiple times per user token \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,dlbook}. Compiler plans must declare which tensors sampling reads and whether logits stay on device \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Framework policy changes to sampling should not require re-superoptimize unless carrier layout changes \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Handoff contracts deserve semver and golden-token tests at each phase boundary \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. CI should run prefill-only, decode-only, and full generate paths with shared artifact digests \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Fail closed when manifest major version mismatches---never silently fall back to eager per-op decode \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}.

DeepSeek-class \textbf{disaggregated} stacks extend handoff across network boundaries: prefill pools write KV to remote decode pools \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Bytes per token then includes transfer cost; compiler and runtime optimizations inside decode pools still matter but must compose with transfer scheduling \citep{liu2024deepseekv3,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Profile end-to-end goodput, not in-pool ms alone \citep{patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,dlbook}.

\begin{example}
Prefill completes at ctx${=}8192$; decode PK warmed only to ctx${=}2048$ bucket---tail latency spikes on long sessions until runtime bucket table matches framework max context policy \citep{kwon2023vllm,memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.
\end{example}

\paragraph{Multi-hardware delta.} Prefill may use wide batch tiles on GPU while decode PK targets batch one; CPU and NPU backends need separate handoff manifests per Chapter~29 same-backend rule \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,semianalysis2024tma,taxbreak2026,memoryfloor2026,kwon2023vllm,nvidia2022hopper,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,dlbook}.

Handoff latency budgets belong in SLO documents with separate allocations for framework marshalling, plugin sync, and device execution \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Teams that collapse these into one in-step millisecond metric misallocate optimization effort \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Log each segment with shared request id so postmortems identify whether regression entered at prefill commit, decode launch, or sampling return \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Long context sessions stress handoff most at prefill-to-decode transition when bucket tables must expand---warm expanded buckets during prefill completion, not on first decode token \citep{kwon2023vllm,memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{Timing diagram discipline.} Draw prefill, decode, and sampling phases on one timeline per request class \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,dlbook}. Mark where KV becomes visible to decode workers in disaggregated setups \citep{liu2024deepseekv3,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Mark where logits cross back to framework for sampling \citep{nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Ambiguous timing diagrams produce integrations that double-copy KV or synchronize too aggressively \citep{kwon2023vllm,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Review diagrams in design docs with the same rigor as API schemas \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

Streaming responses add another handoff: partial tokens must egress while decode continues \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Framework streaming APIs should not force synchronous device sync each token unless product requires it \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,dlbook}. Compiler and runtime plans should declare which steps tolerate async logits delivery \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Profile streaming workloads separately from batch benchmark harnesses \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Tail latency under streaming often traces to handoff sync policy, not fusion quality \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,taxbreak2026,dlbook}.

\section{vllm yirage stack}
\label{sec:ch30_vllm_yirage_stack}

\textbf{Vllm yirage stack} is the reference \textbf{integration} pattern for Part~VIII: vLLM continuous batching and paged KV plus YiRage fused decode core invoked each step \citep{kwon2023vllm,yirage2026,patel2025llminference,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper}. Measure queue wait and in-step counters separately---fleet tokens per second can rise while launches per token inside the step stays at TaxBreak hundreds \citep{taxbreak2026,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,dlbook}. The stack does not replace vLLM; it shrinks the decode cell vLLM calls \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{Integration architecture.} Typical flow: vLLM scheduler assigns slots and block tables; a custom attention or decoder backend loads YiRage artifact by digest at worker start; each decode step marshals descriptors into PersistentKernel \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dlbook}. Plugin code lives at the framework boundary documented in Chapter~28 partial-export bucket \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dlbook}. Keep plugin surface minimal: slot mask, page table handle, step id, artifact digest---not raw tensor graphs per request \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{vLLM responsibilities retained.} Paging, continuous batching, speculative decoding orchestration, and disaggregated routing stay in vLLM \citep{kwon2023vllm,patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,dlbook}. YiRage does not reimplement block allocators or queue policies \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dlbook}. When KV utilization drops, tune framework batch windows before re-running superoptimize \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. When bytes per token inside the step stalls, escalate to compiler and runtime \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,dlbook}.

\textbf{YiRage responsibilities absorbed.} Fusion across attention, norm, and MLP seams; softmax carrier residency; PK launch collapse; graph capture buckets aligned to vLLM shape policies \citep{yirage2026,dao2022flashattention,chen2020compilerbasedhardw,nvidia2024cudagraphs,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,dlbook}. Artifacts pin at worker boot from \texttt{deps/YiRage} submodule SHA recorded in fleet config \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Submodule bumps trigger regression triplet from Section~\ref{sec:ch30_production_regression} before traffic shift \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook}.

\textbf{Integration testing template.} Stand up vLLM worker with pinned artifact; run BS${=}1$ decode at ctx 512, 2048, 8192; log queue ms, in-step ms, launches per token, bytes per token \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Compare against vLLM plus FlashAttention baseline on same SKU \citep{dao2022flashattention,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Promote only when both in-step counters improve or regressions are explained with profiler evidence \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dlbook}. Run cancel and resume stress under continuous batching before fleet promotion \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

Open components under \texttt{deps/DeepEP} and \texttt{deps/FlashMLA} illustrate MoE and MLA patterns that must compose with vLLM paging rather than bypass it \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Integration docs should show where expert parallel all-to-all sits relative to PK entry---below scheduler, adjacent to decode cell \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\begin{example}
vLLM upgrade changes block table layout: YiRage cache hit serves stale gathers---integration contract requires artifact invalidation on paging ABI bump, not plugin hotfix alone \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.
\end{example}

\textbf{Operational ownership.} vLLM SREs own scheduler SLOs and paging capacity; YiRage owners own artifact pipeline and PK stability \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Shared dashboards correlate framework version, artifact digest, and runtime descriptor version per request id \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Integration failures at this boundary are the most common Part~VIII incident class \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Rollout playbooks for vLLM plus YiRage integration should mirror compiler artifact promotion: dev cluster golden tokens, staging canary at five percent traffic, production shift with rollback digest pinned \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Document which vLLM worker flags enable the custom backend and which remain framework defaults \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Training new operators on integration dashboards prevents conflating queue depth spikes with PK failures \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,dlbook}. When disaggregated prefill is enabled, verify YiRage artifacts on decode pool workers separately from prefill pool configs \citep{liu2024deepseekv3,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\textbf{Plugin minimization principle.} The vLLM integration surface should stay thin enough to audit in one code review \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Heavy logic in plugins duplicates compiler and runtime responsibilities and forks across teams \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Push fusion and residency decisions into superoptimize artifacts; push launch stability into Persistent Kernel runtime \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Plugins marshal descriptors and handle error propagation only \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. When plugins grow past that scope, refactor boundary before next fleet promotion \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Capacity planning for integrated stacks counts both framework KV footprint and runtime pool high-water marks \citep{kwon2023vllm,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Scaling replicas without warming PK buckets on new nodes produces temporary tail spikes \citep{memoryfloor2026,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Autoscalers should call runtime warmup hooks before marking pods ready \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Integration runbooks document expected warmup duration per artifact digest and context bucket \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Ignoring warmup in capacity math causes false headroom during traffic ramps \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\section{mode selection fleet}
\label{sec:ch30_mode_selection_fleet}

\textbf{Mode selection fleet} applies Chapter~12 execution modes at production scale: \textbf{eager} for debug and sampling hulls; \textbf{graph} for launch-bound decode cells; \textbf{megakernel} and Persistent Kernel paths for byte-bound fused cores---chosen per subgraph, not per monolithic binary \citep{nvidia2024cudagraphs,yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper}. Fleet configs bucket context lengths, batch widths, and backend SKUs consistently so capture templates and PK warmup tables match scheduler policies \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Eager mode.} Use for development, numerics bisect, and any branch outside graph legality---sampling, dynamic control flow, debugging hooks \citep{nvidia2024cudagraphs,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Eager baselines establish golden tokens for PK and graph paths \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Never ship eager per-op decode at scale when TaxBreak launch counts exceed fleet budget \citep{taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Graph mode.} CUDA Graph replay removes host launch storms when fusion depth is insufficient to reach MegaKernel but launches per token dominate \citep{memoryfloor2026,nvidia2024cudagraphs,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Bucket graphs by shape signatures aligned to framework max batch and context policies \citep{kwon2023vllm,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,dlbook}. Graph mode without byte wins indicates need for compiler fusion, not more buckets \citep{memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\textbf{MegaKernel and PK mode.} Compiler-owned decode cores collapse layers inside Persistent Kernel loops, targeting bytes per token and residual launches \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,dao2022flashattention,dlbook}. Combine with graph capture where legality allows---PK provides fusion; graphs stabilize host overhead \citep{yirage2026,nvidia2024cudagraphs,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Profile carrier lifetimes when bytes per token stall despite PK adoption \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,dlbook}.

\textbf{Fleet bucketing policy.} Document which mode applies to each subgraph in fleet config: prefill profile, decode profile, sampling hull, MoE communication \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dlbook}. Heterogeneous fleets maintain parallel artifact lines per hardware registry id from Chapter~29 \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024xdna,amd2024aie,semianalysis2024tma,dlbook}. Never reuse Hopper PK binaries on Blackwell without re-search \citep{yirage2026,chen2020compilerbasedhardw,nvidia2022hopper,semianalysis2024tma,patel2025llminference,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,amd2024aie,amd2024xdna,dao2023flashdecoding,dlbook}.

Mode selection meetings should attach counter worksheets: if launches per token move but bytes do not, stop adding graph buckets and fix fusion \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. If bytes move but queue wait dominates p99, return to framework scheduling \citep{patel2025llminference,kwon2023vllm,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Fleet-wide mode policy prevents per-team improvisation that breaks artifact compatibility \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}.

\paragraph{Multi-hardware delta.} Graph mode is CUDA-centric; XDNA fleets use static AIE binaries; CPU fleets use OpenMP-blocked eager or PK host loops \citep{amd2024xdna,amd2024aie,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,semianalysis2024tma,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. Same fleet config schema should encode backend-specific mode enums \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024aie,amd2024xdna,dlbook}.

\begin{example}
Team enables graph capture on entire decoder including sampling---illegal capture causes silent fallback to eager launches; mode map must exclude sampling hull \citep{nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,dlbook}.
\end{example}

Fleet configuration management should treat mode maps as first-class config objects with review gates \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dlbook}. Changing a subgraph from graph to PK mode without re-warming pools causes cold-start p99 spikes \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Autoscaling policies must account for warmup time when mode maps include capture-heavy paths \citep{memoryfloor2026,nvidia2024cudagraphs,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Document mode map version in artifact manifest so bisect correlates performance changes with policy edits \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Multi-region fleets replicate mode maps only after triplet passes in each region's SKU mix \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,semianalysis2024tma,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}.

\textbf{Debug versus production mode policy.} Development clusters may run eager decode for fast iteration \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Production clusters must enforce mode maps checked into config management \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,dlbook}. Accidental eager fallback in production reads as cost overrun and tail latency inflation \citep{taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,dlbook}. Alert when launch counts exceed eager thresholds on production endpoints \citep{taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Separate debug mode toggles from feature flags so marketing demos do not become silent production configs \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Hybrid mode maps are normal: prefill eager or wide-tile, decode PK plus graph, sampling eager hull, MoE communication custom \citep{nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,taxbreak2026,memoryfloor2026,dlbook}. Document each subgraph mode in fleet config with owner and counter expectation \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Changing one subgraph mode without re-running triplet on affected rows invites silent regressions \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Mode maps should be diff-reviewed like code \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,dlbook}.

SRE teams should store mode map version in incident tickets alongside framework and artifact versions \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Replaying incidents without mode context wastes hours tuning the wrong subgraph \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. Quarterly drills rotate mode map edits through staging triplet before any production diff \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook}. Fleet mode policy is co-design infrastructure as critical as paging block size \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,dlbook}.

\section{production regression}
\label{sec:ch30_production_regression}

\textbf{Production regression} gates fleet promotion on triplet counter suites: bytes per token, launches per token, and framework-level tokens per second---measured on GPU, CPU, and NPU backends where applicable \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,amd2024xdna,amd2024aie,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,nvidia2024cudagraphs,dlbook}. Version pins for \texttt{deps/YiRage} submodule, framework release, and artifact digest must move together through canary deploys \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Submodule bumps require re-run of search caches and PK warmup profiles before traffic shift \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.

\textbf{Triplet worksheet.} Row one: launches per token on warmed PK at BS${=}1$ for ctx 512, 2048, 8192 \citep{taxbreak2026,yirage2026,patel2025llminference,kwon2023vllm,dlbook}. Row two: bytes per token on same buckets \citep{memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,dlbook}. Row three: end-to-end tokens per second including queue wait at representative fleet load \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,dlbook}. Sign off only when each row improves or regressions carry profiler evidence and owner \citep{patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dlbook}.

\textbf{Regression triggers.} Framework paging ABI change; compiler manifest major bump; runtime descriptor version change; hardware registry id addition; CUDA driver or toolkit upgrade \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,nvidia2024cudagraphs,semianalysis2024tma,dlbook,taxbreak2026,memoryfloor2026,amd2024xdna,amd2024aie,liu2024deepseekv3,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,dlbook}. Each trigger reruns triplet on primary SKU plus one secondary backend in CI matrix \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024aie,amd2024xdna,dlbook,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,dao2023flashdecoding,nvidia2022hopper,dlbook}.

\textbf{Canary and rollback.} Shift artifact digest on fraction of replicas; compare triplet dashboards against control cohort \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Rollback redeploys prior artifact bundle---not framework scheduler alone \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. Store rollback pins beside fleet config for one-click revert \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\textbf{Bytes-focused regression.} Memory-Floor style suites catch HBM traffic regressions graph replay hides \citep{memoryfloor2026,nvidia2024cudagraphs,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,dlbook}. Pair byte counters with carrier lifetime profiles when fusion changes \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,dlbook}. Silent staging copies from paging integration show as byte regressions with unchanged launch counts \citep{memoryfloor2026,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,dlbook}.

CI pipelines pin submodule SHA, compile native extensions, run superoptimize smoke, PK decode ten tokens, and compare golden hash \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Extend smoke with vLLM integration template from Section~\ref{sec:ch30_vllm_yirage_stack} before production promotion \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. Record worksheet JSON in artifact store for audit \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}.

\begin{example}
Submodule bump passes import test but fails PK golden hash---regression gate blocks merge; fleet stays on prior digest until triplet reruns green \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,dlbook}.
\end{example}

Long-horizon \textbf{regression} debt accumulates when teams skip triplet on minor framework patches \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.
