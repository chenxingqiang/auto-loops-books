\chapter{AMD XDNA/AIE Dataflow Architecture}
\label{chap:ch05}
\typeout{START_CHAPTER "ch05" \theabspage}

GPU teams port a decoder kernel to Ryzen AI and expect ``slower CUDA.'' What they get is often a \emph{compile failure} or a silent fallback to DDR-bound scalar loops \citep{amd2024xdna,amd2024aie}. XDNA is not a SIMT machine with smaller shared memory---it is a spatial \emph{dataflow} array where buffer lifetimes, DMA chains, and tile assignments must be known before the first token \citep{patel2025llminference,chen2020compilerbasedhardw}. Chapter~3 listed NPU static legality in the constraint matrix; this chapter explains AMD's XDNA/AIE tiles, the static scheduling rules that decode must obey, and how those rules map to CUDA concepts (TMA, shared memory, softmax carriers) that GPU engineers already understand \citep{yirage2026,nvidia2022hopper}.

The decode metric lens does not change on NPU. TaxBreak still counts launches/token on GPUs; on XDNA the analogue is DMA transactions and DDR egress per token \citep{taxbreak2026,memoryfloor2026}. Batch-1 interactive decode still reads every weight and every KV vector each step---only the orchestration mechanism differs \citep{dao2022flashattention,dlbook}. YiRage's XDNA backend exists because hand-written ``one op at a time'' schedules violate the array's static contract and waste the watt advantage that makes edge NPUs attractive \citep{yirage2026,liu2024deepseekv3}.

Dense Llama-3.2-1B issues 847.5 GPU kernel launches per output token on H100 in TaxBreak decode sweeps; OLMoE-1B/7B issues 9,305 \citep{taxbreak2026}. XDNA cannot interpret those numbers as a launch budget---each logical ``launch'' must become a static DMA edge or be fused away entirely \citep{amd2024xdna,yirage2026}. Memory-Floor shows CUDA Graphs removing ${\approx}3.05$\,ms host overhead per step on Qwen-2.5-7B at BS${=}1$---on XDNA the comparable win is eliminating redundant DDR round trips, not replaying kernels \citep{memoryfloor2026,nvidia2024cudagraphs}.


\paragraph{Silicon context.} Ryzen AI combines Zen cores with XDNA NPU tiles aimed at laptop and edge inference \citep{amd2024xdna,amd2024aie}. Decode workloads stress DDR interfaces shared with CPU graphics and I/O---contention shows up as tail latency \citep{memoryfloor2026,patel2025llminference,dlbook}. Planning must include system-level bandwidth, not isolated NPU peaks \citep{yirage2026,taxbreak2026,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dao2022flashattention,nvidia2024cudagraphs,semianalysis2024tma,nvidia2022hopper,dao2023flashdecoding}.

\paragraph{Decode vs prefill on NPU.} Prefill parallelizes over prompt tokens; decode appends one token with full KV history \citep{dao2022flashattention,dao2023flashdecoding,patel2025llminference}. Static graphs compiled for long rectangular matmuls may waste tile arrays on thin decode matmuls \citep{amd2024aie,yirage2026,memoryfloor2026,taxbreak2026,amd2024xdna,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper}. Always qualify decode-shaped graphs separately \citep{yirage2026,amd2024xdna,memoryfloor2026,taxbreak2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\section{XDNA tiles}
\label{sec:ch05_xdna_tiles}

\paragraph{Ryzen AI deployment context.} XDNA appears in Ryzen AI NPUs paired with x86 cores---decode often splits pre/post on CPU and MegaKernel on AIE \citep{amd2024xdna,patel2025llminference,dlbook}. The tile grid is fixed per SKU; compiler passes must query exact SRAM sizes, not datasheets from a different stepping \citep{amd2024aie,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026}.

\paragraph{DDR vs tile bandwidth.} Decode is DDR-bandwidth bound when weights and KV exceed tile residency \citep{memoryfloor2026,dao2022flashattention}. Roofline on XDNA uses DDR bytes/token as the primary axis---identical to GPU decode thinking once HBM replaces DDR \citep{patel2025llminference,semianalysis2024tma,nvidia2022hopper,liu2024deepseekv3,kwon2023vllm,dao2023flashdecoding,yirage2026,amd2024aie,taxbreak2026,chen2020compilerbasedhardw}.

AMD \textbf{XDNA} groups AI Engine (\textbf{AIE}) tiles into a spatial array: each \textbf{tile} owns local SRAM, connects to neighbors through stream switches, and executes MAC-heavy datapaths under compiler-scheduled DMA \citep{amd2024xdna,amd2024aie}. Unlike GPU warps, there is no runtime scheduler to hide a missing buffer plan---if a decoder stage needs a temporary softmax statistic and no tile slot was reserved at compile time, the backend either rejects the graph or spills to DDR \citep{yirage2026,memoryfloor2026}.

Table~\ref{tab:xdna_tiles} is the mental model GPU engineers need before writing XDNA code: think ``DMA + tile SRAM budget,'' not ``launch another kernel'' \citep{semianalysis2024tma,nvidia2022hopper}. A Llama-class decoder layer that fits in Hopper shared memory with careful tiling may still \emph{illegalize} on XDNA if the same math requires three simultaneous tile-resident tensors---the array enforces a stricter simultaneous residency cap \citep{patel2025llminference,amd2024aie}.

\begin{example}
A single decode matmul tile on XDNA: weights stream from DDR into tile SRAM while activations arrive on a ping-pong DMA chain; if the compiler assigns overlapping lifetimes to two weight tiles, peak SRAM exceeds hardware limits and compile fails---there is no HBM spill fallback \citep{yirage2026,amd2024xdna}.
\end{example}

\paragraph{Worked contrast.} Hopper decode can recover from a suboptimal schedule with CUDA Graphs and extra launches \citep{nvidia2024cudagraphs,taxbreak2026}. XDNA decode rarely gets that slack: the graph is static or it does not ship \citep{amd2024aie,yirage2026}. Identical FLOPs, different bytes/token because DDR traffic dominates when tile residency fails \citep{memoryfloor2026,dao2023flashdecoding}.

\paragraph{Tile array topology.} Multiple AIE tiles connect in a grid; decode megakernels assign each pipeline stage to a tile column or row \citep{amd2024xdna,amd2024aie}. Compiler passes must respect inter-tile bandwidth limits---not every fusion that fits one tile fits the array \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference}.

\section{Static dataflow rules}
\label{sec:ch05_static_dataflow_rules}

\paragraph{Host-side responsibilities.} The host configures graph runners, submits execute calls, and updates KV metadata---but must not allocate or copy full tensors per token. KV growth appends through pre-planned DDR regions referenced by static descriptors. When KV grows beyond compiled max context, the product must recompile or reject---not fall back to unplanned buffers \citep{kwon2023vllm,amd2024xdna,yirage2026}.

\paragraph{Synchronization model.} GPUs use cudaDeviceSynchronize at unfortunate times; XDNA relies on DMA completion tokens wired in the dataflow graph. Inserting host waits between stages hides parallelism and adds jitter to p99 decode \citep{amd2024aie,patel2025llminference,memoryfloor2026}. Compiler backends should emit single submission boundaries per decoder step where possible \citep{yirage2026,taxbreak2026}.

\paragraph{Testing static graphs.} Validate graphs at min and max context buckets, not only nominal length. Off-by-one errors in carrier buffer sizes often appear only at max context \citep{dao2023flashdecoding,amd2024aie,memoryfloor2026}. Run byte counters on DDR ports during simulation before silicon \citep{amd2024xdna,chen2020compilerbasedhardw}.

\paragraph{Compiler IR implications.} MLIR and YiRage attach static lifetime intervals to memref views before lowering to AIE \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna}. Generic bufferization that inserts heap fallback breaks static rules immediately \citep{memoryfloor2026,kwon2023vllm,taxbreak2026,patel2025llminference,dao2022flashattention,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding}.

\paragraph{Recompile economics.} Static specialization implies compile minutes amortized over thousands of tokens \citep{yirage2026,patel2025llminference,amd2024aie}. Edge products with cold starts must budget compile latency in UX---unlike GPU JIT that hides in first batch \citep{taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,chen2020compilerbasedhardw,amd2024xdna,liu2024deepseekv3,kwon2023vllm,dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,nvidia2022hopper}.

\textbf{Static} XDNA schedules mean every tensor lifetime, DMA descriptor, and tile assignment is resolved at compile time \citep{amd2024xdna,amd2024aie}. \textbf{Dataflow} execution propagates tokens through spatial pipelines rather than a sequence of host-launched kernels \citep{chen2020compilerbasedhardw,yirage2026}. The rules decode compilers must internalize:

\begin{enumerate}
\item No hidden host allocations on the per-token path---KV and temporaries alias compile-time buffer slots \citep{kwon2023vllm,memoryfloor2026}.
\item No dynamic control flow inside the hot dataflow unless specialized into padded static variants \citep{patel2025llminference,liu2024deepseekv3}.
\item Producer--consumer edges must map to DMA events with known latency; ``sync later'' is not a plan \citep{amd2024aie,semianalysis2024tma}.
\item Fusion is all-or-nothing for many decode cells: partial fusion that exports intermediates to DDR often erases the NPU watt advantage \citep{yirage2026,taxbreak2026}.
\end{enumerate}

PyTorch-style eager op dispatch is the anti-pattern Chapter~1 diagnosed on GPUs---on XDNA it is often unsupported outright \citep{taxbreak2026,amd2024xdna}. The compiler must emit one static graph per shape bucket (batch, context class) and recompile when buckets change, amortizing compile cost across many tokens \citep{yirage2026,patel2025llminference}.

\paragraph{Deployment pitfall.} Teams benchmark prefill-length graphs on XDNA, then deploy batch-1 decode with growing KV---dynamic sequence growth violates static buffer tables unless the compiler padded or bucketed context \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026}. Treat context buckets as first-class compile parameters, not runtime surprises \citep{kwon2023vllm,yirage2026}.

\paragraph{Static legality review.} Every merge should attach a buffer lifetime diagram: which tensors live in tile SRAM simultaneously at each decode step \citep{amd2024aie,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026}.

\section{CUDA XDNA mapping}
\label{sec:ch05_cuda_xdna_mapping}

\paragraph{Shared memory illusion.} GPU shared memory is explicitly sized and banked; tile SRAM is smaller but deterministic. Engineers who oversize tiles ``because HBM is huge'' fail XDNA legality immediately \citep{nvidia2022hopper,amd2024aie,semianalysis2024tma}. Start from tile SRAM datasheet bytes, not GPU comfort zones \citep{yirage2026,memoryfloor2026}.

\paragraph{TMA lesson transfer.} TMA teaches asynchronous movement while compute runs---XDMA chains must encode the same overlap explicitly. Without overlap, MAC utilization collapses and watt/token rises even if correctness holds \citep{semianalysis2024tma,amd2024xdna,patel2025llminference}. Profile MAC active cycles, not just end-to-end latency \citep{amd2024aie,yirage2026}.

\paragraph{Graph capture analogy.} CUDA Graphs freeze launch order; XDNA freezes DMA order. Mutating either without re-record/re-compile breaks decode silently \citep{nvidia2024cudagraphs,amd2024xdna,taxbreak2026,memoryfloor2026}. Treat shape buckets like graph capture templates \citep{yirage2026,kwon2023vllm}.

\paragraph{Double buffering depth.} Hopper TMA double buffering hides HBM latency; XDNA ping-pong DMA chains require explicit buffer descriptor pairs in the static graph \citep{semianalysis2024tma,nvidia2022hopper,amd2024aie,yirage2026}. Shallow pipelines under-utilize MAC arrays; deep pipelines overflow tile SRAM \citep{memoryfloor2026,patel2025llminference,dao2022flashattention,taxbreak2026,amd2024xdna,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dlbook,dao2023flashdecoding,nvidia2024cudagraphs}.

\paragraph{Host orchestration.} CUDA uses streams; XDNA uses graph runners that submit entire dataflow frames \citep{nvidia2024cudagraphs,amd2024xdna,yirage2026}. Per-token host work must be O(1) and allocation-free---TaxBreak's launch tax has a direct analogue in graph submission overhead \citep{taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,amd2024aie,semianalysis2024tma,nvidia2022hopper,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dao2023flashdecoding,dlbook}.

GPU engineers think in TMA descriptors and shared-memory tiles; XDNA engineers think in \textbf{buffer descriptor}s and tile SRAM assignments \citep{nvidia2022hopper,semianalysis2024tma,amd2024xdna}. The mapping is conceptual, not one-to-one:

\begin{tabular}{ll}

\midrule

Hopper \textbf{TMA} hides DDR latency by double-buffering into shared memory while MMA units compute \citep{semianalysis2024tma,nvidia2022hopper}. XDNA achieves a similar \emph{software pipeline} with ping-pong \textbf{buffer descriptor}s into tile SRAM---but the compiler must assign both buffers and DMA edges statically \citep{amd2024aie,yirage2026}. A decode attention stage that uses TMA-backed softmax carriers on GPU must become a fixed carrier pipeline on AIE; you cannot insert a host-side \texttt{malloc} between carriers \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026}.

\begin{example}
GPU decode step: TMA loads Q tile while previous softmax partials remain in shared memory. XDNA equivalent: DMA chain loads Q into tile $A$ while tile $B$ holds running max/sum carriers---both buffers named in the static graph; swapping roles is compile-time ping-pong, not runtime alloc \citep{yirage2026,amd2024xdna,semianalysis2024tma}.
\end{example}

\paragraph{Review gate.} Before merging XDNA backend changes, diff static buffer tables against tile SRAM caps and list every DDR transaction per decode step \citep{amd2024aie,memoryfloor2026,yirage2026}. If DDR bytes/token exceed GPU baselines, residency planning failed---not ``the NPU is slow'' \citep{patel2025llminference,taxbreak2026}.

\section{Online softmax HW}
\label{sec:ch05_online_softmax_hw}

\paragraph{Numerical stability.} Online softmax relies on stable accumulation order; spatial reductions must preserve mathematical equivalence to reference implementations \citep{dao2022flashattention,dao2023flashdecoding}. Validation should compare logits against GPU reference on short contexts before trusting long-context deployments \citep{memoryfloor2026,amd2024aie,yirage2026}.

\paragraph{Carrier placement.} Assign carrier buffers adjacent to attention tiles in the dataflow to minimize stream latency. Poor placement increases stage cycles without changing FLOPs---visible only in cycle-accurate simulation \citep{amd2024xdna,patel2025llminference,chen2020compilerbasedhardw}. Compiler search should include placement, not only tile sizes \citep{yirage2026,semianalysis2024tma}.

\paragraph{Carrier precision.} Online softmax carriers need width headroom for long contexts \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026}. Fixed-point carriers save SRAM but risk saturation; compiler must pick formats per tile budget \citep{amd2024aie,yirage2026,semianalysis2024tma,patel2025llminference,taxbreak2026,amd2024xdna,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,nvidia2022hopper,nvidia2024cudagraphs,dlbook,dao2022flashattention}.

\paragraph{Attention tile fusion.} Fusing QK$^\top$, softmax carrier update, and AV into one spatial pipeline is the XDNA analogue of FlashAttention fusion \citep{dao2022flashattention,yirage2026,amd2024aie,memoryfloor2026,taxbreak2026,patel2025llminference,semianalysis2024tma,amd2024xdna,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dao2023flashdecoding,nvidia2024cudagraphs,dlbook,nvidia2022hopper}.

Attention decode needs numerically stable \textbf{softmax} over growing KV context \citep{dao2022flashattention,dao2023flashdecoding}. FlashAttention-style algorithms keep running max and sum statistics in on-chip carriers rather than materializing full softmax matrices to DDR \citep{memoryfloor2026}. On GPU, those carriers live in shared memory with warp-synchronous updates; on XDNA they must be assigned to specific \textbf{carrier} buffer slots in the static dataflow \citep{amd2024aie,yirage2026}.

The hardware failure mode is familiar: a ``simple'' softmax op lowered independently forces a full attention map to DDR, doubling bytes/token \citep{taxbreak2026,memoryfloor2026}. Online softmax on AIE requires fusing the carrier update with the QK$^\top$ pipeline so partial statistics never leave tile SRAM \citep{dao2022flashattention,semianalysis2024tma}. GPU-first teams often import unfused softmax kernels and wonder why XDNA compile rejects the graph---the rejection is correct \citep{amd2024xdna,yirage2026,patel2025llminference}.

\begin{example}
Decode softmax carrier width: at context 2048, carrier buffers need slots sized for one KV tile; if the compiler assumes prefill-length vectors, tile SRAM overflows and the backend splits the graph into DDR-bound pieces \citep{dao2023flashdecoding,memoryfloor2026,amd2024aie}.
\end{example}

\paragraph{Worked contrast.} Hopper can recover with Flash-Decoding parallelism along sequence \citep{dao2023flashdecoding}. XDNA lacks an equivalent ``launch more blocks'' escape---sequence parallelism must be encoded as spatial lanes or rejected at compile time \citep{amd2024xdna,yirage2026,liu2024deepseekv3}.

\section{Architecture tradeoffs}
\label{sec:ch05_architecture_tradeoffs}

\paragraph{Latency vs watt.} Edge products advertise watt/token; cloud products advertise ms/token under load \citep{amd2024xdna,patel2025llminference,taxbreak2026}. XDNA wins watt; CUDA wins elastic throughput \citep{liu2024deepseekv3,kwon2023vllm,nvidia2024cudagraphs}. Architecture reviews should pick which metric is primary before choosing silicon \citep{yirage2026,memoryfloor2026,dlbook}.

\paragraph{Software stack depth.} Ryzen AI ships runtime, compiler, and drivers---teams must coordinate versions like CUDA toolkit pins \citep{amd2024aie,amd2024xdna,chen2020compilerbasedhardw}. Upgrading one layer without requalifying decode graphs repeats MLIR upgrade horror stories \citep{yirage2026,taxbreak2026,memoryfloor2026}.

\paragraph{Hybrid fleets.} Production may run GPU decode pools plus XDNA edge nodes---schedules must diverge by design, not by accident \citep{patel2025llminference,liu2024deepseekv3,amd2024xdna,yirage2026,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,amd2024aie,dao2022flashattention,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.

\paragraph{Power envelopes.} Watt/token often drives XDNA product wins even when raw ms/token loses to H100 \citep{amd2024xdna,amd2024aie,memoryfloor2026,patel2025llminference,yirage2026,taxbreak2026,chen2020compilerbasedhardw,liu2024deepseekv3,dao2022flashattention,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.

\textbf{CUDA} decode optimization trades launches against bytes: CUDA Graphs and fusion reduce orchestration; TMA and shared memory reduce HBM traffic \citep{nvidia2024cudagraphs,taxbreak2026,semianalysis2024tma,nvidia2022hopper}. \textbf{XDNA} decode optimization trades compile-time staticism against DDR egress: the array wins on watt/token when the entire decoder step is one spatial pipeline; it loses when dynamicism forces recompile or scalar fallbacks \citep{amd2024xdna,amd2024aie,memoryfloor2026}.

MoE models expose the starkest CUDA vs XDNA split: TaxBreak reports thousands of kernel launches per token on GPU MoE decode \citep{taxbreak2026,liu2024deepseekv3}. XDNA cannot absorb that launch count---experts must be fused, batched, or compiled as separate static graphs with explicit routing DMA \citep{yirage2026,amd2024aie}. Choosing XDNA for MoE without a static routing plan repeats GPU orchestration failure \citep{patel2025llminference,memoryfloor2026}.

\paragraph{When XDNA wins.} Fixed-model edge assistants with bounded context, watt caps, and fully static megakernels \citep{amd2024xdna,yirage2026,dao2022flashattention}. \paragraph{When CUDA wins.} Dynamic batching, speculative decoding, long-context cloud serving, and MoE with frequent graph changes \citep{kwon2023vllm,nvidia2024cudagraphs,taxbreak2026,liu2024deepseekv3}.

\section{Dataflow conclusion for inference}
\label{sec:ch05_dataflow_conclusion}

\paragraph{Documentation for operators.} Runbooks should list compiled shape buckets, max context, recompile triggers, and expected DDR bytes/token \citep{patel2025llminference,amd2024xdna,yirage2026}. Operators otherwise misdiagnose KV growth as model quality issues \citep{kwon2023vllm,memoryfloor2026,taxbreak2026}.

\paragraph{Failure modes summary.} Compile rejection, DDR bandwidth saturation, carrier overflow, host submission jitter, and stale graphs after model updates \citep{amd2024aie,amd2024xdna,yirage2026,dao2022flashattention,dao2023flashdecoding,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper}. Each maps to a measurable counter \citep{yirage2026,amd2024xdna,memoryfloor2026,taxbreak2026}.

\paragraph{Speculative decoding caveat.} Speculative paths introduce dynamic branches hostile to static XDNA graphs \citep{patel2025llminference,liu2024deepseekv3,amd2024xdna,yirage2026,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,amd2024aie,dao2022flashattention,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}. Either compile separate graphs per speculation policy or keep speculation on GPU \citep{yirage2026,patel2025llminference,amd2024xdna,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,amd2024aie,liu2024deepseekv3,dao2022flashattention,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.

\paragraph{Continuous batching.} vLLM-style batching on GPU does not port literally---XDNA batching requires padded static batch classes \citep{kwon2023vllm,amd2024xdna,yirage2026,patel2025llminference,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,amd2024aie,liu2024deepseekv3,dao2022flashattention,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.

\textbf{Dataflow} is the operational model for NPU \textbf{inference} when decode must meet watt and latency envelopes \citep{amd2024xdna,chen2020compilerbasedhardw,yirage2026}. Production conclusions:

\begin{enumerate}
\item Treat decode as a static MegaKernel per shape bucket; recompile when buckets change \citep{yirage2026,patel2025llminference,amd2024aie}.
\item Measure DDR bytes/token and DMA count per token---not FLOPs \citep{memoryfloor2026,taxbreak2026}.
\item Reject graphs that materialize KV or softmax intermediates to DDR \citep{dao2022flashattention,dao2023flashdecoding,kwon2023vllm}.
\item Pair GPU and XDNA baselines from the same high-level IR \citep{chen2020compilerbasedhardw,yirage2026,dlbook}.
\end{enumerate}

Cross-hardware inference fleets run GPU pools for elastic decode and NPU nodes for edge---the dataflow lesson is that identical weights do not imply identical schedules \citep{liu2024deepseekv3,patel2025llminference,amd2024xdna}. Compiler stacks must fork early; ``port the CUDA kernel'' without static replanning fails \citep{yirage2026,taxbreak2026}.

\section{YiRage XDNA backend}
\label{sec:ch05_yirage_xdna_backend}

\paragraph{Multi-backend CI.} YiRage CI should compile the same decoder export for CUDA and XDNA on every merge, publishing legality diffs \citep{yirage2026,chen2020compilerbasedhardw,amd2024aie,amd2024xdna,taxbreak2026,memoryfloor2026,patel2025llminference,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\paragraph{Developer ergonomics.} When XDNA compile fails, error messages should cite buffer table rows and tile IDs---not opaque backend codes \citep{yirage2026,amd2024aie,patel2025llminference,memoryfloor2026,amd2024xdna,chen2020compilerbasedhardw,taxbreak2026,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Good diagnostics shorten the static scheduling learning curve \citep{yirage2026,amd2024xdna}.

\paragraph{Debug workflow.} On XDNA compile failure, dump static buffer table and DMA edge list before reading assembly \citep{yirage2026,amd2024aie,amd2024xdna,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,patel2025llminference,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.

\paragraph{Regression tests.} CI should compile canonical decoder layers for each supported Ryzen AI SKU and fail on new allocs in hot path \citep{yirage2026,amd2024xdna,amd2024aie,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,patel2025llminference,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.

\textbf{YiRage} targets \textbf{XDNA} through the same residency vocabulary as its CUDA backend: buffer lifetimes, fusion boundaries, and decode metrics \citep{yirage2026,amd2024xdna,amd2024aie}. The XDNA backend rejects dynamic allocations on the hot path, maps fused decoder cells to static AIE graphs, and runs legality checks from Chapter~3 before codegen \citep{chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026}.

Practical integration: attach tile SRAM and DMA metadata during MLIR import; compile separate graphs per shape bucket; fuse online softmax with attention; triplet-test GPU vs XDNA on identical IR \citep{yirage2026,dao2022flashattention,dao2023flashdecoding,taxbreak2026,kwon2023vllm,amd2024aie,chen2020compilerbasedhardw}.

\begin{example}
YiRage merge gate: Llama-3.2-1B decoder layer at BS${=}1$, ctx${=}2048$ must compile as one static XDNA graph or document failing ops; report GPU launches/token for the same IR export \citep{taxbreak2026,yirage2026,memoryfloor2026,amd2024xdna}.
\end{example}

\paragraph{Operational checklist.} Verify static buffer table fits tile SRAM; count DDR transactions per token; run decode p99; log claims to \texttt{verified\_facts.jsonl} \citep{memoryfloor2026,kwon2023vllm,dao2022flashattention,yirage2026,patel2025llminference,taxbreak2026}.



\paragraph{Engineering narrative.} Treat XDNA as a compiler target with hard legality, not as a slow GPU. Your schedule is a DMA graph; your on-chip memory is tile SRAM; your failure telemetry is DDR bytes per token. Every section below connects those primitives to decode pain already measured on GPUs in Chapter~1---launch storms become DMA chains, shared-memory spills become DDR egress, and framework fragmentation becomes compile-time rejection \citep{taxbreak2026,memoryfloor2026,amd2024xdna}.

\paragraph{Who should read this chapter.} GPU kernel engineers porting to Ryzen AI, compiler authors lowering MLIR to AIE, and serving teams splitting cloud GPU pools from edge NPU nodes. If you only deploy CUDA, read the CUDA--XDNA mapping and tradeoff sections---they explain why your GPU recipe will not copy verbatim \citep{yirage2026,patel2025llminference,nvidia2022hopper}.

\paragraph{Tile SRAM accounting walkthrough.} List every tensor live during one decode step: Q slice, K/V cache lines, softmax carriers, MLP intermediates, rotary embeddings. Sum bytes per tile. If the sum exceeds SRAM, the compiler must pipeline in time (sequential stages) or space (more tiles), not spill to DDR silently \citep{amd2024aie,memoryfloor2026,dao2022flashattention}. GPU engineers perform a similar exercise with shared-memory budgets; the difference is that XDNA seldom forgives overflow \citep{amd2024xdna,yirage2026}.

\paragraph{Static dataflow worked example.} Imagine a two-stage decode cell: attention then MLP. Stage one DMA-loads K/V while MAC arrays update softmax carriers entirely in tile SRAM. Stage two consumes attention output directly from stream registers without DDR round trip. If MLP weights are DMA-loaded mid-stage without double buffering, MAC units stall---visible as higher ms/token even though FLOPs counters look healthy \citep{semianalysis2024tma,patel2025llminference,amd2024aie}.

\paragraph{Buffer descriptors in practice.} A buffer descriptor names source address, destination tile buffer, length, and stride---analogous to TMA copy descriptors on Hopper. Ping-pong uses two descriptors alternating roles each token step. Decode graphs must freeze descriptor contents per shape bucket; changing context length without recompile implies descriptors were padded for max context upfront \citep{nvidia2022hopper,amd2024xdna,yirage2026,kwon2023vllm}.

\paragraph{Online softmax on hardware.} Carriers track running maximum and accumulated exponential sum per row tile. Each new KV tile updates carriers without writing attention probabilities to DDR. Breaking fusion at softmax exports an $L\times L$ map---fatal for long context on bandwidth-limited DDR \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026}. GPU warp-level reductions become spatial reduction trees on AIE---different mechanism, same invariant: carriers stay on-chip \citep{amd2024aie,yirage2026}.

\paragraph{CUDA vs XDNA product framing.} Cloud assistants with dynamic batching and MoE remain CUDA-first \citep{taxbreak2026,liu2024deepseekv3,kwon2023vllm}. Fixed-model copilots on laptops with watt caps are XDNA candidates when compile time is amortized \citep{amd2024xdna,patel2025llminference}. Hybrid designs route elastic traffic to GPU and pinned edge traffic to NPU---two schedules, one product \citep{yirage2026,chen2020compilerbasedhardw,dlbook}.

\paragraph{Inference conclusion expanded.} Dataflow inference means the runtime executes a compiled graph with predictable DMA timing---no Python loop launching ops \citep{amd2024xdna,amd2024aie}. Latency variance comes from DDR contention and host submission, not from kernel selection \citep{memoryfloor2026,taxbreak2026}. Qualify XDNA builds with p99 decode traces, not prefill throughput alone \citep{patel2025llminference,dao2023flashdecoding}.

\paragraph{YiRage backend depth.} YiRage shares IR between CUDA and XDNA backends: the same fused decoder cell exported from PyTorch passes target-specific legality checks \citep{yirage2026,chen2020compilerbasedhardw}. Failures surface as compile errors with buffer tables attached---preferable to silent DDR fallback \citep{amd2024aie,memoryfloor2026}. Merge gates require triplet metrics: GPU launches/token, XDNA DMA count, bytes/token both sides \citep{taxbreak2026,patel2025llminference,amd2024xdna}.

\paragraph{Common mistakes checklist.} Importing unfused PyTorch ops one by one. Assuming dynamic batch/seq like CUDA. Omitting softmax carrier fusion. Skipping compile rejections because GPU numbers look fine. Measuring prefill only. Each mistake reproduces Chapter~1 pathologies on different silicon \citep{taxbreak2026,dao2022flashattention,yirage2026,amd2024xdna,kwon2023vllm,memoryfloor2026,liu2024deepseekv3,nvidia2024cudagraphs,dlbook,semianalysis2024tma,nvidia2022hopper,dao2023flashdecoding,amd2024aie,chen2020compilerbasedhardw,patel2025llminference}.

\paragraph{Forward pointer.} Chapter~6 formalizes residency planning across GPU, CPU, and NPU; Chapter~14 covers MLIR lowering that feeds YiRage backends \citep{chen2020compilerbasedhardw,yirage2026}. The XDNA-specific lesson here is static legality: encode it early or pay DDR costs every token \citep{amd2024xdna,amd2024aie,memoryfloor2026,taxbreak2026,patel2025llminference,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,chen2020compilerbasedhardw}.

\paragraph{Metric closure.} End every XDNA experiment reporting DDR bytes/token, DMA transactions per token, compile legality pass rate, and p99 decode at batch one \citep{memoryfloor2026,taxbreak2026,yirage2026,patel2025llminference,amd2024xdna,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Without those counters, debates devolve into FLOPs and die area---useless for decode \citep{taxbreak2026,memoryfloor2026,amd2024xdna,yirage2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Measure before merging; merge before celebrating \citep{yirage2026,amd2024xdna,taxbreak2026,memoryfloor2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\section{XDNA decode review gates}
\label{sec:ch05_review_gates}

\begin{enumerate}
\item \textbf{Tile SRAM budget:} no simultaneous buffers exceed per-tile caps \citep{amd2024aie,amd2024xdna,yirage2026}.
\item \textbf{Static graph:} no dynamic dims on hot path without bucket recompile \citep{patel2025llminference,kwon2023vllm,chen2020compilerbasedhardw}.
\item \textbf{DDR ledger:} bytes/token vs GPU baseline on same model \citep{memoryfloor2026,taxbreak2026,dao2022flashattention}.
\item \textbf{Softmax carriers:} fused online; no full-map materialization \citep{dao2023flashdecoding,semianalysis2024tma,amd2024aie}.
\item \textbf{Triplet IR:} same export tested on CUDA and XDNA backends \citep{yirage2026,liu2024deepseekv3,dlbook,nvidia2024cudagraphs,nvidia2022hopper}.
\end{enumerate}

\begin{example}
Review gate failure: new MLP fusion adds third tile-resident buffer---compile passes on GPU shared mem, fails XDNA legality; gate blocks merge until tile assignment replanned \citep{yirage2026,amd2024xdna,amd2024aie,memoryfloor2026,taxbreak2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.
\end{example}

These gates mirror Chapter~3 benchmark method: measure first, optimize second \citep{memoryfloor2026,taxbreak2026,yirage2026,patel2025llminference,amd2024xdna,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}. Skipping them ships DDR-bound decode that wastes Ryzen AI silicon \citep{amd2024xdna,yirage2026,memoryfloor2026,taxbreak2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding,nvidia2022hopper}.


\paragraph{Deep dive: tile grid scheduling.} Compiler schedulers assign operations to AIE tiles in a grid. Neighboring tiles exchange data through stream switches with finite bandwidth. A decode MegaKernel must map attention heads or hidden dimensions to tiles without oversubscribing switches \citep{amd2024xdna,amd2024aie,yirage2026}. Oversubscription does not always fail compile---it stalls runtime and inflates p99 \citep{patel2025llminference,memoryfloor2026,taxbreak2026}. Simulation before tape-out catches switch hotspots \citep{chen2020compilerbasedhardw,amd2024aie}.

\paragraph{Deep dive: weight streaming.} Weights are read every decode token. On GPU, weight bandwidth competes with KV in HBM; on XDNA, weights DMA from DDR into tile SRAM each step unless partially resident \citep{memoryfloor2026,dao2022flashattention,amd2024xdna}. Static plans may pin hot slices in tile SRAM across tokens while streaming cold slices---analogous to cache blocking on CPU \citep{dlbook,yirage2026,kwon2023vllm}. Plans that reload full layers each token saturate DDR \citep{patel2025llminference,taxbreak2026,amd2024aie}.

\paragraph{Deep dive: KV cache on DDR.} KV typically lives in DDR on edge NPUs; attention reads entire history each token \citep{kwon2023vllm,dao2023flashdecoding,amd2024xdna}. Paged KV layouts still require static page tables in many XDNA stacks \citep{yirage2026,memoryfloor2026,chen2020compilerbasedhardw}. Fragmentation hurts DDR efficiency---qualify with bytes/token, not parameter count \citep{taxbreak2026,patel2025llminference,amd2024aie,liu2024deepseekv3,dao2022flashattention,nvidia2024cudagraphs,semianalysis2024tma,nvidia2022hopper,dlbook,amd2024xdna,chen2020compilerbasedhardw,yirage2026}.

\paragraph{Deep dive: rotary and norms.} Layer norm and rotary embedding are easy to dismiss as tiny ops; unfused they become separate DMA rounds \citep{taxbreak2026,amd2024aie,yirage2026}. Fuse into attention or MLP stages where carriers already resident \citep{dao2022flashattention,patel2025llminference,memoryfloor2026,amd2024xdna,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,taxbreak2026,yirage2026,amd2024aie}.

\paragraph{Deep dive: quantization.} INT8 and INT4 weights reduce DDR pressure---critical on XDNA \citep{amd2024xdna,memoryfloor2026,patel2025llminference,yirage2026}. Quantization scales must be embedded in static graphs; runtime scale lookup per op reintroduces dynamicism \citep{amd2024aie,chen2020compilerbasedhardw,taxbreak2026,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,yirage2026,amd2024xdna,memoryfloor2026}.

\paragraph{Deep dive: multi-layer decode step.} A full decoder step chains multiple layer graphs or one fused multi-layer graph \citep{yirage2026,amd2024aie,amd2024xdna}. Chaining increases host submissions; fusing increases SRAM pressure \citep{patel2025llminference,memoryfloor2026,taxbreak2026,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,yirage2026,amd2024aie,amd2024xdna}. Autotune Pareto fronts across those axes \citep{yirage2026,memoryfloor2026,taxbreak2026}.

\paragraph{Deep dive: observability.} Instrument DMA bytes, MAC active ratio, and graph submission time separately \citep{amd2024xdna,patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Without decomposition, teams misallocate effort to weight quantization when scheduling is the bottleneck \citep{yirage2026,amd2024xdna,taxbreak2026,memoryfloor2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\paragraph{Epilogue for practitioners.} XDNA rewards teams that think like compiler engineers first and kernel micro-optimizers second \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Static dataflow is the price of watt-efficient decode; pay it deliberately \citep{yirage2026,amd2024xdna,memoryfloor2026,taxbreak2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. When graphs fail, celebrate early rejection---it is cheaper than DDR-bound production \citep{yirage2026,amd2024aie,amd2024xdna,memoryfloor2026,taxbreak2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\section{Chapter Summary}
\label{sec:ch05_summary}

XDNA/AIE decode rewards static spatial pipelines and punishes GPU-style dynamicism \citep{amd2024xdna,amd2024aie,yirage2026}. Map CUDA TMA/shared-memory intuition to DMA/tile-buffer planning; enforce online softmax carriers on-chip; measure DDR bytes/token instead of peak FLOPs \citep{semianalysis2024tma,dao2022flashattention,memoryfloor2026,taxbreak2026}. YiRage encodes these rules as legality gates \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference}. Part~IV megakernels apply only after the dataflow graph is fully static \citep{liu2024deepseekv3,nvidia2024cudagraphs,dlbook,kwon2023vllm,dao2023flashdecoding,amd2024xdna,nvidia2022hopper}.

When static planning fails, trust compile rejections and bytes/token telemetry over microbench FLOPs \citep{memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,amd2024aie,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,dao2023flashdecoding}.


\paragraph{Worked scheduling exercise.} Sketch one decode token for a single transformer layer. Count DDR reads for weights, KV, and any spilled activations. Count tile SRAM peaks with all live buffers. If peaks exceed hardware, split the layer into two dataflow subgraphs with an intermediate DDR tile---then measure whether bytes/token still beats GPU baseline for your product metric \citep{memoryfloor2026,taxbreak2026,yirage2026,amd2024xdna,amd2024aie,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\paragraph{Integration with CPU host.} Zen cores run tokenization, sampling, and small control flow while XDNA runs the MegaKernel \citep{amd2024xdna,dlbook,patel2025llminference,yirage2026}. PCIe and memory fabric contention between CPU and NPU shows up as decode jitter \citep{memoryfloor2026,taxbreak2026,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,amd2024xdna,yirage2026}. Co-locate KV buffers and pin pages when runtimes allow \citep{kwon2023vllm,amd2024xdna,memoryfloor2026,taxbreak2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,yirage2026}.

\paragraph{Lesson from GPU decode.} Chapter~1 showed launches dominating MoE decode on H100 \citep{taxbreak2026,liu2024deepseekv3}. XDNA removes launches but retains bytes/token physics---weights and KV still move \citep{memoryfloor2026,dao2022flashattention,amd2024xdna,amd2024aie,yirage2026,patel2025llminference,chen2020compilerbasedhardw,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding,taxbreak2026,yirage2026,amd2024aie,amd2024xdna,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Success requires fusion and residency; infrastructure alone is insufficient on any hardware \citep{yirage2026,amd2024xdna,taxbreak2026,memoryfloor2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.

\paragraph{Closing checklist.} Before declaring XDNA decode ready: static graph compiles for all shape buckets; softmax carriers fused; DDR bytes/token measured; p99 stable over long sessions; GPU triplet comparison documented; compile failures published as negative results \citep{yirage2026,amd2024xdna,amd2024aie,memoryfloor2026,taxbreak2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. This checklist is the minimum bar Part~III expects before MegaKernel chapters \citep{yirage2026,amd2024xdna,memoryfloor2026,taxbreak2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.


\paragraph{Plain-language recap.} XDNA is a spatial accelerator: data moves on scheduled paths through tile memories while arithmetic happens locally. GPU programming hides much of that schedule behind launches and caches. XDNA exposes the schedule to the compiler. Decode hurts on both platforms because each new token re-reads weights and cache history. GPUs hide orchestration cost until launches explode. XDNA rejects bad schedules before run. Your job is to make the static graph tight enough that DDR bits per token compete with fused GPU baselines.

\paragraph{Plain-language recap II.} Start from metrics, not dialects. Measure bytes per token and tail latency at batch one. Build buffer tables that fit tile SRAM. Fuse softmax carriers with attention. Map CUDA concepts deliberately: TMA to DMA, shared memory to tile SRAM, graphs to static dataflow. Use YiRage or an equivalent stack to enforce legality. Document compile failures. Compare against TaxBreak launch counts on GPU for the same model export.

\paragraph{Plain-language recap III.} Edge inference products fail when teams treat NPU bring-up as a weekend port. Plan shape buckets, recompile policy, and DDR bandwidth upfront. Qualify decode separately from prefill. Publish negative results when graphs illegalize. The hardware is unforgiving by design---static dataflow is how the watt budget is won.


\paragraph{Final synthesis.} AMD XDNA and AIE tiles implement a static dataflow machine for inference: compiler-planned DMA into tile SRAM, spatial MAC arrays, and strict buffer lifetimes. GPU decode optimization fights launch count and HBM bytes; XDNA decode optimization fights DDR bytes and compile legality. Map Hopper TMA and shared memory mental models to buffer descriptors and tile buffers. Keep online softmax carriers on-chip. Run YiRage-style triplet tests from one IR export. Measure bytes per token, DMA activity, and p99 batch-one latency---not peak FLOPs. When graphs fail static checks, fix scheduling before micro-tuning arithmetic. That discipline carries forward through residency design, megakernels, and MLIR lowering in later chapters.


\paragraph{Reader takeaway.} If you remember nothing else: XDNA decode is a static DMA and tile-SRAM scheduling problem \citep{amd2024xdna,amd2024aie,yirage2026}. Dynamic GPU habits---per-op alloc, runtime shape changes, unfused softmax---fail fast here \citep{taxbreak2026,memoryfloor2026,dao2022flashattention,patel2025llminference,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}. Build buffer tables, fuse carriers, measure DDR bytes per token, and treat compile rejections as success \citep{yirage2026,amd2024xdna,memoryfloor2026,taxbreak2026,patel2025llminference,amd2024aie,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper,dao2023flashdecoding}.
