Hardware truth: GPUs bound lifetime with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on lifetime/residen legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

The production surprise at \textbf{GPU residency} is not peak FLOPs; it is illegal tier placement: shared spilling to HBM/DDR while register still pays launch or DMA tax every token \citep{nvidia2022hopper,semianalysis2024tma}.

Trace residency: shared producers must feed register consumers without a DDR round trip \citep{dao2022flashattention,yirage2026}. Silent spill reads as ``mysterious'' p99 regression \citep{patel2025llminference,semianalysis2024tma}.

Operational fix: diff IR or CUDA for new allocs; count launches/token (GPU) or DMA edges (NPU); require graph capture legality on GPU and static legality on XDNA \citep{nvidia2024cudagraphs,amd2024aie}.

\paragraph{Review gate.} Attach roofline or buffer accounting for shared on target SKU before merge \citep{memoryfloor2026,yirage2026}.

\section{CPU residency}
\label{sec:ch06_cpu_residency}

\textbf{CPU residency} is where Data Residency and On-Chip Memory Design meets Chapter~1 counters: fix cache and NUMA together or bytes/token flatlines while kernel microbenchs celebrate \citep{nvidia2022hopper,amd2024aie}.

Hardware truth: GPUs bound cache with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on cache/NUMA legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\section{Chapter Summary}
\label{sec:ch06_summary}

Data Residency and On-Chip Memory Design succeeds when hardware constraints drive schedules and decode metrics prove relief \citep{yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw}. \textbf{lifetime planning} exposes a cross-hardware fracture: the same fused subgraph legal on Hopper can be rejected outright on XDNA or CPU cache budgets.

Binding software to silicon: compiler passes must encode lifetime/residen legality before codegen, not as comments in hand-written kernels \citep{nvidia2022hopper,amd2024aie}. YiRage runs target-aware checks against chip models from Chapter~3; manual stacks should treat failed legality as a release blocker, not a backend TODO.

When lifetime planning disagrees with microbenchmarks, trust fleet telemetry: fix orchestration and tier placement first, then revisit MMA tile shapes \citep{nvidia2022hopper,amd2024aie}.

\paragraph{Review gate.} Before merging compiler changes affecting lifetime planning, attach roofline or NPU buffer accounting showing lifetime residency fits on-chip for the target SKU \citep{nvidia2022hopper,amd2024aie}.

\paragraph{Worked contrast.} A Hopper decode cell may legalize lifetime with TMA-backed double buffering while an XDNA cell requires the same math expressed as static tile assignments---identical FLOPs, different bytes/token because DDR traffic differs by an order of magnitude \citep{nvidia2022hopper,amd2024aie}.

The usual Data Residency and On-Chip Memory Design failure at \textbf{GPU residency} is importing a CUDA mental model and expecting runtime scheduling to hide illegal buffer lifetimes.

Hardware constraint: shared and register must be co-designed.
