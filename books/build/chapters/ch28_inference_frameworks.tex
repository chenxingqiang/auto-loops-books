Who owns decode goodput---the inference \emph{framework} that batches requests and manages KV, or the \emph{compiler/runtime} that fuses kernels and cuts bytes/token? The industry default splits responsibility badly: frameworks optimize scheduling and capacity; compilers optimize isolated GEMMs; neither measures the same counters on batch-1 decode \citep{patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,yirage2026}. Part~VIII opens by naming what frameworks \emph{must} keep (request lifecycle, paging, sampling policy) versus what YiRage and MegaKernel stacks can absorb (intra-step residency, launches/token) \citep{chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dlbook}. The chapter goal is operational clarity: every engineer can point to the decode step box on a diagram and attach counters \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Chapter~27 flagged codesign and autonomous-kernel trends; Part~VIII opens by naming who owns each decode counter when those trends land in production \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference}. TaxBreak dense Llama-3.2-1B averages 847.5 kernel launches per output token on H100---often before framework-level batching helps at BS${=}1$ \citep{taxbreak2026}. vLLM-style serving reduces KV fragmentation and raises fleet utilization, but leaves operator boundaries inside the decoder step unless a compiler reunifies them \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,yirage2026}. Memory-Floor reports $1.259\times$ median speedup on H100 from CUDA Graph replay alone at BS${=}1$, ctx${=}2048$---orchestration relief that paging cannot replace when bytes/token stay high inside the step \citep{memoryfloor2026,nvidia2024cudagraphs}. Those three citations anchor the split: launches, paging, and graphs solve different layers of the same decode story \citep{taxbreak2026,memoryfloor2026,kwon2023vllm,nvidia2024cudagraphs,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. This chapter maps framework layers to Chapter~1 counters so Part~VI--VII compiler theory lands in a deployable stack, not a slide \citep{chen2020compilerbasedhardw,taxbreak2026,semianalysis2024tma,amd2024xdna,amd2024aie}.

Readers should leave with a deployment taxonomy: classify any production stack into framework-native, partial-export, or compiler-owned decode core (Table~\ref{tab:ch28_framework_buckets}), then profile \emph{both} scheduler queue time and in-step ms/token/bytes/token \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026}. Chapter~29 descends into YiRage Layer~5 Persistent Kernel runtime; Chapter~30 stitches the three-party contract \citep{yirage2026,chen2020compilerbasedhardw,dlbook}. Until then, treat every framework feature as a hypothesis that must survive BS${=}1$ counter worksheets \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026}. If you remember nothing else, remember the decode step box: everything inside is compiler and runtime turf; everything outside is framework turf \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Draw that box before every integration meeting and attach Chapter One counters to each side per decode token \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

Fregly-style mechanical sympathy applies at the \emph{boundary} between framework and compiler: the framework must not hide allocator behavior that forces HBM spill; the compiler must not assume contiguous KV or static batch shapes that break paging \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,patel2025llminference,dao2022flashattention,nvidia2024cudagraphs,dlbook}. Part~VI--VII taught pass ordering and dialect lowering; Part~VIII asks where those passes \emph{attach} in a live server \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. The answer is never ``replace the framework'' wholesale---it is to shrink the decode step until launches/token and bytes/token move on the same SKU you ship \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,dlbook}.

This chapter also connects backward to Chapter~12 execution modes and Chapter~23 LLM compile specialization \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,liu2024deepseekv3,dao2022flashattention,dlbook}. Eager versus graph versus MegaKernel is not only a training-time PyTorch concern---it is reproduced inside serving binaries through framework dispatch, CUDA graphs, and custom fused ops \citep{nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook}. DeepSeek-scale stacks add expert parallel communication below the scheduler; open components in \texttt{deps/DeepEP} and \texttt{deps/FlashMLA} show kernels that must compose with paging and batching rather than bypass them \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Read framework documentation as architecture diagrams with counters attached, not as marketing feature lists \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Framework landscape} spans HuggingFace \emph{eager} paths, vLLM/SGLang \emph{continuous-batching} servers, TensorRT-LLM/ONNX Runtime \emph{compiled} deployments, and custom C++ runtimes \citep{kwon2023vllm,patel2025llminference,dlbook,chen2020compilerbasedhardw,yirage2026}. Each makes different fusion depth assumptions: TRT-LLM may fuse within exported regions; HF PyTorch often does not; vLLM optimizes KV and batching while reusing framework attention backends \citep{patel2025llminference,dao2022flashattention,dao2023flashdecoding,memoryfloor2026,taxbreak2026}. SGLang extends the same serving layer with structured generation and radix-cache reuse---still framework-side orchestration unless a compiler owns the decoder cell \citep{patel2025llminference,kwon2023vllm,yirage2026}.

Production stacks rarely pick a single label from the list above. A common pattern is HuggingFace weights loaded into a vLLM server with FlashAttention backends and optional TensorRT plugins for narrow subgraphs \citep{kwon2023vllm,patel2025llminference,dao2022flashattention,dao2023flashdecoding,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. The compiler engineer job is to diagram where the decode step begins and ends in that stack: from slot assignment and block table lookup through final hidden state before sampling \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. Anything inside that window is fair game for fusion and residency optimization; anything outside is framework policy \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Ambiguity at the boundary is the main reason teams debate whether a regression is serving or compiler \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

Compiler engineers should classify any deployment into three buckets (Table~\ref{tab:ch28_framework_buckets}): (1) \emph{framework-native} decode (Python loop + vendor libs); (2) \emph{partial export} (compiled attention or MLP only); (3) \emph{compiler-owned decode core} (YiRage PersistentKernel or MegaKernel inside a thin framework shell) \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,liu2024deepseekv3,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}. Bucket choice determines whether your lever is batching policy or bytes/token \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026}. Teams that pick bucket (1) for time-to-market but benchmark only fleet tokens/s will misattribute compiler regressions to ``scheduling'' when launches/token never moved \citep{taxbreak2026,memoryfloor2026,patel2025llminference}.

\textbf{HuggingFace vs serving servers.} HF \texttt{transformers} optimizes researcher ergonomics: dynamic shapes, easy checkpoint swap, and Python-visible tensors \citep{patel2025llminference,dlbook}. Production servers trade that flexibility for slot-based scheduling, pinned memory pools, and optional CUDA graphs \citep{kwon2023vllm,nvidia2024cudagraphs,memoryfloor2026}. Compiler integration rarely starts in HF notebooks---it starts when a serving team exports a decode cell and asks which IR metadata (softmax carriers, page tables) survives the cut \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs}.

Research notebooks prove model correctness; serving binaries prove goodput under load \citep{patel2025llminference,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Teams should freeze a serving configuration---block size, max batch, dtype---before comparing compiler exports \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. Otherwise HF eager baselines become moving targets that invalidate A/B conclusions \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{TensorRT-LLM and compiled paths.} TRT-LLM and similar engines push fusion into builder-time regions but still sit behind a framework or C++ server for batching and KV \citep{patel2025llminference,nvidia2024cudagraphs,dlbook}. Partial export wins when attention or MLP dominates bytes/token; it fails when export boundaries fall \emph{inside} online-softmax state or paged KV gather/scatter \citep{dao2022flashattention,dao2023flashdecoding,kwon2023vllm,memoryfloor2026,yirage2026}. Profile export candidates on BS${=}1$ decode, not only engine build logs \citep{taxbreak2026,patel2025llminference,memoryfloor2026}.

\textbf{Operational ownership.} In production, three teams touch the same decode path: serving (framework), compiler (graph lowering), and hardware (SKU drivers) \citep{patel2025llminference,chen2020compilerbasedhardw,yirage2026,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024xdna,amd2024aie,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma}. Without Table~\ref{tab:ch28_framework_buckets} vocabulary, incidents become blame loops while bytes/token flatlines \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Runbooks should record bucket, shape bucket, paging block size, and export boundary hash alongside fleet dashboards \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,dlbook}.

\textbf{SGLang and structured serving.} Structured generation adds mask and grammar state to the framework layer---still outside compiler fusion unless exported \citep{patel2025llminference,kwon2023vllm,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. RadixAttention-style prefix reuse compounds paging benefits; compiler cores must not invalidate shared prefixes with per-request recompilation \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}.

\paragraph{Multi-hardware delta.} Framework support matrices differ: CUDA-first servers dominate cloud; CPU/OpenVINO paths serve edge; Ascend/MACA stacks require vendor runtimes \citep{amd2024aie,amd2024xdna,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}. YiRage targets the same decode core across backends---framework integration must not reintroduce GPU-only assumptions at the API boundary \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,liu2024deepseekv3,kwon2023vllm,taxbreak2026,memoryfloor2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2022hopper,nvidia2024cudagraphs}. An integration that hard-codes CUDA graph capture without NPU static-shape buckets will pass GPU CI and fail edge deployment \citep{amd2024aie,chen2020compilerbasedhardw,yirage2026}.

\begin{example}
A team runs vLLM for KV paging but keeps PyTorch-eager MLP boundaries: paging wins capacity while launches/token stay TaxBreak-high---framework choice fixed scheduling, not fusion \citep{kwon2023vllm,taxbreak2026,patel2025llminference,yirage2026,memoryfloor2026}. Moving to bucket (3) requires exporting the full decode cell with page-table layout IR, not swapping attention alone \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dao2022flashattention}.
\end{example}

\textbf{Migration path from eager to compiler-owned core.} Most teams land in bucket (1), prove fleet demand, then partial-export attention or MLP while paging stabilizes \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}. The risky middle phase is partial export with hidden boundaries inside RMSNorm or rotary embedding: microbenchmarks show wins while bytes/token flatlines because carriers spill between exported regions \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,kwon2023vllm,dlbook}. A disciplined migration keeps the framework scheduler and block allocator fixed while shrinking the decode cell boundary each release, measuring launches/token and bytes/token on the same context buckets after every export change \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. Skip the worksheet step and you will ship bucket (2) forever, wondering why graph capture helped but cost per million tokens did not \citep{memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Continuous batching} interleaves prefill and decode requests to raise GPU occupancy \citep{kwon2023vllm,patel2025llminference,liu2024deepseekv3,dlbook}. Classic static batching waits until $N$ requests align; continuous batching admits new sequences when others finish prefill or free decode slots---improving \emph{fleet} goodput (tokens/s per GPU) \citep{kwon2023vllm,patel2025llminference,memoryfloor2026}. It does not automatically cut \emph{per-request} bytes/token at BS${=}1$ inside a single decode step \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw}. A scheduler that doubles fleet utilization while leaving hundreds of launches per token untouched has solved capacity, not mechanical sympathy inside the decoder \citep{taxbreak2026,kwon2023vllm,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}.

Serving engineers often first encounter continuous batching as a throughput win in load tests \citep{kwon2023vllm,patel2025llminference,dlbook,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. Compiler engineers should insist on paired microbench: same server binary, same paging policy, but forced BS${=}1$ decode cell measurement between scheduler iterations \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. If only fleet metrics move, the next investment belongs in export boundaries and PersistentKernel integration, not longer batching windows \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{Prefill vs decode in the scheduler.} Prefill is compute-heavy and benefits from large token-parallel batches; decode at BS${=}1$ per sequence is memory- and orchestration-bound \citep{dao2022flashattention,patel2025llminference,memoryfloor2026}. Frameworks therefore use iteration-level scheduling: each engine step may mix prefill chunks and decode tokens from different clients \citep{kwon2023vllm,patel2025llminference,liu2024deepseekv3}. Compiler integration must preserve batching invariants---dynamic sequence lengths, slot assignment, cancellation---while still allowing graph capture or MegaKernel regions for steady decode cells \citep{nvidia2024cudagraphs,yirage2026,kwon2023vllm,liu2024deepseekv3,taxbreak2026,memoryfloor2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,dlbook,amd2024xdna,amd2024aie,nvidia2022hopper}. Exporting a fused kernel that assumes fixed batch shapes breaks continuous batching unless the runtime provides shape buckets \citep{nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,memoryfloor2026}.

\textbf{Fleet goodput vs per-request counters.} Fleet metrics---tokens/s, cost per million tokens, GPU utilization---reward batching and paging \citep{kwon2023vllm,patel2025llminference,dlbook}. Per-request counters---ms/token, bytes/token, launches/token at BS${=}1$---reward fusion and residency inside the step \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw}. Production teams need both: fleet metrics for capacity planning; per-request counters for compiler A/B and p99 latency under load \citep{patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,yirage2026}. Framework schedulers expose \emph{when} a decode step runs; compilers decide \emph{what} executes inside the step \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,dlbook}. Profile both: scheduler queue time vs in-step ms/token \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook}.

Document both metric families on the same release dashboard \citep{patel2025llminference,kwon2023vllm,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Fleet tokens/s may rise while BS${=}1$ bytes/token stalls---label that as a scheduling win, not a fusion win \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Fusion wins that break batching legality should fail integration tests even when microbenchmarks improve \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. The contract runs both directions: compilers preserve paging and mask invariants; frameworks expose stable plugin ABIs \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}.

\textbf{Chunked prefill and decode interleaving.} Long prefills can starve decode slots; frameworks chunk prefills and interleave decode steps to bound latency \citep{kwon2023vllm,patel2025llminference,liu2024deepseekv3}. Compiler-owned cores must support partial prefill shapes without recompilation storms---IR bucket tags for context length and batch composition are framework-facing contracts \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,memoryfloor2026}. DeepSeek-scale MoE adds expert-parallel scheduling on top of token batching \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026}.

\textbf{Scheduling policies and SLOs.} Frameworks expose FCFS, priority queues, and preemption for premium tenants \citep{kwon2023vllm,patel2025llminference,dlbook,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. p99 latency is often scheduler-bound under burst load even when BS${=}1$ kernels are optimal \citep{patel2025llminference,memoryfloor2026,kwon2023vllm,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook}. Capacity planning should separate queueing SLOs from in-step SLOs \citep{patel2025llminference,kwon2023vllm,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,amd2024aie,amd2024xdna,dlbook}.

\textbf{Watermarking and backpressure.} When KV pools exhaust, frameworks reject or delay requests \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,liu2024deepseekv3,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,dlbook}. Compiler compression of KV shifts watermarks but requires framework allocator awareness \citep{liu2024deepseekv3,patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}.

\paragraph{Multi-hardware delta.} CPU-served frameworks batch differently---OpenVINO and llama.cpp paths use thread pools and blocked KV rather than GPU slot tables \citep{dlbook,amd2024aie,patel2025llminference,yirage2026,chen2020compilerbasedhardw}. NPU serving often static-buckets requests by shape; continuous batching semantics must be emulated in software or sacrificed for compile-time legality \citep{amd2024xdna,amd2024aie,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026}.

\begin{example}
Under load, fleet tokens/s rises 40\% after enabling continuous batching, but BS${=}1$ microbench launches/token unchanged---expected if batching fixed occupancy, not fusion \citep{kwon2023vllm,taxbreak2026,patel2025llminference,memoryfloor2026,yirage2026}. Next lever: compiler-owned decode core inside the same scheduler shell \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs}.
\end{example}

\textbf{Iteration-level scheduling mechanics.} Continuous batching implementations maintain a running batch tensor whose rows map to live sequence slots \citep{kwon2023vllm,patel2025llminference,dlbook,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. Each engine iteration may append new prefill tokens for one client while advancing decode by one token for others in the same launch \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,amd2024aie,amd2024xdna,dlbook}. Compiler-owned cores must accept dynamic active-row masks without recompilation per arrival: shape buckets cover max batch width while runtime masks disable finished sequences \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Framework engineers own the slot table; compiler engineers own legality of fused kernels across masked rows \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

\section{kv paging serv}
\label{sec:ch28_kv_paging_serv}

\textbf{KV paging serv} (PagedAttention and descendants) treats KV cache as allocatable blocks---framework memory management, not compiler math \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dao2023flashdecoding,liu2024deepseekv3,dlbook,amd2024xdna,amd2024aie,semianalysis2024tma,dao2022flashattention,nvidia2022hopper,nvidia2024cudagraphs}. Without paging, contiguous KV reservations fragment GPU memory as sequences grow and shrink; paging maps logical token positions to physical blocks, improving fleet capacity \citep{kwon2023vllm,patel2025llminference,memoryfloor2026}. Compilers must consume page tables as layout IR (Chapter~13), not fight them with contiguous-KV-only fusion plans \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,dlbook}.

Paging is the reason large-context serving became economically viable on single-GPU deployments: operators trade small internal fragmentation for predictable allocation \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dao2022flashattention,dao2023flashdecoding,liu2024deepseekv3,semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,amd2024xdna,amd2024aie,dlbook}. It is not a substitute for lowering bytes moved during attention itself \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Teams celebrating paging-only wins should still run Memory-Floor style byte ledgers on decode steps \citep{memoryfloor2026,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dao2022flashattention,dlbook}.

\textbf{Block size and attention tiles.} Block size trades fragmentation vs attention tile efficiency: smaller blocks reduce waste; larger blocks improve vectorized loads \citep{kwon2023vllm,dao2022flashattention,dao2023flashdecoding,patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,dlbook,liu2024deepseekv3,amd2024xdna,amd2024aie,nvidia2022hopper,nvidia2024cudagraphs}. FlashAttention-style kernels assume block-aligned KV access; paging block size should co-tune with compiler tile search on the same context buckets \citep{dao2022flashattention,dao2023flashdecoding,yirage2026,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. A paging policy chosen by SREs without compiler input can force gather/scatter patterns that erase fusion wins \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026}.

Run a joint review when either team changes block size or attention tile width \citep{kwon2023vllm,dao2022flashattention,dao2023flashdecoding,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dlbook,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,nvidia2024cudagraphs,dlbook}. The review output is a signed counter sheet, not a slide deck \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Prefix caching and radix reuse.} SGLang-style radix caches reuse KV blocks across prompts with shared prefixes---framework-side deduplication \citep{patel2025llminference,kwon2023vllm,dlbook}. Compiler cores must treat cached blocks as valid layout inputs: same page table semantics, different allocation provenance \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference}. Exporting attention without page-table metadata breaks both vanilla paging and prefix reuse \citep{dao2022flashattention,dao2023flashdecoding,yirage2026,memoryfloor2026}.

Prefix reuse changes the economics of long system prompts: the framework avoids recomputing shared KV while the compiler still must legalize attention reads against block tables for the suffix tokens \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,dao2023flashdecoding,memoryfloor2026,taxbreak2026,dlbook,liu2024deepseekv3,semianalysis2024tma,nvidia2024cudagraphs,amd2024xdna,amd2024aie,nvidia2022hopper,dlbook}. Compiler teams should test prefix-hit and prefix-miss paths separately because gather patterns and occupancy differ \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dao2022flashattention,memoryfloor2026,taxbreak2026,dlbook}. Treat prefix cache as another paging policy knob co-tuned with tile search \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}.

\textbf{MLA and compressed KV.} DeepSeek-V3 MLA reduces KV footprint; paging and compiler layout must agree on compressed head dimensions \citep{liu2024deepseekv3,patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026}. Open components under \texttt{deps/FlashMLA} illustrate MLA kernel expectations that framework paging must feed \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference}.

\textbf{Memory accounting worksheet.} Fleet operators should track bytes per KV block, active blocks per sequence, peak concurrent sequences, and fragmentation ratio \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,liu2024deepseekv3,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,dlbook}. Compiler teams add bytes moved per decode step at BS${=}1$ for the same paging policy \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dao2022flashattention,dlbook}. Disagreement between allocator metrics and profiler HBM counters signals layout IR mismatch \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,dao2023flashdecoding,dlbook}.

\textbf{Copy and migration costs.} Disaggregated prefill/decode copies KV blocks---framework orchestration with compiler-visible layout \citep{patel2025llminference,kwon2023vllm,liu2024deepseekv3,dlbook,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,dlbook}. Fusion plans that assume KV stays local break under migration unless gathers are legal across NUMA or NVLink paths \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,liu2024deepseekv3,dlbook}.

\paragraph{Multi-hardware delta.} GPU paging uses CUDA virtual memory and block pools; CPU paths use pinned host buffers; NPUs use statically reserved SRAM windows \citep{dlbook,amd2024aie,amd2024xdna,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,dao2023flashdecoding}. A compiler that lowers paged KV only for contiguous GPU tensors will fail on XDNA static layouts \citep{amd2024xdna,amd2024aie,chen2020compilerbasedhardw,yirage2026}.

\begin{example}
Block size 16 cuts fragmentation but increases attention loop trips; co-tuning with FlashDecoding tile width recovered bytes/token while keeping paging benefits \citep{kwon2023vllm,dao2023flashdecoding,memoryfloor2026,yirage2026,patel2025llminference,taxbreak2026}.
\end{example}

\textbf{PagedAttention internals for compiler authors.} Logical token position maps through a block table to physical GPU memory blocks; attention kernels index KV via block id and offset rather than a flat stride \citep{kwon2023vllm,dao2022flashattention,dao2023flashdecoding,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dlbook,liu2024deepseekv3,semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,amd2024xdna,amd2024aie,dlbook}. Compiler layout IR must represent gathers from block tables as first-class ops with known alignment and dtype, not as opaque custom calls that break fusion legality analysis \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,dao2022flashattention,dao2023flashdecoding,memoryfloor2026,taxbreak2026,dlbook,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,nvidia2022hopper,nvidia2024cudagraphs,dlbook}. When MegaKernel fusion plans assume contiguous KV tensors, paging forces either illegal fusion or a gather prologue that dominates bytes/token \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,patel2025llminference,dao2022flashattention,dao2023flashdecoding,dlbook,semianalysis2024tma,liu2024deepseekv3,amd2024xdna,amd2024aie,nvidia2022hopper,nvidia2024cudagraphs,dlbook}. Treat block tables like weight layouts in Chapter~13: co-design block policy with tile search \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}.

\section{framework bottlenecks}
\label{sec:ch28_framework_bottlenecks}

\textbf{Framework bottlenecks} mirror Chapter~1: launches/token, bytes/token, and orchestration tax---often introduced \emph{above} the decoder kernel by Python loops, dispatcher churn, and unfused vendor op sequences \citep{taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,nvidia2022hopper}. Framework A/B that reports only tokens/s without per-step byte ledgers misattributes compiler regressions to scheduling \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}.

When triaging a slow server, split the timeline before optimizing kernels \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Queue wait implicates admission control and batching policy \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024aie,amd2024xdna,dlbook}. High in-step milliseconds with moderate launches implicates bytes/token and fusion gaps \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,dao2023flashdecoding,kwon2023vllm,dlbook}. High launches with moderate bytes implicates orchestration and graph capture opportunities \citep{memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Skipping this split sends compiler teams to rewrite attention while the framework still dispatches hundreds of tiny kernels \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Python and host overhead.} Eager HF loops pay GIL, tensor metadata, and per-op Python dispatch \citep{patel2025llminference,dlbook,taxbreak2026,memoryfloor2026}. Serving frameworks move hot paths to C++/CUDA but still coordinate sampling, stop criteria, and RPC on host \citep{kwon2023vllm,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026}. CUDA Graph capture removes launch storms for steady decode shapes; it does not fuse illegal HBM traffic \citep{memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dao2022flashattention}. Teams that graph-capture unfused graphs lock in high bytes/token at lower launch count \citep{memoryfloor2026,taxbreak2026,patel2025llminference,nvidia2024cudagraphs,yirage2026}.

\textbf{Operator chains inside the step.} Even ``optimized'' servers often call separate attention, norm, and MLP kernels each token \citep{taxbreak2026,kwon2023vllm,patel2025llminference,dao2022flashattention,dao2023flashdecoding,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3}. TaxBreak counts the launches; Memory-Floor counts the bytes \citep{taxbreak2026,memoryfloor2026}. Framework bottlenecks end where compiler fusion begins---but only if integration exports the whole decode cell, not a single op \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,dlbook}.

Walk one token through the server call stack in a profiler and count CUDA API entries between scheduler entry and sampling exit \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Each entry is a fusion boundary candidate \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,dlbook}. Prioritize boundaries that move activations to HBM between norm and matmul or between attention and MLP \citep{dao2022flashattention,dao2023flashdecoding,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,dlbook,liu2024deepseekv3,semianalysis2024tma,nvidia2024cudagraphs,amd2024aie,amd2024xdna,nvidia2022hopper,dlbook}. That ordered list becomes the export roadmap for bucket (3) migration \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{Speculative decoding overhead.} Draft models add scheduler complexity: verify steps, acceptance sampling, and dual KV streams \citep{patel2025llminference,kwon2023vllm,liu2024deepseekv3,dlbook}. Frameworks must slot draft and target sequences; compilers should fuse verify cells where numerics allow \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,memoryfloor2026,taxbreak2026,patel2025llminference,nvidia2024cudagraphs}. Profile speculative paths separately---fleet speedup can hide verify-step launch storms \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,kwon2023vllm,liu2024deepseekv3,dlbook}.

\textbf{Observability split.} Instrument four spans per request: queue wait, prefill steps, decode steps, and sampling/host \citep{patel2025llminference,kwon2023vllm,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. nsys rows alone are insufficient if framework queue time dominates p99 \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Export traces with bucket id, block size, and export hash for compiler A/B \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,dlbook}.

\textbf{RPC and tokenizer tax.} Front-end frameworks pay tokenization, detokenization, and gRPC serialization---outside GPU counters but inside user-perceived latency \citep{patel2025llminference,dlbook,kwon2023vllm,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Compiler wins on bytes/token do not remove host bottlenecks; they prevent misallocated engineering weeks \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\paragraph{Multi-hardware delta.} CPU-served frameworks hit GIL and memcpy costs; GPU frameworks hit launch storms; NPU frameworks hit static-shape bucket misses \citep{dlbook,amd2024aie,amd2024xdna,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,dao2023flashdecoding}. Bottleneck triage starts with counter worksheets per target, not framework brand \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook}.

\begin{example}
Replacing Python loop with CUDA graphs improved ms/token 12\% at BS${=}1$ but bytes/token flat---graphs fixed orchestration, not fusion \citep{memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,yirage2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,dlbook}.
\end{example}

\textbf{Fair framework comparison methodology.} Benchmark posts that compare vLLM, SGLang, and TRT-LLM using only aggregate tokens/s at fixed concurrency often reflect scheduler defaults and paging block sizes rather than intrinsic decode efficiency \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dlbook,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,dlbook}. A fair comparison fixes model checkpoint, dtype, context distribution, block size, and GPU SKU, then reports BS${=}1$ launches/token and bytes/token alongside fleet curves \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. Without the fixed-cell worksheet, framework marketing numbers cannot inform compiler roadmap prioritization \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Chapter~1 counters exist precisely to prevent apples-to-oranges serving benchmarks from driving fusion decisions \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dlbook}.

\section{compiler integration hooks}
\label{sec:ch28_compiler_integration_hooks}

\textbf{Compiler integration hooks} are the seams where YiRage attaches: exported FX/ONNX regions, custom op \emph{plugin} registration, CUDA graph capture buckets, and PersistentKernel call sites \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,liu2024deepseekv3,dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,nvidia2022hopper}. Hooks should preserve numerics carriers from Chapter~10 and residency metadata from Chapter~6---exporting a subgraph without those invariants recreates silent HBM spill \citep{dao2022flashattention,yirage2026,memoryfloor2026,taxbreak2026,patel2025llminference,chen2020compilerbasedhardw,dlbook,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper}.

Integration design meetings often fail because participants use the word kernel loosely \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. A framework custom op may wrap one CUDA kernel or an entire PersistentKernel program with internal loops \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. The hook contract must specify which side owns slot tables, which side owns softmax state, and which side owns graph capture replay \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dlbook}. Ambiguous ownership produces double-free style bugs in performance: two layers both ``optimize'' the same boundary and break numerics \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,memoryfloor2026,taxbreak2026,kwon2023vllm,dlbook}.

\textbf{FX and ONNX export regions.} PyTorch FX traces decode cells for compiler lowering; ONNX export targets TRT and ORT runtimes \citep{patel2025llminference,chen2020compilerbasedhardw,dlbook,nvidia2024cudagraphs,yirage2026}. Export boundaries must include page-table gathers, rotary embeddings, and online-softmax state---not only matmuls \citep{dao2022flashattention,dao2023flashdecoding,kwon2023vllm,yirage2026,memoryfloor2026,taxbreak2026,chen2020compilerbasedhardw,patel2025llminference,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024aie,amd2024xdna,dlbook}. Broken exports pass unit tests on square tensors and fail on paged decode \citep{kwon2023vllm,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dao2022flashattention}.

\textbf{Custom op plugins.} Frameworks register \texttt{torch.library} or vLLM custom ops that delegate to compiled .so modules \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,dlbook,taxbreak2026,memoryfloor2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,nvidia2022hopper,dao2023flashdecoding}. Plugin APIs are the stable contract: framework passes slot ids, page tables, and sampling flags; compiler returns fused launch descriptors \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. Version skew between framework paging and plugin layout IR is a common production outage mode \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026}.

Treat each plugin like a narrow ABI: document struct layouts, alignment, dtype, and error codes; reject silent fallbacks to eager ops \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Fallbacks hide fusion failures behind correct-but-slow paths that pass functional tests while burning fleet budget \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}. Integration CI should fail when fallback triggers except in explicitly whitelisted debug modes \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,dao2022flashattention,dlbook}.

\textbf{CUDA graph buckets and PersistentKernel.} Graph capture requires static launch templates per shape bucket \citep{nvidia2024cudagraphs,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,dlbook,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper}. YiRage PersistentKernel mode collapses launches inside the compiled core while the framework keeps scheduling---Chapter~29 details runtime handoff \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,liu2024deepseekv3,dao2022flashattention,dlbook,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper}. Integration tests should cover: bucket switch, cancel mid-sequence, and prefix-cache hit \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,dao2022flashattention,dlbook}.

Shape buckets should align with framework batch width limits and max context policies so capture replay does not miss rare production shapes \citep{nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. PersistentKernel programs extend buckets with internal iteration over layers while presenting one host launch per decode step---the pattern Chapter~29 documents in \texttt{deps/YiRage} \citep{yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,dlbook}. Frameworks must not assume one graph per model; they should register one graph per bucket per compiled fingerprint \citep{nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}.

Design rule: frameworks own \emph{request and KV lifecycle}; compilers own \emph{decode-step residency}; runtime owns \emph{launch and synchronization} inside the compiled core (Chapter~29) \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dlbook,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper}. Chapter~30 formalizes the split for production stacks \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook}. DeepSeek open components (\texttt{deps/DeepEP}, \texttt{deps/FlashMLA}) show what must plug below framework schedulers for MoE at scale \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

\textbf{Integration test matrix.} Minimum CI scenarios: BS${=}1$ ctx 512/2k/8k; continuous batch mix; prefix-cache hit; cancel mid-stream; bucket switch after compile \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Each scenario logs launches/token and bytes/token \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dlbook}.

\textbf{Rollback and fingerprint caches.} YiRage superoptimize caches fused artifacts by IR fingerprint; framework upgrades that change page-table ABI must bump fingerprint salts \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,dlbook,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. Silent cache hits across incompatible paging policies are production footguns \citep{yirage2026,kwon2023vllm,patel2025llminference,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,dlbook}.

\paragraph{Multi-hardware delta.} Hooks differ by backend: CUDA plugins vs XDNA AIE binaries vs CPU AVX512 kernels---the API shape should stay constant while layout IR lowers per target \citep{amd2024xdna,amd2024aie,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook,semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,liu2024deepseekv3}.

\begin{example}
vLLM custom op wrapping YiRage superoptimized decode cell: framework passes block tables and slot map; plugin returns single PK launch---launches/token dropped from hundreds to single digits while paging unchanged \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,taxbreak2026,patel2025llminference,memoryfloor2026,nvidia2024cudagraphs,dao2022flashattention,dlbook}.
\end{example}

\textbf{ABI stability across releases.} Plugin boundaries serialize slot indices, block table pointers, sampling parameters, and optional speculative-draft handles \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,nvidia2024cudagraphs,memoryfloor2026,taxbreak2026,dao2022flashattention,liu2024deepseekv3,semianalysis2024tma,dlbook,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper,dlbook}. Framework minor version bumps that reorder block table fields without coordinated compiler releases produce silent wrong-token bugs worse than a hard crash \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,dlbook}. Pin paired framework and compiler versions in production manifests; run golden-token tests on every bump \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dao2022flashattention,memoryfloor2026,taxbreak2026,nvidia2024cudagraphs,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. Chapter~30 extends this into a three-party release train \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.

\section{Key Takeaways}
\label{sec:ch28_key_takeaways}

\begin{itemize}
\item \textbf{Frameworks schedule; compilers fuse.}
Continuous batching interleaves prefill and decode requests to raise GPU occupancy and fleet utilization, but it does nothing about the batch-1 bytes/token inside a single decode step \citep{kwon2023vllm,patel2025llminference}. The framework owns request scheduling and the compiler owns intra-step fusion, so a serving win at one layer must not be mistaken for a win at the other \citep{taxbreak2026,memoryfloor2026}.

\item \textbf{Paged KV is framework layout the compiler must lower.}
PagedAttention treats the KV cache as allocatable blocks---framework memory management, not compiler math---but the compiler must still lower those page tables as first-class IR so fusion can see page boundaries \citep{kwon2023vllm,yirage2026}. Hiding the paging from the compiler forces gather/scatter that breaks on-chip residency at the attention seam \citep{chen2020compilerbasedhardw,patel2025llminference}.

\item \textbf{Measure both layers, not one.}
Framework bottlenecks mirror Chapter~1---launches/token, bytes/token, orchestration tax---but are often introduced \emph{above} the kernel by Python loops and dispatcher churn, so the diagnosis must report queue/scheduling time alongside in-step ms/token and bytes/token \citep{taxbreak2026,patel2025llminference}. A fast kernel behind a slow scheduler still misses the SLO \citep{memoryfloor2026,dlbook}.

\item \textbf{Integration hooks are contracts, and parity is explicit.}
The seams where a compiler attaches---exported FX/ONNX regions, custom-op plugins, CUDA-graph capture buckets, PersistentKernel call sites---must preserve softmax carriers and capture legality across the boundary, or fusion silently regresses \citep{dao2022flashattention,yirage2026,nvidia2024cudagraphs}. Framework defaults are CUDA-centric, so multi-hardware parity requires explicit triplet tests on the YiRage backends rather than assumed equivalence \citep{amd2024xdna,amd2024aie}.

\item \textbf{Migrate by shrinking the decode export, and version the ABI.}
Move a workload bucket by bucket, shrinking the decode export boundary, and never confuse a paging win with a fusion win; before any PersistentKernel work, log the bucket, decode-box boundaries, queue wait, and in-step counters at context 512/2k/8k \citep{kwon2023vllm,patel2025llminference}. Framework paging and compiler plugins must release together with golden-token tests, since an ABI skew between them corrupts output in ways a counter dashboard will not catch \citep{yirage2026,memoryfloor2026}.
\end{itemize}

\section{Conclusion}
\label{sec:ch28_conclusion}

Inference frameworks solve fleet scheduling and KV capacity; they do not replace compiler-side fusion and runtime-side launch collapse \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,dao2022flashattention,dlbook,semianalysis2024tma,amd2024xdna,amd2024aie,dao2023flashdecoding,nvidia2022hopper}. Classify your stack with Table~\ref{tab:ch28_framework_buckets}, profile scheduler and in-step counters separately, and treat every plugin boundary as a numerics and layout contract \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,dao2022flashattention,nvidia2024cudagraphs,dlbook}. The practical outcome of this chapter is a shared vocabulary between serving and compiler teams: when paging wins but bytes/token stalls, the next owner is the decode cell export, not the scheduler \citep{kwon2023vllm,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,liu2024deepseekv3,semianalysis2024tma,amd2024xdna,amd2024aie,dlbook}. Before opening Chapter~29, run one end-to-end worksheet on your production stack: name the bucket, draw the decode step box, log queue wait and in-step counters for ctx 512/2k/8k, and list every plugin boundary with numerics carriers attached \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,dlbook,liu2024deepseekv3,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding,nvidia2022hopper,dlbook}. That worksheet becomes the acceptance criteria for PersistentKernel integration in the next chapter \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}. Chapter~29 descends into YiRage Layer~5 Persistent Kernel runtime (\texttt{deps/YiRage} submodule); Chapter~30 stitches framework, compiler, and runtime into one decode goodput workflow \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,dlbook}.
