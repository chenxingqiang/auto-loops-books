The production surprise at \textbf{GPU TMA} is not peak FLOPs; it is illegal tier placement: TMA spilling to HBM/DDR while Hopper still pays launch or DMA tax every token \citep{nvidia2022hopper,semianalysis2024tma}.

Trace residency: TMA producers must feed Hopper consumers without a DDR round trip \citep{dao2022flashattention,yirage2026}. Silent spill reads as ``mysterious'' p99 regression \citep{patel2025llminference,semianalysis2024tma}.

Operational fix: diff IR or CUDA for new allocs; count launches/token (GPU) or DMA edges (NPU); require graph capture legality on GPU and static legality on XDNA \citep{nvidia2024cudagraphs,amd2024aie}.

\section{CPU dma}
\label{sec:ch08_cpu_dma}

\textbf{CPU dma} is where TMA Double-Buffer Pipelines and Async Movement meets Chapter~1 counters: fix dma and CPU together or bytes/token flatlines while kernel microbenchs celebrate \citep{amd2024xdna,amd2024aie}.

Hardware truth: GPUs bound dma with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on dma/CPU legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper may legalize dma with TMA double-buffer; XDNA requires static tile assignment; CPU needs cache-blocked CPU---identical math, different bytes/token \citep{semianalysis2024tma,amd2024xdna,dlbook}.
