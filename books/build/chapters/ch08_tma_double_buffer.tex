Teams hit \textbf{pipeline theory} when TMA Double-Buffer Pipelines and Async Movement looks correct in isolation yet decode telemetry disagrees---usually because pipeline was planned without double-buffer residency on the SKU you ship \citep{dao2022flashattention,yirage2026}.

Hardware truth: GPUs bound pipeline with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on pipeline/double-buffer legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper may legalize pipeline with TMA double-buffer; XDNA requires static tile assignment; CPU needs cache-blocked double-buffer---identical math, different bytes/token \citep{semianalysis2024tma,amd2024xdna,dlbook}.

\end{example}

The production surprise at \textbf{GPU TMA} is not peak FLOPs; it is illegal tier placement: TMA spilling to HBM/DDR while Hopper still pays launch or DMA tax every token \citep{nvidia2022hopper,semianalysis2024tma}.

Trace residency: TMA producers must feed Hopper consumers without a DDR round trip \citep{dao2022flashattention,yirage2026}. Silent spill reads as ``mysterious'' p99 regression \citep{patel2025llminference,semianalysis2024tma}.

Operational fix: diff IR or CUDA for new allocs; count launches/token (GPU) or DMA edges (NPU); require graph capture legality on GPU and static legality on XDNA \citep{nvidia2024cudagraphs,amd2024aie}.

\paragraph{Review gate.} Attach roofline or buffer accounting for TMA on target SKU before merge \citep{memoryfloor2026,yirage2026}.

\section{CPU dma}
\label{sec:ch08_cpu_dma}

\textbf{CPU dma} is where TMA Double-Buffer Pipelines and Async Movement meets Chapter~1 counters: fix dma and CPU together or bytes/token flatlines while kernel microbenchs celebrate \citep{amd2024xdna,amd2024aie}.

Hardware truth: GPUs bound dma with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on dma/CPU legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper may legalize dma with TMA double-buffer; XDNA requires static tile assignment; CPU needs cache-blocked CPU---identical math, different bytes/token \citep{semianalysis2024tma,amd2024xdna,dlbook}.

\begin{example}
Decode cell (BS${=}1$, ctx${=}2048$): unfused dma raises bytes/token; fused schedule must keep CPU on-chip across GPU, CPU, and NPU baselines \citep{taxbreak2026,memoryfloor2026,yirage2026}.
\end{example}

\section{Chapter Summary}
\label{sec:ch08_summary}

TMA Double-Buffer Pipelines and Async Movement succeeds when hardware constraints drive schedules and decode metrics prove relief \citep{yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw}. \textbf{pipeline theory} exposes a cross-hardware fracture: the same fused subgraph legal on Hopper can be rejected outright on XDNA or CPU cache budgets.

Binding software to silicon: compiler passes must encode pipeline/double-buffer legality before codegen, not as comments in hand-written kernels \citep{dao2022flashattention,yirage2026}. YiRage runs target-aware checks against chip models from Chapter~3; manual stacks should treat failed legality as a release blocker, not a backend TODO.

When pipeline theory disagrees with microbenchmarks, trust fleet telemetry: fix orchestration and tier placement first, then revisit MMA tile shapes \citep{dao2022flashattention,yirage2026}.

\paragraph{Review gate.} Before merging compiler changes affecting pipeline theory, attach roofline or NPU buffer accounting showing pipeline residency fits on-chip for the target SKU \citep{dao2022flashattention,yirage2026}.

\paragraph{Worked contrast.} A Hopper decode cell may legalize pipeline with TMA-backed double buffering while an XDNA cell requires the same math expressed as static tile assignments---identical FLOPs, different bytes/token because DDR traffic differs by an order of magnitude \citep{dao2022flashattention,yirage2026}.

The usual TMA Double-Buffer Pipelines and Async Movement failure at \textbf{GPU TMA} is importing a CUDA mental model and expecting runtime scheduling to hide illegal buffer lifetimes.

Hardware constraint: TMA and Hopper must be co-designed. On GPUs that means shared-memory/TMA budgets and warp-grain barriers; on XDNA/AIE it means static tile buffers, DMA chains, and carrier slots with compile-time lifetimes; on CPUs it means L2/L3 blocking and NUMA-local KV placement \citep{nvidia2022hopper,semianalysis2024tma}. Decode at batch-1 keeps arithmetic intensity on the memory-bandwidth slope---micro-optimizing isolated GEMMs rarely moves p99 latency.

Compiler takeaway for GPU TMA: lower the same decoder IR, but predicate passes on target residency tables---never ship a GPU-only schedule to NPU fleets \citep{nvidia2022hopper,semianalysis2024tma}. Success metrics: bytes/token, launches/token, and p99 decode latency; compare GPU, CPU, and NPU cells on identical context lengths.
