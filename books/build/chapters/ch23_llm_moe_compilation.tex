\label{chap:ch23}
\typeout{START_CHAPTER "ch23" \theabspage}

Dense decoder LLMs and mixture-of-experts (MoE) models break the assumptions generic compiler stacks were built for: batch-1 decode, growing KV residency, routed expert subgraphs, and framework-owned paging policies \citep{patel2025llminference,kwon2023vllm,liu2024deepseekv3,taxbreak2026}. Part~VI taught backend lowering; Chapter~22 taught E2E tuning loops; this chapter specializes those tools for \textbf{LLM decode compilation} and \textbf{MoE-aware schedules} across GPU, CPU, and NPU targets \citep{chen2020compilerbasedhardw,yirage2026,amd2024xdna,amd2024aie,dlbook}.

TaxBreak dense Llama-3.2-1B averages 847.5 kernel launches per output token on H100; OLMoE-1B/7B averages 9,305 \citep{taxbreak2026}. Memory-Floor CUDA Graphs on Qwen-2.5-7B (BS${=}1$, ctx${=}2048$) yields $1.259\times$ on H100 by removing host overhead---yet cannot fix illegal HBM traffic when MoE routing and KV export remain unfused \citep{memoryfloor2026,nvidia2024cudagraphs,liu2024deepseekv3}. Specialized compilation means every pass knows whether it is optimizing prefill, decode, or expert subgraphs---and refuses to merge them into one misleading benchmark \citep{patel2025llminference,dao2022flashattention,dao2023flashdecoding,kwon2023vllm,yirage2026,chen2020compilerbasedhardw}.

Four topics anchor the chapter: \textbf{LLM decode compile} (single-request residency), \textbf{MoE scheduling} (routing + expert fusion), \textbf{KV hardware adaptation} (layout and paging boundaries), and \textbf{heterogeneous MoE load balancing} (multi-SKU expert placement) \citep{liu2024deepseekv3,amd2024xdna,semianalysis2024tma,dlbook}. DeepSeek-class deployments disaggregate prefill and decode; compiler specialization must follow that split instead of fighting it \citep{patel2025llminference,liu2024deepseekv3,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,amd2024aie,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Anti-patterns: compiling whole-model graphs without decode role tags; fusing through PagedAttention handles; ignoring routing entropy in MoE profiles; promoting GPU expert schedules to CPU fallback without re-profile \citep{kwon2023vllm,liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Each section below replaces one anti-pattern with a specialized compile practice \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Speculative decoding and draft models add another decode subgraph: compiler specialization must treat draft and target as separate capture regions with shared KV invariants \citep{kwon2023vllm,patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Merging draft control flow into main decode capture breaks graph legality \citep{nvidia2024cudagraphs,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Quantized decode (INT4/FP8 weights, FP16 activations) changes fusion legality: specialized passes must preserve dequant residency on-chip across attention--MLP seams \citep{semianalysis2024tma,nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,amd2024xdna}. Profile bytes/token with quantization enabled---kernel names change but counters do not \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Industrial runbooks tie LLM/MoE compile artifacts to model card revisions: tokenizer, rope, expert count, and paging policy must hash into schedule fingerprints \citep{kwon2023vllm,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Silent drift between training and serving configs is a top source of ``compiler regression'' tickets that are actually model mismatches \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

\textbf{LLM decode compile} starts when the compiler owns a \emph{decoder slice}---attention, MLP, norms, rotary, residual---exported at BS${=}1$ with explicit KV shapes \citep{kwon2023vllm,patel2025llminference,dao2022flashattention,yirage2026,chen2020compilerbasedhardw}. Training-shaped graphs (large batch, no paging) mislead tile choices: shared-memory budgets that fit prefill often spill KV on decode \citep{memoryfloor2026,taxbreak2026,semianalysis2024tma,dlbook}.

Compiler passes should tag operators with \emph{decode role}: KV producer, KV consumer, stateless compute \citep{dao2022flashattention,dao2023flashdecoding,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,amd2024xdna,amd2024aie,dlbook,liu2024deepseekv3,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper}. Fusion without role tags reunifies nodes that frameworks intentionally split for paging---breaking vLLM/SGLang invariants \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

FlashAttention and Flash-Decoding shrink attention bytes but MLP/residual boundaries often remain separate kernels unless MegaKernel plans reunify them \citep{dao2022flashattention,dao2023flashdecoding,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,nvidia2024cudagraphs,nvidia2022hopper}. Decode compile therefore tracks \emph{whole-step} bytes/token, not attention microbenchs alone \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Export boundaries matter: FX/ONNX dumps often freeze prefill shapes \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Specialized LLM pipelines inject dynamic seq-len and paging metadata before lowering \citep{kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Without metadata, ``successful'' compile produces undeployable schedules \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Rotary, norm, and sampling tails are easy to ignore in decode compile yet dominate launches/token on small models \citep{taxbreak2026,patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Include them in fusion plans and profile whole-step counters \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

\paragraph{Multi-hardware delta.} Hopper decode favors TMA-friendly KV tiles; XDNA requires static KV slot assignment; CPU decode fights LLC capacity---identical transformer block, different fusion limits \citep{semianalysis2024tma,amd2024xdna,amd2024aie,dlbook,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\begin{example}
Decode cell (BS${=}1$, ctx${=}2048$): compiler fusion that materializes full KV tensors between layers doubles bytes/token even if attention kernel is optimal \citep{memoryfloor2026,dao2022flashattention,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.
\end{example}

\textbf{MoE scheduling} is where decode compilation meets \emph{dynamic routing} \citep{liu2024deepseekv3,taxbreak2026,patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw}. The production surprise is not expert GEMM peak FLOPs---it is launch storms when every token activates a different expert subset without batched grouping \citep{taxbreak2026,liu2024deepseekv3,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Compiler specialization options:
\begin{enumerate}
\item \textbf{Compile expert templates once} per shape bucket; runtime selects indices \citep{liu2024deepseekv3,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Fuse router + top-$k$ + expert prologue} when graph capture legal; isolate when control flow diverges \citep{nvidia2024cudagraphs,kwon2023vllm,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Expert parallelism passes} that align with DeepEP-style dispatch when disaggregated \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\end{enumerate}

OLMoE-scale launch counts (TaxBreak) warn that MoE decode without fusion remains orchestration-bound even when experts are ``fast'' \citep{taxbreak2026,liu2024deepseekv3,patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Profile launches/token \emph{per layer} with routing entropy logged---mean ms/token hides expert imbalance tails \citep{patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\paragraph{Review gate.} MoE compile merges require expert activation histograms from production-like traffic, not uniform random routing \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Shared-expert and routed-expert splits (DeepSeek-class) need distinct compile templates: shared experts fuse like dense MLP; routed experts batch by token groups \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Collapsing both into one schedule loses routing visibility \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

When expert counts exceed on-chip weight residency, compilation must coordinate with runtime prefetch---bytes/token includes HBM traffic for cold experts \citep{liu2024deepseekv3,semianalysis2024tma,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna,amd2024xdna}. MoE scheduling is a memory problem disguised as a routing problem \citep{patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

\section{kv hardware}
\label{sec:ch23_kv_hardware}

\textbf{KV hardware adaptation} splits responsibility: frameworks page and schedule KV blocks; compilers choose layout, fusion around KV edges, and on-chip residency for attention/MLP seams \citep{kwon2023vllm,dao2022flashattention,dao2023flashdecoding,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024xdna,amd2024aie,dlbook,semianalysis2024tma,nvidia2024cudagraphs,nvidia2022hopper}. Illegal fusion that assumes dense KV tensors breaks PagedAttention invariants \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Hardware-specific KV tactics:
\begin{itemize}
\item \textbf{GPU:} block-sparse attention tiles, TMA-friendly KV loads, graph-safe KV pointer stability \citep{nvidia2022hopper,semianalysis2024tma,dao2022flashattention,nvidia2024cudagraphs,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2023flashdecoding,amd2024xdna}.
\item \textbf{CPU:} cache-line blocked KV, NUMA-aware pinning for long context \citep{dlbook,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,yirage2026,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.
\item \textbf{NPU/XDNA:} static KV slot tables, bounded context buckets compiled upfront \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma}.
\end{itemize}

Context-length buckets matter: a schedule legal at ctx${=}2048$ may spill at ctx${=}8192$ when KV exceeds shared memory or LLC \citep{memoryfloor2026,dao2022flashattention,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna,amd2024xdna}. Compile separate artifacts per bucket rather than one dynamic kernel that silently allocates \citep{nvidia2024cudagraphs,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Multi-query and grouped-query attention change KV head layout; KV hardware passes must read head counts from model metadata, not hard-coded MHA \citep{kwon2023vllm,dao2022flashattention,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Wrong layout passes compile yet corrupt generation \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

Prefix caching and long system prompts shift KV working set: compilers should not assume empty cache at step zero \citep{kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Profile cells include warm prefix lengths representative of production \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,liu2024deepseekv3,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.

\begin{example}
Paged KV + fused MLP: compiler must treat KV pointers as opaque handles through attention--MLP seam---materializing paged KV to dense breaks the framework contract \citep{kwon2023vllm,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}.
\end{example}

\section{hetero moe}
\label{sec:ch23_hetero_moe}

\textbf{Heterogeneous MoE load balancing} spans \emph{compile-time} expert placement and \emph{runtime} routing policy \citep{liu2024deepseekv3,patel2025llminference,chen2020compilerbasedhardw,yirage2026,amd2024xdna,amd2024aie,dlbook,kwon2023vllm,taxbreak2026,memoryfloor2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Expert parallelism across GPU tiers (prefill pool vs decode pool) fails when compiled expert weights assume uniform NVLink topology \citep{patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Compiler stacks should emit \emph{expert placement metadata}: which experts are colocated, which tiers use INT4/FP8 kernels, and which backends are legal per chip \citep{liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Runtime load balancers (eplb-class policies in open DeepSeek infra) consume that metadata; compilers should not hard-code routing \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

On CPU/NPU fallback paths, subset experts may run on host while GPU serves hot experts---compilation must quarantine backends per expert, not share one schedule \citep{amd2024xdna,amd2024aie,dlbook,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,taxbreak2026,memoryfloor2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Hetero MoE gates in Chapter~22 apply per tier: bytes/token and launches/token on each participating SKU \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Fleet skew---one expert dominating activations---is a \emph{routing} problem first, but compilation can mitigate via duplicated hot-expert kernels on multiple chips when policy allows \citep{liu2024deepseekv3,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Measure load imbalance with production histograms; synthetic uniform routing hides tail latency \citep{patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,taxbreak2026,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Chapter~24 extends heterogeneity to cluster deployment; Chapter~28 clarifies framework vs compiler KV ownership \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Specialized LLM/MoE compilation succeeds when it respects those boundaries \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\section{Key Takeaways}
\label{sec:ch23_key_takeaways}

\begin{itemize}
\item \textbf{Own the decoder slice at batch-1.}
LLM decode compilation begins when the compiler owns a whole decoder slice---attention, MLP, norms, rotary, residual---exported at batch-1 with explicit KV shapes, and gates it on whole-step bytes/token rather than per-operator FLOPs \citep{kwon2023vllm,patel2025llminference}. Optimizing the slice as a unit while respecting the framework's paging is what keeps the attention-MLP seam resident \citep{dao2022flashattention,memoryfloor2026}.

\item \textbf{MoE compilation meets dynamic routing.}
The hard part of MoE scheduling is data-dependent routing: template the experts so they share a kernel, fuse the router prologue only when legal, and profile launches/token under realistic routing entropy rather than a uniform-load proxy \citep{liu2024deepseekv3,taxbreak2026}. A schedule that looks fine on balanced routing can thrash when real traffic skews load across experts \citep{patel2025llminference,kwon2023vllm}.

\item \textbf{KV is a split responsibility.}
Frameworks page and schedule KV blocks; the compiler chooses layout, fuses around the KV edges, and secures on-chip residency for the attention/MLP seam---and that split is realized differently per GPU, CPU, and NPU \citep{kwon2023vllm,amd2024xdna}. Treating KV layout as a compiler transform rather than a runtime afterthought is what makes the paged attention fast instead of merely correct \citep{dao2022flashattention,semianalysis2024tma}.

\item \textbf{Heterogeneous MoE spans compile-time and runtime.}
Load balancing splits across compile-time expert placement and runtime routing policy, so the compiler emits placement metadata and per-tier gates while the runtime routes, and backends are quarantined across fallback paths \citep{liu2024deepseekv3,patel2025llminference}. Without per-tier gates a placement tuned for one accelerator silently regresses another \citep{yirage2026,amd2024aie}.

\item \textbf{MoE and KV specialization still defer to the decode counters.}
However specialized the compilation, the acceptance metric is unchanged: bytes/token and launches/token at the batch-1 decode operating point, not peak FLOPs \citep{patel2025llminference,taxbreak2026}. MoE's launch density and KV's bandwidth pressure are precisely the terms those counters expose \citep{memoryfloor2026,dlbook}.
\end{itemize}

\section{Conclusion}
\label{sec:ch23_conclusion}

LLM and MoE specialized compilation narrows generic compiler stacks to decode-shaped graphs, routed experts, and KV contracts frameworks own \citep{kwon2023vllm,liu2024deepseekv3,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Gate merges on bytes/token, launches/token, and paging legality---not expert GEMM peaks alone \citep{taxbreak2026,patel2025llminference,liu2024deepseekv3,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,semianalysis2024tma,amd2024xdna}. Chapter~24 carries heterogeneity to cluster scale; Chapter~25 automates search inside the same invariants \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
