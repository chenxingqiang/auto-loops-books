Teams hit \textbf{LLM decode compile} when LLM and MoE Specialized Compilation looks correct in isolation yet decode telemetry disagrees---usually because LLM was planned without decode residency on the SKU you ship \citep{patel2025llminference,dlbook}.

Hardware truth: GPUs bound LLM with shared memory/TMA; CPUs with cache/NUMA; NPUs with static tile SRAM and DMA chains \citep{nvidia2022hopper,amd2024xdna,dlbook}. Decode at batch-1 stays bandwidth- and orchestration-limited---thin matmuls rarely dominate \citep{memoryfloor2026,taxbreak2026}.

Compiler/kernel fix: predicate fusion on LLM/decode legality before codegen; fork passes per target \citep{chen2020compilerbasedhardw,yirage2026}. YiRage encodes iron rules from Chapter~2 as checkable IR metadata \citep{kwon2023vllm,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper may legalize LLM with TMA double-buffer; XDNA requires static tile assignment; CPU needs cache-blocked decode---identical math, different bytes/token \citep{semianalysis2024tma,amd2024xdna,dlbook}.

The production surprise at \textbf{moe scheduling} is not peak FLOPs; it is illegal tier placement: moe spilling to HBM/DDR while expert still pays launch or DMA tax every token \citep{liu2024deepseekv3,patel2025llminference}.

\textbf{LLM decode compile} exposes a cross-hardware fracture: the same fused subgraph legal on Hopper can be rejected outright on XDNA or CPU cache budgets.

When LLM decode compile disagrees with microbenchmarks, trust fleet telemetry: fix orchestration and tier placement first, then revisit MMA tile shapes \citep{patel2025llminference,dlbook}.

\paragraph{Review gate.} Before merging compiler changes affecting LLM decode compile, attach roofline or NPU buffer accounting showing LLM residency fits on-chip for the target SKU \citep{patel2025llminference,dlbook}.

\paragraph{Worked contrast.} A Hopper decode cell may legalize LLM with TMA-backed double buffering while an XDNA cell requires the same math expressed as static tile assignments---identical FLOPs, different bytes/token because DDR traffic differs by an order of magnitude \citep{patel2025llminference,dlbook}.

The usual LLM and MoE Specialized Compilation failure at \textbf{moe scheduling} is importing a CUDA mental model and expecting runtime scheduling to hide illegal buffer lifetimes.

Hardware constraint: moe and expert must be co-designed. On GPUs that means shared-memory/TMA budgets and warp-grain barriers; on XDNA/AIE it means static tile buffers, DMA chains, and carrier slots with compile-time lifetimes; on CPUs it means L2/L3 blocking and NUMA-local KV placement \citep{liu2024deepseekv3,patel2025llminference}. Decode at batch-1 keeps arithmetic intensity on the memory-bandwidth slope---micro-optimizing isolated GEMMs rarely moves p99 latency.

Compiler takeaway for moe scheduling: lower the same decoder IR, but predicate passes on target residency tables---never ship a GPU-only schedule to NPU fleets \citep{liu2024deepseekv3,patel2025llminference}. Success metrics: bytes/token, launches/token, and p99 decode latency; compare GPU, CPU, and NPU cells on identical context lengths.
