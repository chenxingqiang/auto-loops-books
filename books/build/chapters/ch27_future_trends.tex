TaxBreak dense Llama-3.2-1B averages 847.5 kernel launches per output token on H100; OLMoE-1B/7B averages 9,305 \citep{taxbreak2026}. Memory-Floor CUDA Graphs on Qwen-2.5-7B (BS${=}1$, ctx${=}2048$) yields $1.259\times$ on H100 by removing host overhead---yet tomorrow's silicon adds new memory tiers and routing paths that obsolete today's fusion maps \citep{memoryfloor2026,nvidia2024cudagraphs,semianalysis2024tma,amd2024xdna,amd2024aie}. Trends matter only when they name \emph{which counter moves} and \emph{which gate breaks} \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Four trajectories close Part~VII: \textbf{hardware--software co-design}, \textbf{new architecture adaptation}, \textbf{edge--cloud unified compilation}, and \textbf{autonomous kernel generation with legality gates} \citep{chen2020compilerbasedhardw,yirage2026,amd2024xdna,amd2024aie,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,dlbook}. Part~VIII (Chapters~28--30) implements the three-party contract among inference frameworks, YiRage runtime, and compiler co-design that these trends converge toward \citep{yirage2026,kwon2023vllm,patel2025llminference,chen2020compilerbasedhardw,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Trend chapters fail operationally when teams adopt silicon before IR contracts exist---the same failure mode as shipping eager fallbacks after graph capture \citep{nvidia2024cudagraphs,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Each section below names a trend, the counter it should move, and the gate that must not break \citep{patel2025llminference,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

Anti-patterns for trend adoption: chasing peak TFLOPs without decode cells; porting GEMM first; treating autonomous kernels as exempt from legality; merging edge and cloud fingerprints \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. The fix is always the same methodology from Chapter~22---profile, gate, ship \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

\textbf{Hardware--software co-design} stops being a slogan when ISA and memory hierarchy changes ship with \emph{compiler metadata contracts}---what can be fused, what must stay paged, which tiers exist \citep{chen2020compilerbasedhardw,yirage2026,semianalysis2024tma,amd2024xdna,amd2024aie,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Hopper-class TMA and warp specialization pushed fusion boundaries; the next co-design wave encodes those boundaries in IR so backends do not rediscover them per vendor \citep{nvidia2022hopper,semianalysis2024tma,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

Co-design for \emph{decode} differs from training: batch-1 residency, paging, and graph capture dominate; co-design documents must lead with bytes/token impact, not peak TFLOPs \citep{patel2025llminference,memoryfloor2026,taxbreak2026,dao2022flashattention,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. YiRage-style iron rules (Chapter~2) foreshadow vendor-neutral co-design: checkable predicates shipped alongside silicon \citep{yirage2026,chen2020compilerbasedhardw,kwon2023vllm,patel2025llminference,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Vendor roadmaps will keep adding memory tiers, expert routing, and capture modes---co-design is the process of turning those into compiler-visible facts before backends guess \citep{semianalysis2024tma,liu2024deepseekv3,nvidia2024cudagraphs,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

\paragraph{Multi-hardware delta.} GPU co-design chases bandwidth and launch orchestration; NPU co-design chases static DMA depth; CPU co-design chases cache residency---shared methodology, incomparable constraints \citep{amd2024xdna,amd2024aie,dlbook,chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\textbf{New architecture adaptation} is the recurring tax every AI compiler team pays---MLIR dialects, backend forks, and HardwareRegistry entries---when silicon generations turn over \citep{chen2020compilerbasedhardw,yirage2026,amd2024xdna,amd2024aie,semianalysis2024tma,nvidia2022hopper,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,dlbook}. The anti-pattern is ``port GEMM first, decode later''; decode-first ports expose paging and graph legality early \citep{patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Trend: modular adapter passes per architecture generation, each gated by the same decode cells from Chapter~22 \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Architecture trend lines without regression suites become marketing, not engineering \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,taxbreak2026,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

MoE-heavy architectures (DeepSeek-class) accelerate the trend: expert routing and disaggregated pools become first-class IR concepts, not runtime hacks \citep{liu2024deepseekv3,patel2025llminference,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Compiler teams should budget one adapter generation per silicon refresh---trying to stretch a Hopper-only fusion map across Blackwell and XDNA without adapter passes guarantees launch explosions \citep{taxbreak2026,semianalysis2024tma,amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,dlbook,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. \paragraph{Review gate.} New architecture support merges only when bytes/token and launches/token match or beat prior generation on agreed buckets \citep{memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

\section{edge cloud}
\label{sec:ch27_edge_cloud}

\textbf{Edge--cloud unified compilation} targets one model artifact with tier-specific fingerprints---edge NPU static schedules, cloud GPU graph+capture, CPU fallback---without pretending one schedule fits all \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. The edge trend is longer context on device; the cloud trend is fleet goodput---unified IR must express both without lowest-common-denominator fusion \citep{patel2025llminference,kwon2023vllm,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Adaptive compilation at runtime (detect chip, select backend) is the operational face of this trend---HardwareRegistry patterns from YiRage generalize to edge fleets \citep{yirage2026,chen2020compilerbasedhardw,amd2024xdna,amd2024aie,kwon2023vllm,patel2025llminference,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Cloud offload decisions (which layers run edge vs cloud) belong in policy metadata compiled into the same IR family \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

Privacy and bandwidth drive edge trends, but \textbf{decode counters} still arbitrate: edge wins only if ms/token and bytes/token beat cloud RTT-inclusive baselines for the target workload \citep{patel2025llminference,dlbook,memoryfloor2026,taxbreak2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Unified edge--cloud compile pipelines version fingerprints per tier and forbid silent cross-tier promotion \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

Fleet operators increasingly run hybrid graphs: prefill on cloud GPU, decode on edge NPU, or the reverse for privacy-sensitive prompts \citep{amd2024xdna,amd2024aie,kwon2023vllm,patel2025llminference,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,memoryfloor2026,taxbreak2026,dlbook,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Unified compilation must emit \emph{split fingerprints}---not one schedule pretending both tiers share memory \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

\begin{example}
Same LLM runs on XDNA edge and Hopper cloud: two fingerprint namespaces, one IR export, shared decode-cell methodology \citep{amd2024xdna,yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.
\end{example}

\section{autonomous kernels}
\label{sec:ch27_autonomous_kernels}

\textbf{Autonomous kernel generation}---LLM-assisted or search-driven---is the most hyped trend and the highest risk \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Chapter~25 placed RL and auto-optimization under decode rewards; the trend extends toward models proposing fusion patterns or kernel text \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.

Non-negotiable guardrails for autonomous kernels:
\begin{enumerate}
\item \textbf{Legality oracle before profile} --- same as human-authored schedules \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.
\item \textbf{Decode-cell reward} --- bytes/token, launches/token, ms/token; no GEMM-only scores \citep{memoryfloor2026,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}.
\item \textbf{Human prior + fingerprint rollback} --- autonomy subordinate to production gates (Chapter~26) \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.
\end{enumerate}

Autonomous \emph{kernel} text without IR integration repeats CUDA string injection failures---trend winners compile through the same multi-backend stacks Part~VI built \citep{chen2020compilerbasedhardw,yirage2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. The autonomous trend succeeds when it accelerates search under existing gates, not when it bypasses them \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

Search budgets for autonomous kernels should cap candidate evaluations per merge request---unbounded search is how illegal schedules reach main \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Chapter~25's RL autotune patterns extend here: reward decode goodput, penalize legality violations, require human-readable diffs \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.

\section{Key Takeaways}
\label{sec:ch27_key_takeaways}

\begin{itemize}
\item \textbf{Co-design ships as compiler metadata contracts.}
Hardware--software co-design stops being a slogan when an ISA or memory-hierarchy change arrives with a contract the compiler can read---what may be fused, what must stay paged, which tiers exist \citep{yirage2026,semianalysis2024tma}. Encoding silicon capabilities as IR legality and proving each on decode cells is what turns a new feature into a usable, checkable optimization rather than a press release \citep{chen2020compilerbasedhardw,patel2025llminference}.

\item \textbf{New architectures are a recurring, decode-first tax.}
Every silicon generation forces adapter passes, backend forks, and hardware-registry entries, and the discipline that keeps the cost bounded is decode-first porting with architecture-specific regression \citep{chen2020compilerbasedhardw,yirage2026}. Bringing a new target up on prefill alone hides exactly the residency and launch gaps that batch-1 decode will later expose \citep{memoryfloor2026,taxbreak2026}.

\item \textbf{Edge--cloud means one artifact, tier-specific fingerprints.}
Unified compilation targets a single model with per-tier fingerprints---edge NPU static schedules, cloud GPU graph capture, CPU fallback---without pretending one schedule fits all \citep{amd2024xdna,amd2024aie}. Adaptive backend selection at runtime with no cross-tier promotion is what prevents a schedule tuned for one tier from silently regressing another \citep{patel2025llminference,kwon2023vllm}.

\item \textbf{Autonomous kernels need legality, reward, and rollback.}
LLM-assisted and search-driven kernel generation is the most hyped and highest-risk trend, because a generated kernel can be fast and wrong \citep{chen2020compilerbasedhardw,yirage2026}. It is safe only inside a frame that verifies legality, rewards decode goodput rather than microbench latency, and can roll back---treating the generator as a hint source, not a bypass of the residency contract \citep{memoryfloor2026,patel2025llminference}.

\item \textbf{Part VIII makes these trends deployable.}
The co-design, portability, and automation trends above are not abstractions; framework, runtime, and compiler co-design is where they become a shipped serving stack \citep{yirage2026,kwon2023vllm}. The next part descends from trend to implementation, carrying the same decode-goodput yardstick \citep{patel2025llminference,taxbreak2026}.
\end{itemize}

\section{Conclusion}
\label{sec:ch27_conclusion}

Future trends in AI compilers converge on one discipline: faster silicon and smarter automation must still answer to decode goodput \citep{patel2025llminference,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Co-design, new architectures, edge--cloud unity, and autonomous kernels are directions---not excuses to drop bytes/token gates \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference,memoryfloor2026,taxbreak2026,kwon2023vllm,liu2024deepseekv3,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}. Part~VIII opens with inference frameworks (Chapter~28), YiRage PersistentKernel runtime (Chapter~29), and three-party co-design (Chapter~30) \citep{yirage2026,kwon2023vllm,patel2025llminference,chen2020compilerbasedhardw,liu2024deepseekv3,memoryfloor2026,taxbreak2026,amd2024aie,dlbook,semianalysis2024tma,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper}. Gate merges on buffer accounting and reproducible decode cells---the trend line starts where Chapter~1 started \citep{semianalysis2024tma,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,taxbreak2026,amd2024aie,dlbook,amd2024xdna,dao2022flashattention,dao2023flashdecoding,nvidia2024cudagraphs,nvidia2022hopper,amd2024xdna}.
