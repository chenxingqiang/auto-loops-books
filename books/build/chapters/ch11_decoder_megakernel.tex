\chapter{Full Decoder MegaKernel Implementation}
\label{chap:ch11}
\typeout{START_CHAPTER "ch11" \theabspage}

FlashAttention removed the attention $\mathcal{O}(N^2)$ memory cliff; CUDA Graphs removed a slice of host overhead---yet batch-1 decode on Llama-class models still issues hundreds of discrete GPU kernels \emph{per output token} \citep{taxbreak2026,memoryfloor2026,dao2022flashattention}. The remaining pain is not ``one slow matmul'' but \emph{layer fragmentation}: QKV projection, online softmax attention, RMS norm, and MLP each export full-width activations to HBM between launches \citep{nvidia2024cudagraphs,kwon2023vllm}. A \textbf{decoder MegaKernel} fuses those stages into one residency-aware schedule---one graph capture on GPU, one static dataflow on XDNA---so bytes/token and launches/token fall together \citep{chen2020compilerbasedhardw,yirage2026}.

Chapters~4--10 built the ingredients: TMA double-buffering, online softmax carriers, sync hierarchy, execution-mode trade-offs. This chapter assembles them into a \emph{full-layer} implementation narrative: structure, fused QKV--attention--MLP boundaries, a source-level control-flow walkthrough, and benchmark methodology that separates real fusion wins from graph-capture illusions \citep{semianalysis2024tma,amd2024xdna,patel2025llminference}. The mistake to avoid is declaring victory when CUDA Graphs replay the same HBM traffic with fewer host calls---Memory-Floor's $1.259\times$ on Qwen-2.5-7B (BS${=}1$, ctx${=}2048$) proves orchestration relief without residency relief is a partial fix \citep{memoryfloor2026}.

TaxBreak dense Llama-3.2-1B averages 847.5 kernel launches per output token on H100; OLMoE-1B/7B averages 9,305 \citep{taxbreak2026}. A MegaKernel target is not zero launches---it is \emph{O(1) launches per decoder layer per token} (or per captured graph step) with intermediates resident in shared memory, registers, or tile SRAM \citep{nvidia2022hopper,amd2024aie}. YiRage lowers the same decoder IR to CUDA MegaKernels, CPU blocked schedules, and XDNA static graphs; hand-written stacks should treat unfused baselines as regression tests, not shipping defaults \citep{yirage2026,liu2024deepseekv3}.

\paragraph{Scope of ``full decoder.''} This chapter focuses on \emph{one transformer decoder layer} fused end-to-end---the atomic unit repeated $L$ times in depth. Stacking layers inside a single MegaKernel is a further optimization (amortizing launch and pipeline fill) but explodes register and shared-memory pressure; production systems often MegaKernel per layer and capture an $L$-step graph \citep{nvidia2024cudagraphs,taxbreak2026}. The engineering question is identical: where do activations touch HBM between sub-stages \citep{memoryfloor2026,patel2025llminference}? DeepSeek-V3-style wide MoE decoders may require \emph{expert-local} MegaKernels rather than one monolith---routing and all-to-all remain outside the fused region \citep{liu2024deepseekv3,taxbreak2026}.

\paragraph{Relationship to prior chapters.} Chapter~4 supplies TMA and shared-memory ceilings; Chapter~5 translates residency to XDNA tile rules; Chapters~8--9 supply double-buffer and sync patterns; Chapter~10 supplies online softmax carriers that must survive into MLP in this chapter \citep{semianalysis2024tma,amd2024aie,dao2022flashattention}. If any ingredient is missing, teams build ``MegaKernels'' that are graph-capture wrappers around unchanged HBM traffic \citep{memoryfloor2026}.

\paragraph{Industrial motivation.} Serving stacks that ship 847+ launches/token pay a fixed tax on every interactive token \citep{taxbreak2026}. At 4.7\,$\mu$s null-launch floor on H100, launch path alone approaches tens of milliseconds per token for MoE models before any math \citep{taxbreak2026,memoryfloor2026}. MegaKernels attack both launch count \emph{and} the bytes moved per launch---the only durable fix when arithmetic intensity is already on the memory roofline \citep{dlbook,patel2025llminference}. Cloud fleets also pay PCIe and CPU orchestration energy; edge Ryzen AI fleets pay DDR contention with CPU graphics \citep{amd2024xdna,amd2024aie}. The same fused schedule story applies---only residency tables change \citep{yirage2026,chen2020compilerbasedhardw}.

\paragraph{What this chapter is not.} Not a PyTorch tutorial \citep{dlbook}. Not a FlashAttention paper recap---Chapter~10 covered carriers \citep{dao2022flashattention}. Not a CUDA Graphs manual---orchestration-only wins are labeled explicitly \citep{memoryfloor2026,nvidia2024cudagraphs}. Not a promise that every model fuses on every SKU---legality failures are expected outputs \citep{amd2024aie,yirage2026}. The deliverable is an implementable fused-layer contract with measurable decode metrics \citep{taxbreak2026,patel2025llminference,chen2020compilerbasedhardw}.

\paragraph{Reader prerequisites.} Assume familiarity with decode vs prefill, paged KV, online softmax, and TMA pipelining from Chapters~1--10 \citep{kwon2023vllm,dao2022flashattention,semianalysis2024tma,memoryfloor2026}. Jumping here from a generic CUDA course produces ``pretty kernels'' that still export activations to HBM \citep{taxbreak2026,yirage2026}.

\paragraph{Notation.} $d$: hidden size; $d_{\mathrm{ff}}$: MLP expansion; BS: batch size; ctx: context length; $L$: layer count \citep{dlbook,patel2025llminference}. ``Layer'' means one transformer decoder block unless stated \citep{liu2024deepseekv3,kwon2023vllm}. ``Token'' means one autoregressive decode step \citep{dao2023flashdecoding,taxbreak2026}. ``Launch'' means one host-visible kernel or graph replay step \citep{nvidia2024cudagraphs,memoryfloor2026}. Align vocabulary in cross-team docs to avoid debating past each other \citep{yirage2026,chen2020compilerbasedhardw,amd2024aie,amd2024xdna,semianalysis2024tma,dao2022flashattention,liu2024deepseekv3}.

\section{MegaKernel Structure}
\label{sec:ch11_megakernel_structure}

\paragraph{Why one kernel still matters after FlashAttention.} Teams assume FlashAttention ``fixed decode.'' It fixed attention's quadratic scratch memory---not the surrounding launch storm. Unfused decoder stacks still separate layer norm, QKV linear, attention, projection, second norm, and three MLP matrices into distinct kernels \citep{dao2022flashattention,dao2023flashdecoding}. Each boundary writes a $d$-model vector to HBM and reads it back---at batch-1 that is pure bandwidth tax on a thin arithmetic intensity slope \citep{dlbook,memoryfloor2026}. MegaKernel structure means redesigning the \emph{control flow} so those stages share one on-chip residency plan for the duration of a decoder step \citep{chen2020compilerbasedhardw,taxbreak2026}.

Figure~\ref{fig:megakernel_structure_pipeline} is the chapter spine. Unlike a prefill megamatmul, decode MegaKernels must interleave \textbf{KV cache read--append} with fused math every token \citep{kwon2023vllm,patel2025llminference}. The host submits one graph (GPU) or one static runner invocation (XDNA); the device loop advances sequence position, bumps KV pointers, and keeps softmax statistics in carrier registers described in Chapter~10 \citep{nvidia2024cudagraphs,amd2024xdna}.

\paragraph{Pipeline stage notes (Figure~\ref{fig:megakernel_structure_pipeline}).}
\textbf{Input hidden state} arrives as a single token vector (batch-1) or narrow batch slice \citep{patel2025llminference,dlbook}.
It should enter shared memory once---not remain in HBM while QKV loads weights \citep{semianalysis2024tma,memoryfloor2026}.
\textbf{On-chip QKV + carriers} holds projected Q/K/V fragments plus online softmax state \citep{dao2022flashattention}.
Peak pressure occurs here when $d$ is large or head count is high \citep{liu2024deepseekv3,nvidia2022hopper}.
\textbf{Fused attn + MLP tile} is the longest compute phase: KV blocks stream from HBM while MMA pipelines gate/up/down \citep{dao2023flashdecoding,taxbreak2026}.
TMA overlap from Chapter~8 is mandatory---else MegaKernel devolves to sequential micro-kernels inside one launch \citep{semianalysis2024tma,chen2020compilerbasedhardw}.
\textbf{Output + KV append} writes the next hidden state and appends K/V without host callbacks \citep{kwon2023vllm,nvidia2024cudagraphs}.
On XDNA, append is a DMA store into DDR-resident paged pools \citep{amd2024xdna,amd2024aie}.
Breaking the pipeline at any boundary exports a full activation vector---defeating the MegaKernel purpose \citep{memoryfloor2026,taxbreak2026,yirage2026}.

\paragraph{Host responsibilities per token.} Ideal host work: increment sequence counters, update sampling state, enqueue next graph replay \citep{nvidia2024cudagraphs,kwon2023vllm}. Anything heavier (allocation, dtype conversion, tensor contiguity fixes) belongs in session setup---not the hot loop \citep{memoryfloor2026,taxbreak2026}. Python GIL interactions still matter if the host thread prepares each step synchronously \citep{patel2025llminference,dlbook}.

\paragraph{Multi-layer scheduling.} Repeating Figure~\ref{fig:megakernel_structure_pipeline} $L$ times per token can mean $L$ graph replays or one persistent kernel traversing layers \citep{liu2024deepseekv3,nvidia2024cudagraphs}. The per-layer MegaKernel body is identical modulo weight pointers \citep{yirage2026,chen2020compilerbasedhardw}. Benchmarks must state which scheduling variant they use---$L$ replays add host overhead even when each layer is fused \citep{taxbreak2026,memoryfloor2026}.

Table~\ref{tab:megakernel_structure} explains why MegaKernel engineering is a \emph{systems} task: the limiting resource is simultaneous buffer residency, not FLOPs \citep{semianalysis2024tma,nvidia2022hopper}. Hopper may legalize the plan with TMA-backed double buffers and warp-specialized stages \citep{semianalysis2024tma}. XDNA requires every temporary---including online softmax carriers---to fit tile SRAM assignments before compile \citep{amd2024aie,yirage2026}. CPU decode uses the same logical structure with L2-blocked megafunctions; launches/token is less visible but bytes/token still punishes unfused DRAM traffic \citep{dlbook,patel2025llminference}.

\paragraph{Persistent vs.\ captured graphs.} Two GPU patterns dominate production:
\begin{enumerate}
\item \textbf{CUDA Graphs capture} of an unfused or partially fused op sequence---reduces $\Delta FT$ and $\Delta KT$ from Chapter~1 but leaves HBM traffic unchanged \citep{memoryfloor2026,nvidia2024cudagraphs}.
\item \textbf{True MegaKernel fusion}---single kernel body or minimal graph nodes with shared-memory staging across QKV, attention, and MLP \citep{chen2020compilerbasedhardw,taxbreak2026}.
\end{enumerate}
Reviewers must distinguish them in benchmark tables; mixing the two inflates perceived ``fusion speedups'' \citep{patel2025llminference,memoryfloor2026}.

\begin{example}
Llama-3.2-1B decode layer (BS${=}1$, ctx${=}2048$): unfused stack materializes six $d$-model vectors to HBM per layer per token; a MegaKernel keeps QKV fragments, softmax carriers, and MLP gate/up tiles in shared memory until the residual add completes---bytes/token drops with launches/token even when MMA tile shapes are unchanged \citep{taxbreak2026,dao2022flashattention}.
\end{example}

\paragraph{Multi-hardware delta.} GPU MegaKernels exploit shared memory + TMA; XDNA maps the same DAG to spatial tiles with DMA edges; CPU maps to fused BLAS-like blocks with explicit cache blocking \citep{amd2024xdna,dlbook,yirage2026}. Identical math, different legality predicates---never ship a GPU-only residency table to Ryzen AI without recompile \citep{amd2024aie,liu2024deepseekv3}.

\paragraph{Review gate.} Merge requires a buffer accounting diagram: peak simultaneous tensors during one decode step must fit target SRAM budgets \citep{yirage2026,memoryfloor2026}. Failed legality is a release blocker, not a ``fallback to eager'' ticket \citep{chen2020compilerbasedhardw}.

\paragraph{Interaction with disaggregated serving.} Prefill/decode disaggregation moves pools independently \citep{patel2025llminference,liu2024deepseekv3}. Decode pools benefit most from MegaKernels---prefill pools may keep separate wide-GEMM schedules \citep{dao2023flashdecoding,kwon2023vllm}. Do not force one fusion policy across disaggregated roles \citep{taxbreak2026,memoryfloor2026}. Benchmark decode pools with batch-1/interactive distributions, not prefill throughput alone \citep{patel2025llminference,dao2022flashattention}.

\paragraph{Warp specialization and pipeline stages.} Hopper-class MegaKernels often assign warps to roles---TMA load, MMA compute, epilogue---mirroring CUTLASS-style persistent kernels \citep{semianalysis2024tma,nvidia2022hopper}. Decode differs from GEMM persistence: the KV loop length grows each token, so the inner loop trip count changes while the kernel body stays captured \citep{kwon2023vllm,dao2023flashdecoding}. Implementations pad to max context bucket or specialize kernels per bucket; mixing buckets without re-capture violates graph legality \citep{nvidia2024cudagraphs,yirage2026}.

\paragraph{Register and shared-memory budgeting.} A back-of-envelope for Llama-3.2-1B ($d{=}2048$, $h{=}32$): QKV fragments, softmax carriers, and SwiGLU intermediates can exceed 200\,KB shared memory if naively allocated---above many SM limits when combined with MMA tiles \citep{nvidia2022hopper}. MegaKernel structure therefore \emph{sequences} sub-stages with reuse: gate/up may share buffers with Q staging after QKV completes \citep{chen2020compilerbasedhardw,taxbreak2026}. Compiler passes must solve this allocation problem; hand tuning without a lifetime graph does not scale across models \citep{yirage2026,liu2024deepseekv3}.

\paragraph{CPU and NPU structural analogues.} CPU MegaKernels appear as fused oneDNN/BLIS-style micro-kernels with $K$-dimension blocking; ``launch'' becomes function call \citep{dlbook}. XDNA expresses the same DAG as AIE tiles with fixed DMA descriptors---there is no \texttt{\_\_syncthreads}, only token completion on edges \citep{amd2024aie,amd2024xdna}. Fleet parity requires documenting which sub-stages remain unfused on each target and why \citep{patel2025llminference,chen2020compilerbasedhardw}.

\paragraph{Paged KV and MegaKernel entry.} vLLM-style paging changes \emph{where} K/V bytes live, not whether they must be read \citep{kwon2023vllm}. MegaKernel structure adds a block-table walk inside the attention loop---pointer-chasing in device code replaces scatter-gather launches \citep{patel2025llminference,memoryfloor2026}. Poor block sizing interacts with TMA tile shapes: 16-token blocks may underutilize memory bandwidth while 256-token blocks blow shared-memory staging \citep{semianalysis2024tma,dao2023flashdecoding}. Treat block size as a compile-time knob tied to context buckets \citep{yirage2026,kwon2023vllm}.

\paragraph{Quantization and MegaKernels.} INT8/FP8 weights shrink bytes/token but fuse epilogues with dequant scales \citep{nvidia2022hopper,patel2025llminference}. MegaKernels must carry scale vectors in registers or constant cache---another residency consumer \citep{semianalysis2024tma,chen2020compilerbasedhardw}. Mixed-precision decode that quantizes activations per token often breaks fusion unless scales are online in the same kernel \citep{liu2024deepseekv3,memoryfloor2026}.

\paragraph{Failure modes checklist.} Production MegaKernels fail in predictable ways: (1) shared-memory overflow at max context; (2) graph capture invalidation after driver upgrade; (3) silent fallback to eager on one shape bucket; (4) numerics drift vs reference beyond tolerance; (5) p99 regression from bank conflicts \citep{nvidia2024cudagraphs,memoryfloor2026,taxbreak2026}. Runbooks should map each symptom to residency vs orchestration root cause using bytes/token and launches/token splits \citep{patel2025llminference,yirage2026}.

\paragraph{Compiler ownership.} Hand-written MegaKernels do not scale across model zoo churn \citep{chen2020compilerbasedhardw,yirage2026}. The durable path is declarative decoder IR with target-specific legality checks---this chapter's walkthrough is the acceptance spec those compilers must emit \citep{liu2024deepseekv3,kwon2023vllm}. Kernel engineers retain tile tuning; compiler engineers own fusion boundaries and buffer lifetimes \citep{patel2025llminference,semianalysis2024tma}.

\paragraph{Layer replication and weight streaming.} A 32-layer model repeats the same MegaKernel body with different weight bases \citep{patel2025llminference,liu2024deepseekv3}. Weight streaming bandwidth scales with layer count---fusion reduces activation traffic but not parameter reads \citep{dlbook,memoryfloor2026}. Some stacks capture one graph per layer type (attention vs MoE) rather than per layer index \citep{taxbreak2026,nvidia2024cudagraphs}. The MegaKernel \emph{structure} is identical; pointer offsets change \citep{yirage2026,chen2020compilerbasedhardw}.

\paragraph{Safety and determinism.} Fused kernels complicate bitwise determinism for compliance workloads \citep{patel2025llminference,dlbook}. Document reduction order and FP associativity assumptions \citep{dao2022flashattention}. Deterministic mode may disable some fusion paths---trade bytes/token for reproducibility explicitly in \textbf{benchmark} tables \citep{memoryfloor2026,taxbreak2026}.

\section{QKV, Attention, and MLP Fusion}
\label{sec:ch11_qkv_attn_mlp}

\paragraph{The fusion boundary mistake.} Engineers fuse QKV with attention but leave RMS norm and MLP outside the MegaKernel---recovering ${\approx}30\%$ of launches while preserving the expensive mid-layer HBM writes \citep{taxbreak2026,dao2023flashdecoding}. Full-layer fusion treats \textbf{QKV}, \textbf{attention} (with online softmax), and \textbf{MLP} as one producer--consumer chain with a single shared-memory tile budget \citep{dao2022flashattention,semianalysis2024tma}.

\textbf{QKV projection} produces query, key, and value fragments for a single new token while reusing the hidden state tile already in shared memory \citep{nvidia2022hopper}. Fused kernels interleave K/V writes into the paged KV layout (vLLM-style) without a full-width staging buffer in HBM \citep{kwon2023vllm}. On XDNA, QKV maps to a fixed tile column; K/V DMA into DDR-resident cache lines described at compile time \citep{amd2024xdna,yirage2026}.

\textbf{Attention} uses online softmax carriers ($m$, $\ell$ statistics) from Chapter~10 so partial blocks update without materializing full attention maps \citep{dao2022flashattention}. The MegaKernel constraint: carriers live in registers or shared memory \emph{across} the boundary into MLP---spilling carriers to HBM defeats the fusion purpose \citep{memoryfloor2026}. Flash-Decoding-style splitting across KV heads must remain inside the same launch to avoid multiplying graph steps \citep{dao2023flashdecoding}.

\textbf{MLP} (SwiGLU in Llama-class models) is three matmuls with activations---historically three kernels. MegaKernel schedules gate/up in shared memory while down projection consumes the SiLU product without an HBM round trip \citep{patel2025llminference,liu2024deepseekv3}. MoE variants add expert routing; partial MegaKernels that fuse dense MLP but launch per expert reintroduce launch storms---TaxBreak's OLMoE figures (9,305 launches/token) are the warning label \citep{taxbreak2026,liu2024deepseekv3}.

Table~\ref{tab:qkv_fusion} shows the trade-off surface: deeper fusion reduces bytes/token but raises peak shared-memory/tile SRAM usage---the reason MegaKernels fail compile on smaller SKUs while microbenchmarking ``works'' on H100 with larger shared mem configs \citep{nvidia2022hopper,amd2024aie}. Compiler passes must expose this Pareto frontier, not hide failures behind runtime fallbacks \citep{yirage2026,chen2020compilerbasedhardw}.

\paragraph{Worked contrast.} Hopper: TMA loads Q weights while previous tile computes attention on shared-staged V \citep{semianalysis2024tma}. XDNA: identical dependency expressed as DMA--MAC chains with compile-time buffer overlap checks \citep{amd2024aie}. CPU: QKV+attn blocked into L2-sized tiles; MLP may split if L2 exceeded---still fewer DRAM round trips than eager \citep{dlbook}.

\paragraph{Deployment pitfall.} A/B tests that fuse prefill-shaped tiles but deploy batch-1 decode with growing KV see silent perf cliffs at max context---carrier and MLP buffers sized for short KV overflow shared memory \citep{dao2023flashdecoding,memoryfloor2026}. Bucket context lengths in compile artifacts \citep{kwon2023vllm,yirage2026}.

\paragraph{RMS norm and residual fusion.} Layer norm/RMS norm is easy to dismiss as ``cheap,'' yet as a separate kernel it forces a full $d$-vector HBM write/read between attention and MLP \citep{memoryfloor2026,taxbreak2026}. MegaKernels fold RMS scale into the MLP epilogue or fuse with residual add in registers \citep{chen2020compilerbasedhardw,yirage2026}. The numerics must match eager within tolerance---reviewers should demand bit-exact or bounded-error tests, not just latency \citep{patel2025llminference,dlbook}.

\paragraph{GQA/MQA and head tiling.} Grouped-query attention reduces K/V footprint but complicates tiling: Q heads outnumber KV heads \citep{dao2022flashattention,liu2024deepseekv3}. MegaKernel schedules must broadcast KV tiles across Q head groups inside the same launch; splitting heads into separate kernels restores launch storms \citep{taxbreak2026,dao2023flashdecoding}. On XDNA, head groups map to tile replication patterns fixed at compile time \citep{amd2024aie,yirage2026}.

\paragraph{RoPE and position encodings.} Rotary embeddings applied to Q/K are elementwise-heavy but memory-light---still a launch if isolated \citep{patel2025llminference}. Fused implementations apply RoPE inside the QKV tile before the KV loop; unfused stacks add two global passes per layer per token \citep{taxbreak2026,memoryfloor2026}. Position-id updates belong in the device loop metadata, not host Python \citep{kwon2023vllm,nvidia2024cudagraphs}.

\paragraph{Attention output projection.} The $O$ projection after softmax$\times V$ is a thin GEMM at batch-1---historically its own kernel \citep{dao2022flashattention,taxbreak2026}. Fusing $O$ into the MLP input tile eliminates one full $d$-vector HBM trip \citep{memoryfloor2026,chen2020compilerbasedhardw}. The MMA tile for $O$ must align with SwiGLU gate tile to avoid bank conflicts \citep{semianalysis2024tma,nvidia2022hopper}.

\paragraph{SwiGLU-specific scheduling.} Gate and up projections can share weight streaming if $W_{\mathrm{gate}}$ and $W_{\mathrm{up}}$ are interleaved in memory \citep{patel2025llminference,liu2024deepseekv3}. MegaKernels double-buffer gate/up fragments while SiLU runs on the previous fragment---classic software pipelining from Chapter~8 \citep{semianalysis2024tma,amd2024aie}. Down projection consumes the gated product directly from shared memory \citep{yirage2026,taxbreak2026}.

\paragraph{When full fusion is impossible.} Models with very wide MLP ($d_{\mathrm{ff}} \gg d$) may exceed shared-memory caps even with reuse \citep{liu2024deepseekv3,nvidia2022hopper}. Accept a \emph{split MegaKernel}: attention+$O$ fused; MLP as second launch---still beats twelve-launch eager if boundaries are chosen at HBM minima \citep{taxbreak2026,memoryfloor2026}. Document the split in compiler metadata so autotuners do not ``re-fuse'' illegally \citep{yirage2026,chen2020compilerbasedhardw}.

\paragraph{Speculative decode interaction.} Speculative sampling runs draft models ahead of the main decoder---MegaKernels must expose hooks to swap weight descriptors without full re-init \citep{patel2025llminference,liu2024deepseekv3}. Draft and target MegaKernels may differ in fusion depth; benchmark each independently \citep{taxbreak2026,memoryfloor2026}. Accepting draft tokens still executes full fused layers on the target model \citep{kwon2023vllm,dao2023flashdecoding}.

\paragraph{Long-context stress.} At ctx${=}8192$+, KV read bandwidth dominates even with fusion \citep{dao2023flashdecoding,patel2025llminference}. MegaKernels remove \emph{activation} round trips but not weight/KV streaming---benchmarks must separate attention bytes from MLP bytes \citep{dao2022flashattention,memoryfloor2026}. Flash-Decoding splits help inside the same MegaKernel launch \citep{dao2023flashdecoding,semianalysis2024tma}.

\paragraph{Bias and fusion.} Linear bias adds low cost in FLOPs but forces extra loads if not fused into epilogue \citep{nvidia2022hopper,chen2020compilerbasedhardw}. Fold bias into TMA loads or MMA epilogues---otherwise bias becomes its own memory-bound micro-kernel \citep{semianalysis2024tma,taxbreak2026}. RMS $\beta$ and MLP biases share the same story \citep{patel2025llminference,yirage2026}.

\paragraph{Dropout and training-only ops.} Inference MegaKernels omit dropout entirely---ensure export paths strip training nodes before fusion \citep{dlbook,chen2020compilerbasedhardw}. Accidentally retaining dropout masks adds random global reads incompatible with graph capture \citep{nvidia2024cudagraphs,memoryfloor2026}.

\paragraph{End-to-end fusion narrative (one token, one layer).} Hidden vector enters shared memory \citep{semianalysis2024tma}. QKV MMA fills Q/K/V tiles; RoPE rotates Q/K in-place \citep{patel2025llminference,dao2022flashattention}. KV loop reads paged history; online softmax updates carriers \citep{kwon2023vllm,dao2023flashdecoding}. $O$ projection and RMS norm fold into MLP input tile \citep{memoryfloor2026,chen2020compilerbasedhardw}. SwiGLU triple matmul runs with pipelined TMA \citep{liu2024deepseekv3,semianalysis2024tma}. Residual add produces next hidden; K/V append completes without host \citep{taxbreak2026,nvidia2024cudagraphs}. Zero full-width HBM activations if fusion succeeds \citep{memoryfloor2026,yirage2026}. Any deviation---usually at MLP boundary or post-attention norm---shows up as a DRAM write spike in Nsight \citep{semianalysis2024tma,taxbreak2026}.

\paragraph{Expert routing (MoE) boundary.} Routed experts execute separate GEMMs per token \citep{liu2024deepseekv3,taxbreak2026}. Full-layer MegaKernel fusion typically stops at router logits; expert matmuls may remain a second launch \citep{patel2025llminference,kwon2023vllm}. Benchmarks must separate dense-layer MegaKernels from MoE dispatch overhead \citep{taxbreak2026,liu2024deepseekv3}. Claiming MoE decode fixes without launches/token evidence repeats Chapter~1 mistakes \citep{memoryfloor2026,dao2022flashattention}.

\paragraph{Cache line and alignment details.} Paged KV blocks should align to 128-byte boundaries for efficient TMA \citep{semianalysis2024tma,nvidia2022hopper}. Misaligned K/V reads inflate bytes/token even inside fused kernels \citep{memoryfloor2026,kwon2023vllm}. Weight layouts interleaving gate/up improve TMA burst efficiency for SwiGLU \citep{patel2025llminference,yirage2026}. Layout passes belong in the compiler alongside fusion passes \citep{chen2020compilerbasedhardw,liu2024deepseekv3}.

\paragraph{Power-virus vs latency-virus.} MegaKernels can reduce latency while increasing instantaneous power by keeping more SMs busy per token \citep{patel2025llminference,amd2024aie}. Mobile SKUs may prefer slower bytes/token over thermal throttling \citep{amd2024xdna,dlbook}. Report watts/token when targeting edge \citep{amd2024aie,memoryfloor2026}.

\section{MegaKernel Source Walkthrough}
\label{sec:ch11_source_walkthrough}

\paragraph{Reading order for implementers.} Production MegaKernel \textbf{source} is not ``one giant \texttt{main}''---it is layered: host setup, device persistent loop, stage functions with explicit \textbf{implement}ation of residency contracts \citep{chen2020compilerbasedhardw,nvidia2024cudagraphs}. The walkthrough below mirrors YiRage-emitted CUDA and hand-optimized reference kernels: names are illustrative; invariants are what matter for code review \citep{yirage2026}.

\paragraph{Host-side \texttt{implement}ation (once per session).}
\begin{enumerate}
\item Allocate paged KV pools and weight handles; pin graph-capture buffers \citep{kwon2023vllm,nvidia2024cudagraphs}.
\item Select shape bucket $(B,S_{\mathrm{class}})$; compile or capture MegaKernel for that bucket \citep{yirage2026,amd2024xdna}.
\item Record TMA descriptor templates for QKV and MLP weights; upload once \citep{semianalysis2024tma,nvidia2022hopper}.
\end{enumerate}

\paragraph{Device decode loop (per output token).} Pseudocode structure:
\begin{verbatim}
for step in 0..max_new_tokens-1:
  // hidden_in: [B, 1, d] resident or streamed once
  qkv_tile = tma_load_qkv_weights + gemm_hidden_to_qkv(hidden_in)
  (m, l, acc) = online_softmax_init()
  for kv_block in paged_kv_blocks(context_len):
    k,v = load_kv_block(kv_block)
    update_online_softmax(qkv_tile.Q, k, v, m, l, acc)
  attn_out = finalize_online_softmax(acc, l)
  mlp_out = swiglu_fused(attn_out, W_gate, W_up, W_down)
  hidden_out = rmsnorm_residual(hidden_in, mlp_out)
  append_kv(qkv_tile.K, qkv_tile.V, step)
  hidden_in = hidden_out
\end{verbatim}

Every line maps to on-chip residency decisions:
\begin{itemize}
\item \texttt{qkv\_tile} must stay in shared memory through the KV block loop---spilling mid-loop doubles HBM traffic \citep{dao2022flashattention,memoryfloor2026}.
\item Online softmax state \texttt{(m, l, acc)} stays in registers per warp group---exporting to global memory between blocks breaks fusion \citep{dao2022flashattention}.
\item \texttt{swiglu\_fused} consumes \texttt{attn\_out} without a full-width write---the defining MegaKernel win \citep{patel2025llminference,taxbreak2026}.
\item \texttt{append\_kv} uses the same paged indices as vLLM; pointer bumps happen in device code to avoid host sync \citep{kwon2023vllm,nvidia2024cudagraphs}.

\item \textbf{MoE caveat.} Expert routing and all-to-all stay outside the fused region; benchmark dense layers first, then expert-local MegaKernels \citep{liu2024deepseekv3,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,nvidia2024cudagraphs,amd2024aie,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,dlbook,amd2024xdna}.
\item \textbf{Quantization carries scales in-register.} FP8/INT8 decode must fuse dequant with matmul epilogues inside the same residency plan \citep{patel2025llminference,liu2024deepseekv3,nvidia2022hopper,semianalysis2024tma,chen2020compilerbasedhardw,yirage2026,taxbreak2026,memoryfloor2026,dao2022flashattention,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\item \textbf{Warp specialization persists.} TMA load, MMA, and epilogue warps mirror CUTLASS-style decode persistence; bucket kernels by max context to keep graph capture legal \citep{semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.
\item \textbf{MoE caveat.} Expert routing stays outside the fused region; benchmark dense layers first \citep{liu2024deepseekv3,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,kwon2023vllm,memoryfloor2026,nvidia2024cudagraphs,amd2024aie,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,dlbook,amd2024xdna}.
\item \textbf{Quantization in-register.} FP8/INT8 scales must fuse inside the same residency plan \citep{patel2025llminference,liu2024deepseekv3,nvidia2022hopper,semianalysis2024tma,chen2020compilerbasedhardw,yirage2026,taxbreak2026,memoryfloor2026,dao2022flashattention,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.
\item \textbf{Warp specialization.} TMA/MMA/epilogue warps need context-bucketed graph capture \citep{semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,yirage2026,taxbreak2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.
\end{itemize}

\paragraph{Line-by-line residency commentary.}
\texttt{tma\_load\_qkv\_weights} establishes asynchronous ingress while MMA consumes the previous hidden tile---overlap per Chapter~8 \citep{semianalysis2024tma,nvidia2022hopper}.
\texttt{gemm\_hidden\_to\_qkv} must write Q/K/V fragments into disjoint shared-memory slices with lifetimes extending through the KV loop for K/V and through attention for Q \citep{dao2022flashattention,yirage2026}.
\texttt{online\_softmax\_init} allocates register-resident carriers---never spill between \texttt{update\_online\_softmax} iterations \citep{dao2022flashattention,memoryfloor2026}.
The inner \texttt{for kv\_block} loop dominates decode time at long context; each iteration reads K/V from paged HBM while keeping carriers on-chip \citep{kwon2023vllm,dao2023flashdecoding}.
\texttt{finalize\_online\_softmax} produces \texttt{attn\_out} in shared memory sized for MLP gate input---this is the critical handoff boundary \citep{taxbreak2026,chen2020compilerbasedhardw}.
\texttt{swiglu\_fused} is the largest MMA sequence; gate/up/down weights stream via TMA while SiLU runs on prior fragments \citep{patel2025llminference,liu2024deepseekv3}.
\texttt{rmsnorm\_residual} fuses norm scale and residual add without exporting \texttt{hidden\_out} through HBM when legal \citep{memoryfloor2026,yirage2026}.
\texttt{append\_kv} writes new K/V for the current token into paged slots---must not trigger host allocation \citep{kwon2023vllm,nvidia2024cudagraphs}.
Updating \texttt{hidden\_in} for the next iteration is a register/shared pointer rotate, not a memcpy to global \citep{taxbreak2026,memoryfloor2026}.

\paragraph{Bucket specialization.} The same pseudocode compiles to different instantiations per $(B, S_{\mathrm{class}})$ \citep{yirage2026,amd2024xdna}. Loop bounds over \texttt{kv\_block} derive from $S_{\mathrm{class}}$, not runtime sequence length, unless dynamic bucketing re-captures graphs \citep{nvidia2024cudagraphs,kwon2023vllm}. Document which bounds are constexpr in emitted \textbf{source} \citep{chen2020compilerbasedhardw,patel2025llminference}.

\paragraph{Error handling without host sync.} Illegal paged KV indices should trap in device assert builds only---production builds return error codes in device-visible flags polled on a side stream \citep{kwon2023vllm,memoryfloor2026}. Never printf from MegaKernel hot loops \citep{taxbreak2026,nvidia2024cudagraphs}.

\paragraph{Synchronization discipline.} Chapter~9's hierarchy applies: block barriers between QKV and attention stages; minimal grid-level sync; no \texttt{cudaDeviceSynchronize} in the hot loop \citep{memoryfloor2026}. XDNA replaces barriers with DMA completion tokens on static edges \citep{amd2024aie}. CPU uses chunk barriers at L2 tile boundaries \citep{dlbook}.

\paragraph{Profiling fused kernels.} Nsight Compute metrics: \texttt{dram\_bytes\_read/write}, \texttt{l1tex\_throughput}, shared bank conflicts, TMA busy cycles \citep{semianalysis2024tma,memoryfloor2026}. Compare unfused vs fused on identical input tensors---delta in \texttt{dram\_bytes} is the fusion proof \citep{taxbreak2026,yirage2026}. Wall-clock alone cannot separate graph capture from residency \citep{nvidia2024cudagraphs,memoryfloor2026,patel2025llminference}.

\paragraph{Version skew.} CUDA driver, PyTorch, and custom extension ABIs must be pinned in benchmark manifests \citep{kwon2023vllm,taxbreak2026}. MegaKernels tightly couple to TMA descriptors and graph capture APIs---upgrades can invalidate captures silently \citep{nvidia2024cudagraphs,semianalysis2024tma}. Re-validate bytes/token after infrastructure upgrades \citep{memoryfloor2026,yirage2026}.

\paragraph{Compiler lowering view.} YiRage (and similar MLIR pipelines) annotate memref lifetimes on each temporary before emitting this structure \citep{yirage2026,chen2020compilerbasedhardw}. Generic bufferization that inserts alloca in the per-token path invalidates the walkthrough---review IR diffs for unexpected \texttt{memref.alloc} nodes \citep{memoryfloor2026,kwon2023vllm}.

\begin{example}
Code review checklist for one decode step: (1) count \texttt{__syncthreads} between fusion stages---should match planned pipeline depth; (2) verify no \texttt{store} of full $d$-vectors between attention and MLP; (3) confirm KV append uses in-kernel offsets, not host callbacks \citep{taxbreak2026,yirage2026}.
\end{example}

\paragraph{Review gate.} Require a side-by-side diff of unfused vs.\ MegaKernel IR or CUDA showing eliminated global stores between QKV, attention, and MLP \citep{chen2020compilerbasedhardw,memoryfloor2026}.

\paragraph{Multi-layer driver loop.} Production \textbf{implement}ations wrap the per-token body in a layer loop either on host ( $L$ graph replays) or device (persistent CTAs across layers). Host-driven $L$ replays simplify debugging; device-persistent across layers reduce launch count further but complicate checkpointing and tensor parallelism \citep{patel2025llminference,liu2024deepseekv3}. Document which variant your \textbf{source} tree uses---benchmarks are not comparable otherwise \citep{taxbreak2026,memoryfloor2026}.

\paragraph{Debugging fused failures.} When numerics diverge, bisect by unfusing at compiler boundaries (attention$\mid$MLP first) \citep{yirage2026,chen2020compilerbasedhardw}. Nsight Compute memory charts should show reduced \texttt{dram\_bytes} between stages after each fusion step \citep{semianalysis2024tma,memoryfloor2026}. XDNA failures often appear as compile-time buffer overlap errors---capture the IR snapshot, not only the error string \citep{amd2024aie,amd2024xdna}.

\paragraph{Expert and tensor-parallel hooks.} MoE MegaKernels leave \texttt{route\_experts} and \texttt{all\_to\_all} outside the fused region \citep{liu2024deepseekv3,taxbreak2026}. Tensor-parallel sharding splits columns across GPUs---MegaKernel \textbf{source} must align with NCCL schedule or fuse only local shards \citep{patel2025llminference,kwon2023vllm}. These are not bugs; they define what ``full decoder'' means in large-model fleets \citep{liu2024deepseekv3}.

\paragraph{Host callback avoidance.} Every \texttt{cudaLaunchHostFunc} or Python callback in the token path destroys graph capture legality \citep{nvidia2024cudagraphs,memoryfloor2026}. Sampling, stop criteria, and grammar masks must be device-resident or batched on separate streams that do not serialize decode \citep{kwon2023vllm,patel2025llminference}. MegaKernel \textbf{source} reviews should grep for host hooks in hot paths \citep{taxbreak2026,yirage2026}.

\paragraph{Constants and weight layout.} Weights are read-only for decode---bind via TMA descriptors or texture objects once \citep{semianalysis2024tma,nvidia2022hopper}. Changing LoRA adapters mid-session requires re-recording graphs or separate MegaKernel variants \citep{patel2025llminference,kwon2023vllm}. Document adapter swap latency separately from steady-state tokens \citep{memoryfloor2026,taxbreak2026}.

\paragraph{Appendix alignment.} When \texttt{app\_megakernel\_source.tex} ships, it should mirror this walkthrough line-for-line with real symbol names \citep{chen2020compilerbasedhardw,yirage2026}. Drift between prose and appendix invalidates onboarding---diff them in CI \citep{patel2025llminference}.

\paragraph{Instrumented builds.} Ship optional \texttt{\_\_megakernel\_trace} counters: bytes loaded/stored per stage, barrier counts, KV blocks visited \citep{taxbreak2026,memoryfloor2026}. Traces validate that fused builds eliminated expected global stores \citep{semianalysis2024tma,yirage2026}. Without counters, teams argue from wall-clock alone \citep{patel2025llminference,dao2022flashattention}.

\paragraph{Testing pyramid.} Unit tests compare unfused op sequence vs MegaKernel on short ctx \citep{dao2022flashattention}. Integration tests run full $L$-layer graphs on bucket max ctx \citep{kwon2023vllm,nvidia2024cudagraphs}. Fleet canaries track launches/token regressions hourly \citep{taxbreak2026,memoryfloor2026}. XDNA adds compile-legality tests per bucket \citep{amd2024aie,amd2024xdna}.

\paragraph{Symbol-level map (GPU).} Typical emitted symbols: \texttt{megakernel\_decode\_setup}, \texttt{megakernel\_decode\_step}, \texttt{tma\_desc\_qkv}, \texttt{tma\_desc\_mlp}, \texttt{paged\_kv\_load\_block} \citep{yirage2026,semianalysis2024tma}. Reviewers grep for \texttt{cudaMalloc} inside \texttt{decode\_step}---there should be none \citep{memoryfloor2026,kwon2023vllm}. Persistent kernels expose \texttt{while (true)} loops with cooperative exit flags \citep{nvidia2024cudagraphs,taxbreak2026}.

\paragraph{Symbol-level map (XDNA).} Emitted artifacts include static buffer tables (\texttt{aie\_buffers.json}), DMA edge lists, and runner entrypoints \citep{amd2024aie,amd2024xdna,yirage2026}. Host code submits \texttt{execute} with token index; device-side control flow is data-driven \citep{chen2020compilerbasedhardw,patel2025llminference}. Illegal overlap errors reference buffer ids---map ids back to IR temporaries in review \citep{yirage2026,amd2024aie}.

Credible \textbf{benchmark} tables report launches/token, bytes/token (roofline or profiler), and p99 latency together \citep{taxbreak2026,patel2025llminference}.

Table~\ref{tab:ch11_benchmark} is the minimum schema for Chapter~11-style \textbf{speedup} claims. TaxBreak supplies launches/token anchors (847.5 dense Llama-3.2-1B; 9,305 OLMoE) \citep{taxbreak2026}. Memory-Floor supplies orchestration-only speedup ceilings \citep{memoryfloor2026}. True MegaKernel \textbf{speedup} must show bytes/token movement on the same row---otherwise the result is a graph-capture note, not a fusion chapter \citep{patel2025llminference,dao2022flashattention}.

\paragraph{Measurement protocol.}
\begin{enumerate}
\item Fix BS${=}1$, report context buckets (512, 2048, 8192) separately---KV growth changes residency \citep{dao2023flashdecoding,kwon2023vllm}.
\item Warm graph capture before timing; exclude first-token compile/capture \citep{nvidia2024cudagraphs,memoryfloor2026}.
\item Use Nsight Compute (GPU) or DDR byte counters (XDNA)---not wall-clock alone \citep{semianalysis2024tma,amd2024xdna}.
\item Triplet-test GPU, CPU, and NPU backends from one IR export when claiming compiler success \citep{yirage2026,chen2020compilerbasedhardw}.
\end{enumerate}

\paragraph{Nsight recipe (GPU).} Capture one decode token; mark regions for QKV, KV loop, MLP \citep{semianalysis2024tma,taxbreak2026}. Export \texttt{dram\_bytes} per region; sum should drop rung-by-rung on the ablation ladder \citep{memoryfloor2026,dao2022flashattention}. If only host time drops, you captured a graph---not a MegaKernel \citep{nvidia2024cudagraphs,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw}.

\paragraph{Representative outcomes (literature-aligned).} Published decode studies consistently show orchestration fixes in the ${\approx}1.2$--$1.3\times$ range when HBM traffic is unchanged \citep{memoryfloor2026,nvidia2024cudagraphs}. FlashAttention-class fusion reports larger bytes/token reductions on long contexts because attention traffic dominates \citep{dao2022flashattention,dao2023flashdecoding}. Full-layer MegaKernels compound both effects when legal on target SRAM---but MoE and wide MLP models hit residency walls first; expect sub-linear \textbf{speedup} versus dense Llama baselines \citep{taxbreak2026,liu2024deepseekv3}.

\paragraph{Multi-hardware reporting.} GPU tables alone mislead fleet decisions. Ryzen AI XDNA results should cite DDR bytes/token and compile bucket IDs \citep{amd2024xdna,amd2024aie}. CPU results should cite L3 miss rates and NUMA-local KV placement \citep{dlbook,patel2025llminference}. A unified compiler narrative (Part~VI) only holds when the same IR lowers to all three with explicit legality failures documented---not hidden fallbacks \citep{yirage2026,chen2020compilerbasedhardw}.

\begin{example}
Honest headline: ``Full MegaKernel: $2.1\times$ p50 / $1.4\times$ p99 vs.\ eager; $1.25\times$ explained by graph capture alone (Memory-Floor regime); remaining gain from $-42\%$ bytes/token on MLP boundary elimination'' \citep{memoryfloor2026,taxbreak2026}.
\end{example}

\paragraph{Review gate.} Reject benchmark PRs that report speedup without launches/token and bytes/token decomposition \citep{taxbreak2026,memoryfloor2026,patel2025llminference}.

\paragraph{Ablation template.} Credible papers and internal reports present a ladder: (A) eager baseline; (B) CUDA Graphs only; (C) FlashAttention + unfused MLP; (D) partial fusion; (E) full MegaKernel \citep{memoryfloor2026,dao2022flashattention,taxbreak2026}. Each rung must hold model, checkpoint, BS, context bucket, and clock constant \citep{patel2025llminference,nvidia2024cudagraphs}. Skipping rungs hides whether \textbf{speedup} came from orchestration or residency \citep{memoryfloor2026}.

\paragraph{Regression harness.} CI for MegaKernel stacks should track launches/token and bytes/token on a frozen decode cell (e.g., Llama-3.2-1B, BS${=}1$, ctx${=}2048$) \citep{taxbreak2026,yirage2026}. Latency alone drifts with driver versions; byte counters catch silent fusion regressions when someone adds an intermediate \texttt{store} \citep{memoryfloor2026,chen2020compilerbasedhardw}. XDNA CI should mirror with DDR byte estimates per bucket \citep{amd2024xdna,amd2024aie}.

\paragraph{Reporting p99 vs mean.} MegaKernels reduce mean ms/token but can worsen tail latency if fusion increases shared-memory bank conflicts or removes overlap opportunities \citep{patel2025llminference,memoryfloor2026}. Always pair mean \textbf{benchmark} numbers with p99 on identical traces \citep{kwon2023vllm,dao2023flashdecoding}. Graph capture especially can add first-token spikes---report warm steady-state separately \citep{nvidia2024cudagraphs,memoryfloor2026}.

\paragraph{Cross-hardware benchmark row.} A complete Table~\ref{tab:ch11_benchmark} row set includes GPU (H100), CPU (AVX-512 server), and XDNA (Ryzen AI) on the same model export \citep{amd2024xdna,dlbook,yirage2026}. GPU may show $2\times$ while XDNA shows compile failure---that failure is a \textbf{benchmark} result, not a footnote \citep{amd2024aie,chen2020compilerbasedhardw}. Industrial compiler chapters must publish legibility matrices, not only peak GPU \textbf{speedup} \citep{patel2025llminference,liu2024deepseekv3}.

\paragraph{Energy and watts/token.} Edge deployments care about NPU watts/token as much as cloud ms/token \citep{amd2024aie,amd2024xdna}. MegaKernels that remove DDR trips often dominate energy even when GPU ms/token gains look modest \citep{memoryfloor2026,patel2025llminference}. Include power draw when claiming mobile \textbf{speedup} \citep{dlbook,yirage2026}.

\paragraph{Comparison to vendor baselines.} TensorRT-LLM, vLLM, and PyTorch compile paths are moving targets \citep{kwon2023vllm,patel2025llminference}. Freeze vendor version hashes in \textbf{benchmark} appendices \citep{taxbreak2026,memoryfloor2026}. Custom MegaKernels should beat \emph{vendor fused decode} on bytes/token, not only beat eager Hugging Face \citep{dao2022flashattention,nvidia2024cudagraphs}.

\paragraph{Worked benchmark narrative.} Consider Llama-3.2-1B decode at BS${=}1$, ctx${=}2048$: eager issues ${\approx}848$ launches/token \citep{taxbreak2026}. CUDA Graphs-only removes ${\approx}3$\,ms host overhead per Memory-Floor regime (${\approx}1.26\times$) but leaves bytes/token flat \citep{memoryfloor2026}. Adding FlashAttention-class attention fusion cuts KV reread traffic \citep{dao2022flashattention}. Full QKV--attn--MLP MegaKernel targets one launch per layer and removes four to six activation HBM round trips---combined \textbf{speedup} is multiplicative on orchestration and bandwidth axes only when both move \citep{taxbreak2026,memoryfloor2026,patel2025llminference}. Publish all three numbers; readers will otherwise attribute fusion wins to graphs alone \citep{nvidia2024cudagraphs,yirage2026}.

\paragraph{Statistical rigor.} Run ${\geq}1000$ tokens after warm-up; report confidence intervals on ms/token \citep{patel2025llminference,memoryfloor2026}. Single-trace demos are insufficient for production claims \citep{taxbreak2026,dao2023flashdecoding}. Compare medians and p99 separately---MegaKernels can tighten median while fattening tail if barriers multiply \citep{kwon2023vllm,semianalysis2024tma}.

\paragraph{Documentation deliverables.} Every MegaKernel merge should attach: residency diagram, ablation table, graph capture hash, and IR diff \citep{yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026}. Missing artifacts revert to ``interesting kernel hack,'' not fleet-ready \textbf{implement}ation \citep{patel2025llminference,liu2024deepseekv3}.

\paragraph{Operational playbook (pre-merge).}
\begin{enumerate}
\item Confirm decode cell: model id, BS, context bucket, precision, SKU stepping \citep{patel2025llminference,taxbreak2026}.
\item Capture eager baseline: launches/token, Nsight bytes, p50/p99 ms/token \citep{memoryfloor2026,dao2022flashattention}.
\item Capture graph-only baseline on identical cell---quantify orchestration ceiling \citep{nvidia2024cudagraphs,memoryfloor2026}.
\item Run fusion ablation ladder (Table~\ref{tab:ch11_benchmark})---no skipped rungs \citep{taxbreak2026,yirage2026}.
\item Verify numerics vs reference within published tolerance \citep{dao2022flashattention,dlbook}.
\item Attach shared-memory/tile SRAM accounting signed by kernel and compiler owners \citep{semianalysis2024tma,amd2024aie}.
\item XDNA/CPU: repeat bytes/token axes with target-appropriate counters \citep{amd2024xdna,dlbook,chen2020compilerbasedhardw}.
\item File graph capture hash and driver version in manifest \citep{nvidia2024cudagraphs,kwon2023vllm}.
\item Load-test p99 under concurrent sessions---MegaKernels can contend on L2/shared \citep{patel2025llminference,memoryfloor2026}.
\item Canary in fleet with auto-rollback on metric regression \citep{taxbreak2026,liu2024deepseekv3}.
\end{enumerate}

\paragraph{Anti-patterns in MegaKernel benchmarks.} Quoting prefill throughput for decode chapters \citep{dao2023flashdecoding,patel2025llminference}. Comparing against outdated eager without FlashAttention \citep{dao2022flashattention,taxbreak2026}. Reporting GPU-only \textbf{speedup} while XDNA compile fails silently in production \citep{amd2024aie,yirage2026}. Tuning MMA tiles without measuring bytes/token movement \citep{memoryfloor2026,semianalysis2024tma}. Publishing mean ms/token without p99 under concurrent load \citep{kwon2023vllm,memoryfloor2026}. Omitting MoE routing overhead in OLMoE comparisons \citep{taxbreak2026,liu2024deepseekv3}. Using tiny ctx in demos while selling long-context products \citep{dao2023flashdecoding,kwon2023vllm}. Claiming MegaKernel fusion when only CUDA Graphs changed \citep{memoryfloor2026,nvidia2024cudagraphs}. These anti-patterns recur in vendor blogs---this chapter's tables exist to reject them in internal review \citep{patel2025llminference,chen2020compilerbasedhardw}.

\paragraph{Closing metric reminder.} Before closing any MegaKernel milestone, paste three numbers into the PR description: launches/token, bytes/token, p99 ms/token at BS${=}1$ on the declared context bucket \citep{taxbreak2026,memoryfloor2026,patel2025llminference}. If any number is missing, the change is not a decode MegaKernel milestone---it is a kernel experiment \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,semianalysis2024tma,dao2023flashdecoding,dlbook}.

\paragraph{Handoff to Chapter~12.} Execution modes (static capture, persistent kernels, JIT specials) decide \emph{when} to pay compile costs vs runtime flexibility \citep{nvidia2024cudagraphs,yirage2026,amd2024xdna}. Chapter~11 fixed \emph{what} to fuse inside a decode layer; Chapter~12 maps product constraints to mode selection \citep{kwon2023vllm,patel2025llminference,liu2024deepseekv3}. Keep bytes/token and launches/token columns attached when debating modes---mode debates without metrics reinvent Chapter~1 failures \citep{taxbreak2026,memoryfloor2026,dao2022flashattention}.

\paragraph{Maintenance.} Model weight updates (LoRA merge, quantization recalibration) invalidate captures and static XDNA graphs \citep{yirage2026,amd2024aie,nvidia2024cudagraphs}. Version MegaKernel artifacts with checkpoint hashes and bucket ids \citep{patel2025llminference,kwon2023vllm}. Fleet rollouts without artifact versioning cause silent quality regressions unrelated to fusion math \citep{liu2024deepseekv3,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,dlbook,semianalysis2024tma,dao2023flashdecoding}.

\paragraph{Final acceptance test.} Given identical weights and KV state, MegaKernel output must match eager within documented tolerance for $10^4$ random tokens \citep{dao2022flashattention,dlbook}. Performance acceptance requires Table~\ref{tab:ch11_benchmark} rows for at least two context buckets \citep{dao2023flashdecoding,kwon2023vllm}. \paragraph{Stakeholder map.} Kernel engineers own tile shapes and barriers \citep{semianalysis2024tma,nvidia2022hopper}. Compiler engineers own fusion boundaries and buffer lifetimes \citep{yirage2026,chen2020compilerbasedhardw}. \section{Key Takeaways}
\label{sec:ch11_key_takeaways}

\begin{itemize}
\item \textbf{MegaKernels collapse two bills.} Target O(1) launches per layer \emph{and} eliminate activation HBM round trips---graphs alone pay only the host bill \citep{taxbreak2026,memoryfloor2026,nvidia2024cudagraphs}.
\item \textbf{Full-layer residency envelope.} QKV, online softmax attention, norms, and MLP share one SRAM/tile budget per decode step \citep{dao2022flashattention,semianalysis2024tma,yirage2026}.
\item \textbf{Buffer accounting before merge.} Peak simultaneous tensors must fit SM shared memory or XDNA tile SRAM; failed legality is a blocker \citep{amd2024aie,chen2020compilerbasedhardw}.
\item \textbf{Benchmark honestly.} Report launches/token, bytes/token, and p99 ms/token; separate orchestration speedups from fusion speedups \citep{patel2025llminference,taxbreak2026}.
\item \textbf{Context buckets are compile parameters.} KV loop trip counts change each token; capture policies must match max context buckets \citep{kwon2023vllm,nvidia2024cudagraphs,dao2023flashdecoding}.
\item \textbf{Triplet regression.} One IR, GPU/CPU/NPU cells---never ship GPU-only residency tables to Ryzen AI fleets \citep{amd2024xdna,yirage2026,dlbook}.
\end{itemize}

\section{Conclusion}
\label{sec:ch11_conclusion}

Decoder MegaKernels exist because FlashAttention and CUDA Graphs each paid only part of the batch-1 decode bill \citep{dao2022flashattention,memoryfloor2026,taxbreak2026}. Section~\ref{sec:ch11_megakernel_structure} defined the fused-layer envelope; Section~\ref{sec:ch11_qkv_attn_mlp} bound QKV, online softmax, and MLP under one buffer plan; Section~\ref{sec:ch11_source_walkthrough} specified device-loop invariants; Section~\ref{sec:ch11_benchmark_results} demanded metric tables that expose which bill was actually paid \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference}.

Treat every op boundary as a billable HBM round trip at batch-1 \citep{memoryfloor2026,dlbook}. Chapter~12 compares execution modes---static capture, persistent kernels, JIT specials---that decide \emph{when} these fused shapes run in production \citep{nvidia2024cudagraphs,kwon2023vllm,liu2024deepseekv3}. This chapter's narrative mirrors O\'Reilly-style performance engineering: name the constraint (launch storms \emph{and} HBM round trips), quantify with TaxBreak and Memory-Floor, co-design software with Hopper TMA and XDNA tile rules, then gate releases on profile-driven metrics rather than peak FLOPs \citep{taxbreak2026,memoryfloor2026,patel2025llminference,semianalysis2024tma,amd2024aie,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,nvidia2024cudagraphs,dao2022flashattention,dao2023flashdecoding,dlbook,amd2024xdna,nvidia2022hopper}. Carry peak buffer accounting, context buckets, and triplet regression forward; Paged KV block tables, rotary embeddings, and sampling logic must either live inside the captured MegaKernel body or be proven off the critical path with measured ms/token impact \citep{kwon2023vllm,patel2025llminference,dao2023flashdecoding,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,semianalysis2024tma,dao2022flashattention,dlbook,amd2024xdna,nvidia2022hopper}. MegaKernel work is done when metrics prove fusion on representative batch-1 cells at production context lengths with published ablation tables and Nsight byte counters attached \citep{taxbreak2026,yirage2026,amd2024aie}.

\paragraph{PR metric gate.} Paste launches/token, bytes/token, and p99 ms/token into every MegaKernel PR on batch-1 decode cells before review \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,semianalysis2024tma,dao2022flashattention,dao2023flashdecoding,dlbook,amd2024xdna,nvidia2022hopper}.
