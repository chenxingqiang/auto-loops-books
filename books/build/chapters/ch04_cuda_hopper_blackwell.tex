\chapter{CUDA Hopper/Blackwell: Hardware Properties and Optimization Bounds}
\label{chap:ch04}
\typeout{START_CHAPTER "ch04" \theabspage}

Can faster Blackwell tensor cores fix batch-1 decode if activations still round-trip through HBM every op boundary? Fregly's mechanical sympathy answer is no: hardware features pay off only when algorithms and kernels are co-designed for the memory hierarchy they run on \citep{nvidia2022hopper,patel2025llminference,semianalysis2024tma}. FlashAttention and DeepSeek MLA succeeded on constrained H800 clusters by reshaping attention around bandwidth and on-chip residency---not by ignoring silicon limits \citep{dao2022flashattention,liu2024deepseekv3,memoryfloor2026}.

This chapter maps Hopper/Blackwell CUDA mechanisms---registers, shared memory, TMA, thread-block clusters, tensor cores, and PTX softmax micro-ops---to decode metrics from Chapter~1: kernels/token, bytes/token, and $R_{\mathrm{floor}}$ \citep{taxbreak2026,memoryfloor2026}. The mistake is treating Hopper as ``more FLOPs'' while launch storms and spill traffic dominate TaxBreak cells \citep{taxbreak2026,nvidia2024cudagraphs}. YiRage's CUDA backend must legalize fusion against these bounds before codegen; Chapter~5 shows contrasting static rules on XDNA \citep{yirage2026,amd2024xdna,chen2020compilerbasedhardw}.

TaxBreak dense Llama-3.2-1B averages 847.5 kernel launches per output token on H100; Memory-Floor reports $R_{\mathrm{floor}}\approx 0.27$ on H100 vs ${\approx}0.81$ on L4 for Qwen-2.5-7B at batch-1 \citep{taxbreak2026,memoryfloor2026,patel2025llminference}. Hopper tuning must follow profile-first ordering from Chapter~1, not datasheet peaks \citep{dlbook,semianalysis2024tma}.

\paragraph{What this chapter is not.} This is not a CUDA tutorial on writing faster matrix multiply microbenchmarks \citep{dlbook,patel2025llminference}. Nor is it a Blackwell launch brochure \citep{nvidia2022hopper,liu2024deepseekv3}. We document \emph{optimization bounds}: which Hopper mechanisms matter for batch-1 decode residency, which are distractions when $R_{\mathrm{floor}}$ is low, and how a multi-hardware compiler lowers the same IR without forking legality rules \citep{yirage2026,chen2020compilerbasedhardw,memoryfloor2026,amd2024xdna}.

\paragraph{Reader prerequisites.} You should carry Chapter~1 metrics---kernels/token, bytes/token, ms/token, and $R_{\mathrm{floor}}$---and Chapter~3 hardware vocabulary \citep{taxbreak2026,memoryfloor2026,patel2025llminference}. When this chapter cites FlashAttention or Flash-Decoding, it assumes you accept online softmax as the attention control-flow spine; Chapter~10 derives the numerics \citep{dao2022flashattention,dao2023flashdecoding}.

\paragraph{Organization.} Sections~\ref{sec:cuda_memory_hierarchy}--\ref{sec:sync_mechanisms} map CUDA tiers and synchronization to fusion plans \citep{nvidia2022hopper,semianalysis2024tma,dao2022flashattention}. Sections~\ref{sec:ch04_tensor_core_bounds}--\ref{sec:ch04_ptx} explain why tensor-core tuning is usually third in the queue and how PTX choices bind softmax stability \citep{memoryfloor2026,patel2025llminference,liu2024deepseekv3}. Sections~\ref{sec:ch04_yirage_cuda_backend}--\ref{sec:ch04_profiling} connect IR lowering and profilers to ship gates \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026}. The worked example in Section~\ref{sec:ch04_worked_example} ties the story to one decode layer \citep{taxbreak2026,memoryfloor2026,kwon2023vllm}.

\section{CUDA Memory Hierarchy}
\label{sec:cuda_memory_hierarchy}
\paragraph{Memory tier cheat sheet for reviewers.} When reviewing CUDA diffs, ask: where do QKV fragments live between ops? Where do online softmax $(m,\ell,o)$ live during the KV loop? Where does SwiGLU intermediate sit relative to TMA staging buffers? Any answer of ``global'' between fused stages is a bytes/token regression unless proven otherwise \citep{memoryfloor2026,taxbreak2026,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,patel2025llminference,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Shared memory is a hard budget: two concurrent $d$-model vectors plus MMA tiles can exceed 228\,KB if planners double-book buffers \citep{semianalysis2024tma,taxbreak2026,yirage2026}.

\paragraph{KV cache traffic.} Weights and KV still live in HBM---fusion removes \emph{activation} round trips, not parameter reads \citep{kwon2023vllm,dlbook,patel2025llminference,dao2022flashattention,memoryfloor2026}. Roofline thinking still applies: decode AI stays on the bandwidth slope for realistic LLM sizes \citep{taxbreak2026,semianalysis2024tma,nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dao2023flashdecoding}.

\paragraph{The residency envelope.} Decoder MegaKernels simultaneously hold QKV fragments, online softmax carriers, SwiGLU tiles, and TMA staging buffers \citep{taxbreak2026,dao2022flashattention,yirage2026}. Each temporary must map to a CUDA memory tier for the kernel duration---spill to HBM is a bytes/token bill Chapter~1 measured \citep{memoryfloor2026,patel2025llminference}.

Table~\ref{tab:cuda_memory} is the tier model for decode fusion \citep{nvidia2022hopper,semianalysis2024tma}. \textbf{Registers} hold per-thread carriers and MMA fragments; exceeding register caps lowers occupancy and can spill \citep{nvidia2022hopper}. \textbf{Shared memory} is the MegaKernel scratchpad---typically 228\,KB/SM configurable on Hopper---where fused stages exchange data without HBM \citep{semianalysis2024tma,taxbreak2026}. \textbf{Global memory} is for weights, KV cache, and failures \citep{kwon2023vllm,dlbook}.

\paragraph{Register pressure vs fusion depth.} Fusing QKV, attention, and MLP increases live ranges in registers and shared memory \citep{chen2020compilerbasedhardw,yirage2026}. A kernel that fuses FLOPs but spills carriers to global memory loses on bytes/token \citep{memoryfloor2026,dao2022flashattention}. Compiler passes need joint allocation---not per-op peaks \citep{yirage2026,liu2024deepseekv3}.

\paragraph{Multi-hardware delta.} Hopper uses shared memory + registers; XDNA uses fixed tile SRAM; CPUs use L2 blocking \citep{amd2024xdna,amd2024aie,dlbook}. Identical math, different legality \citep{patel2025llminference,chen2020compilerbasedhardw}.

\begin{example}
Llama-3.2-1B decode: six $d$-vector HBM round trips per unfused layer vs one fused shared-memory plan---bytes/token drops with unchanged MMA tiles \citep{taxbreak2026,dao2022flashattention,memoryfloor2026}.
\end{example}

\paragraph{Roofline reminder at batch one.} Decode GEMMs present $M=1$ or small $M$ with large $K/N$ \citep{dlbook,patel2025llminference,nvidia2022hopper}. Arithmetic intensity stays on the memory roof for realistic LLM widths \citep{memoryfloor2026,semianalysis2024tma}. Registers and shared memory are how kernels \emph{buy} effective intensity without changing FLOP counts \citep{dao2022flashattention,yirage2026,chen2020compilerbasedhardw}. Treat every planned global store of length $d$ as a roofline penalty \citep{taxbreak2026,memoryfloor2026}.

\paragraph{Spill versus intentional global traffic.} Weights, KV cache entries, and final layer outputs legitimately live in HBM \citep{kwon2023vllm,dlbook}. Spilled $(m,\ell,o)$ carriers or softmax score tiles do not \citep{dao2022flashattention,memoryfloor2026,patel2025llminference}. Nsight Compute differentiates them: spilled locals show as local memory loads; intentional KV traffic shows as regular DRAM throughput aligned with cache lines \citep{semianalysis2024tma,patel2025llminference,yirage2026}.

\paragraph{Spill signatures in profilers.} Nsight Compute local memory loads and excessive register usage often precede bytes/token regressions \citep{patel2025llminference,semianalysis2024tma}. Treat spill as a fusion bug, not an occupancy knob \citep{memoryfloor2026,yirage2026}.

\section{Tensor Memory Accelerator (TMA)}
\label{sec:tma}
\paragraph{TMA vs manual loads.} Scalar global loads still appear in legacy decode kernels \citep{taxbreak2026,patel2025llminference,semianalysis2024tma,nvidia2022hopper}. TMA reduces instruction overhead and enables asynchronous copy/compute overlap when tile sizes justify descriptor setup \citep{yirage2026,chen2020compilerbasedhardw,dao2023flashdecoding,kwon2023vllm,memoryfloor2026,dao2022flashattention,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook}. Profile both paths on target shapes before mandating TMA in YiRage lowering \citep{yirage2026,taxbreak2026}.

\paragraph{Double-buffer invariant.} At steady state, one buffer is consumed by MMA while TMA fills the other \citep{semianalysis2024tma,dao2022flashattention}. Violating the invariant---compute on a buffer being overwritten---is a silent correctness bug that stability tests may miss if they use small $T$ \citep{dao2023flashdecoding,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,taxbreak2026,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook}.

\paragraph{Why TMA matters for decode.} Hopper's Tensor Memory Accelerator issues bulk global$\to$shared copies asynchronously while MMA units compute on the previous tile \citep{nvidia2022hopper,semianalysis2024tma}. KV-heavy decode loops use TMA to reduce scalar loads and enable double-buffer pipelines Chapter~8 details \citep{dao2023flashdecoding,kwon2023vllm}.

TMA overlaps latency; it does not remove weight/KV bytes \citep{memoryfloor2026,patel2025llminference}. When $R_{\mathrm{floor}}$ is low, fusion must still cut activation traffic \citep{memoryfloor2026,taxbreak2026}.

\paragraph{Paged KV and descriptors.} vLLM block tables complicate TMA descriptor setup---device-side walks must stay inside MegaKernel loops \citep{kwon2023vllm,patel2025llminference,yirage2026}. Illegal descriptors should fail compile \citep{chen2020compilerbasedhardw}.

\paragraph{Batch-1 caveat.} Thin GEMMs may not amortize TMA setup \citep{dlbook,taxbreak2026}. MegaKernels amortize across fused stages in one launch \citep{taxbreak2026,yirage2026,nvidia2024cudagraphs}.

\paragraph{TMA descriptor lifecycle.} Host code or device-side setup creates a TMA descriptor specifying global tensor layout, shared-memory destination, and copy box sizes \citep{nvidia2022hopper,semianalysis2024tma}. In decode MegaKernels, descriptors for weight tiles are often static per compiled bucket while KV descriptors vary with block table indices \citep{kwon2023vllm,yirage2026,patel2025llminference}. Getting layout wrong produces misaligned copies that are hard to debug without numerical diffs \citep{dao2023flashdecoding,chen2020compilerbasedhardw}.

\paragraph{When manual loads win.} Very thin tiles, irregular paged layouts, or early bring-up before descriptor tooling exists may favor vectorized global loads \citep{taxbreak2026,dlbook,patel2025llminference}. YiRage should preserve both lowering paths and let profile evidence choose \citep{yirage2026,semianalysis2024tma,memoryfloor2026}. Mandating TMA everywhere is an anti-pattern listed later in this chapter \citep{taxbreak2026,nvidia2022hopper}.

\section{Thread Block Clusters and Distributed Shared Memory}
\label{sec:ch04_cluster_dsm}
\paragraph{When clusters help softmax.} Online softmax max-reductions across wide score tiles may require cross-block cooperation \citep{dao2022flashattention,dao2023flashdecoding,semianalysis2024tma,nvidia2022hopper}. DSM lets blocks share partial max/sum without writing tiles to HBM \citep{yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook}. If cluster dims are unavailable, Flash-Decoding-style SM splits need careful merge kernels \citep{dao2023flashdecoding,taxbreak2026}.

\paragraph{Compiler cluster dims.} YiRage must emit explicit cluster launch bounds; illegal combos fail at compile time \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,amd2024aie,memoryfloor2026,patel2025llminference,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024xdna,dlbook,dao2023flashdecoding,taxbreak2026}.

\paragraph{Clusters widen on-chip fusion.} Hopper thread-block clusters expose distributed shared memory (DSM) across cooperating blocks \citep{nvidia2022hopper,semianalysis2024tma}. Decode kernels use clusters for wider softmax reductions and larger fused tiles without HBM export \citep{dao2022flashattention,dao2023flashdecoding}.

\paragraph{Legality trade-offs.} Cluster resources are finite; failed plans should fail compile \citep{yirage2026,chen2020compilerbasedhardw,memoryfloor2026}. XDNA uses static multi-tile DMA instead of DSM \citep{amd2024aie,amd2024xdna}.

\section{Synchronization Mechanisms}
\label{sec:sync_mechanisms}
\paragraph{Barrier minimization algorithm (informal).} List producer--consumer edges in the fused DAG; place block sync only on edges crossing shared-memory phases \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,dao2022flashattention,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Extra barriers show up as pipeline bubbles in Nsight timelines \citep{patel2025llminference,semianalysis2024tma}.

\paragraph{Warp shuffle for softmax.} Row-wise max and partial sums can use shuffle before touching shared memory \citep{dao2022flashattention,patel2025llminference,nvidia2022hopper,dlbook,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024aie,amd2024xdna,dao2023flashdecoding}.

\paragraph{Sync sparingly.} Unnecessary barriers serialize TMA pipelines \citep{semianalysis2024tma,nvidia2022hopper}. MegaKernels need barriers only at buffer handoffs \citep{yirage2026,chen2020compilerbasedhardw}.

\section{Tensor Core Bounds at Batch-1 Decode}
\label{sec:ch04_tensor_core_bounds}
\paragraph{Worked example (qualitative).} Team A improves MMA tile from 1\% to 3\% tensor util; Team B fuses MLP boundary saving 40\% activation bytes---only Team B moves p99 at batch-1 on H100 when $R_{\mathrm{floor}}$ is low \citep{memoryfloor2026,taxbreak2026,patel2025llminference,semianalysis2024tma,nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Tensor core vs SFU balance.} Attention decode mixes MMA (QKV, $PV$) with SFU-heavy softmax \citep{dao2022flashattention,nvidia2022hopper,patel2025llminference,semianalysis2024tma,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Hopper generation shifts this balance---profile both units \citep{patel2025llminference,liu2024deepseekv3}.

\paragraph{TC is rarely first fix.} Batch-1 decode presents thin MMA tiles with low utilization \citep{nvidia2022hopper,memoryfloor2026,taxbreak2026,patel2025llminference,semianalysis2024tma}. Memory-Floor shows H100 decode often orchestration-limited \citep{memoryfloor2026,nvidia2024cudagraphs}.

After fusion cuts activation traffic, TMA-fed MMA epilogues matter \citep{semianalysis2024tma,dao2022flashattention,yirage2026}. Blackwell widens low-precision paths but does not fix materialized activations \citep{liu2024deepseekv3,patel2025llminference,memoryfloor2026}.

\paragraph{Quantitative intuition.} Suppose fusion removes six $d$-vector HBM round trips at $d=4096$ and BF16 dtype \citep{memoryfloor2026,taxbreak2026,patel2025llminference}. That is roughly $6 \times 4096 \times 2 \approx 49$ KB saved per layer per token in activation traffic alone, before KV reads \citep{kwon2023vllm,dao2022flashattention}. A ten-point gain in tensor-core utilization on a $1\times4096$ GEMM rarely moves ms/token when DRAM was the bound resource \citep{memoryfloor2026,nvidia2022hopper,dlbook}. Use this back-of-envelope to prioritize PR review \citep{taxbreak2026,yirage2026,chen2020compilerbasedhardw}.

\paragraph{SFU share in attention decode.} Softmax exponentials and reciprocal sums consume SFU pipelines \citep{dao2022flashattention,nvidia2022hopper,patel2025llminference,dlbook}. DeepSeek-style kernels tune PTX \textbf{exp2} because SFU throughput interacts with MMA issue rate in the same SM \citep{liu2024deepseekv3,yirage2026,semianalysis2024tma}. Profiling must report both units, not TC alone \citep{patel2025llminference,memoryfloor2026,taxbreak2026}.

\section{PTX and Softmax Micro-ops}
\label{sec:ch04_ptx}
\paragraph{exp2 numerics.} Use \textbf{exp2} with explicit max-subtraction in online softmax merges \citep{dao2022flashattention,dlbook,patel2025llminference,nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,dao2023flashdecoding,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024aie,amd2024xdna}. Document FTZ behavior for fleet determinism \citep{patel2025llminference,yirage2026}.

\paragraph{PTX review gate.} Softmax-related PTX changes require FP64 reference diffs at $T=8192$ \citep{dao2023flashdecoding,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024aie,amd2024xdna,dlbook}.

\paragraph{Silicon-aware softmax.} PTX \textbf{exp2} sequences implement stable exponentials in online softmax loops \citep{dao2022flashattention,patel2025llminference,nvidia2022hopper,dlbook}. Online softmax uses fast \textbf{exp} and max reductions \citep{dao2022flashattention,patel2025llminference,nvidia2022hopper,dlbook}. PTX choices (\texttt{ex2.approx}, FTZ) affect stability and latency \citep{liu2024deepseekv3,chen2020compilerbasedhardw}.

\paragraph{Inline PTX in production.} DeepSeek-style tuning selects instructions predictably \citep{liu2024deepseekv3}. YiRage should lower stable softmax to reviewed PTX sequences \citep{yirage2026,dao2022flashattention,dao2023flashdecoding}. Separate exp launches break Chapter~10 boundaries \citep{taxbreak2026,memoryfloor2026}.

\section{YiRage CUDA Backend}
\label{sec:ch04_yirage_cuda_backend}
\paragraph{Lowering pipeline.} IR bufferization assigns tiers; CUDA pass selects TMA vs loads; cluster pass chooses DSM vs single block; PTX pass lowers softmax regions \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,dao2022flashattention,taxbreak2026,memoryfloor2026,patel2025llminference,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Handoff to Chapter~5.} CUDA legality is dynamic residency; XDNA legality is static slot assignment \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dlbook,dao2023flashdecoding}.

\paragraph{IR to Hopper.} YiRage checks buffer lifetimes, cluster dims, and TMA rules before emitting CUDA \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper}. Failed shared-memory plans reject---no silent spill \citep{memoryfloor2026,amd2024aie}.

\paragraph{Triplet regression.} Same IR exports to CUDA, CPU, and XDNA; GPU-only schedules are incomplete \citep{amd2024xdna,dlbook,liu2024deepseekv3,kwon2023vllm,patel2025llminference,taxbreak2026}.

\section{Profiling Hopper Decode Changes}
\label{sec:ch04_profiling}
\paragraph{Nsight Systems vs Compute.} Systems timelines expose launch storms and TMA overlap; Compute exposes spill, DRAM throughput, and TC/SFU counters \citep{patel2025llminference,semianalysis2024tma,nvidia2022hopper,taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Use both---matching Fregly's profile-driven methodology \citep{taxbreak2026,memoryfloor2026,patel2025llminference}.

\paragraph{Benchmark matrix.} Fixed models (Llama-3.2-1B, Qwen2.5), BS${=}1$, ctx buckets 512/2048/8192, report median ms/token plus kernels/token and bytes/token \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,patel2025llminference,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Profile-first workflow.} Start from ms/token, decompose launches/token (TaxBreak) and bytes/token (Memory-Floor / Nsight) \citep{taxbreak2026,memoryfloor2026,patel2025llminference,semianalysis2024tma}. Fix residency before MMA tile sweeps \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,dlbook,kwon2023vllm,nvidia2024cudagraphs,liu2024deepseekv3,amd2024aie,amd2024xdna,dao2023flashdecoding}.

\paragraph{Release artifacts.} Hopper PRs attach shared-memory accounting, Nsight DRAM deltas, and ablation rows vs graph-only baselines \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Hopper SM overview for decode.} Each streaming multiprocessor combines scalar cores, tensor cores, TMA units, and configurable shared memory \citep{nvidia2022hopper,semianalysis2024tma}. Decode MegaKernels treat an SM as a residency container---not a place to launch yet another thin epilogue \citep{taxbreak2026,yirage2026,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Occupancy vs fusion.} High register/shared memory usage lowers occupancy but may still win on bytes/token when fusion removes HBM passes \citep{memoryfloor2026,semianalysis2024tma,taxbreak2026,yirage2026,chen2020compilerbasedhardw,patel2025llminference,dao2022flashattention,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Optimize occupancy only after proving fusion legality \citep{yirage2026,taxbreak2026}.

\paragraph{Blackwell trajectory.} Successor chips extend TMA, FP4 tensor paths, and cluster features \citep{nvidia2022hopper,patel2025llminference,liu2024deepseekv3,semianalysis2024tma}. Migration re-tiles kernels but preserves mechanical sympathy invariants: no materialized activations in decode \citep{memoryfloor2026,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,taxbreak2026,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{CUDA Graphs interaction.} Graph capture replays the same TMA/MMA body with fewer host launches \citep{nvidia2024cudagraphs,memoryfloor2026,taxbreak2026}. Graphs without fused residency still pay HBM bytes---label benchmark rows honestly \citep{patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Comparison to CPU AVX paths.} CPU decode uses cache-blocked GEMMs and vectorized \textbf{exp2} on softmax \citep{dlbook,patel2025llminference,amd2024xdna}. Hopper wins when fusion keeps data on SM; CPU wins when GPU launches dominate \citep{taxbreak2026,memoryfloor2026,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,nvidia2022hopper,amd2024aie,amd2024xdna,dao2023flashdecoding}.

\paragraph{Warp specialization preview.} Persistent decode kernels assign warps to TMA, MMA, and epilogue roles \citep{semianalysis2024tma,nvidia2022hopper,taxbreak2026,yirage2026}. Chapter~11 MegaKernels assume this specialization model \citep{dao2022flashattention,dao2023flashdecoding,kwon2023vllm,memoryfloor2026,patel2025llminference,chen2020compilerbasedhardw,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook}.

\paragraph{Quantization on Hopper.} FP8/INT8 weights need dequant in-register or in shared memory \citep{patel2025llminference,liu2024deepseekv3,nvidia2022hopper,semianalysis2024tma,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,dao2022flashattention,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Scales consume the same residency budget as softmax carriers \citep{memoryfloor2026,dao2022flashattention}.

\paragraph{DeepSeek deployment ratio reminder.} DeepSeek-V3 uses far more decode than prefill GPUs \citep{patel2025llminference,liu2024deepseekv3,kwon2023vllm}. Hopper features matter most in decode pools where batch-1 residency dominates \citep{taxbreak2026,memoryfloor2026,dao2022flashattention,dao2023flashdecoding,yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook}.

\paragraph{Anti-pattern catalog.} (1) TMA everywhere on $M{=}1$ tiles; (2) cluster fusion without DSM accounting; (3) TC tile sweeps with flat bytes/token; (4) PTX \textbf{exp2} approximations without stability tests; (5) GPU-only YiRage exports \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}.

\paragraph{Execution checklist.} Before merging Hopper changes: IR shows no illegal global stores; Nsight reports DRAM delta; TaxBreak launches/token on BS${=}1$; long-context softmax stability vs FP64; XDNA compile pass or documented scope \citep{yirage2026,taxbreak2026,memoryfloor2026,amd2024aie,patel2025llminference,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024xdna,dlbook,dao2023flashdecoding}.

\section{Hopper Decode Engineering Patterns}
\label{sec:ch04_patterns}

\paragraph{Pattern 1: Fused layer skeleton.} A production Hopper decode MegaKernel typically sequences: (i) TMA load hidden state and weights into shared memory; (ii) QKV MMA tiles; (iii) KV loop with TMA $K/V$ tiles, online softmax in registers/shared memory, and $PV$ accumulation; (iv) SwiGLU MMA chain; (v) residual write or in-place update---all without global activation stores \citep{taxbreak2026,dao2022flashattention,semianalysis2024tma,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,patel2025llminference,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Deviations at any step re-open bytes/token bills \citep{memoryfloor2026,taxbreak2026}.

\paragraph{Pattern 2: Context buckets.} CUDA Graphs capture requires static loop bounds or padded max context \citep{nvidia2024cudagraphs,kwon2023vllm,dao2023flashdecoding}. Hopper MegaKernels compile separate buckets (512/2048/8192) rather than dynamic reallocation mid-graph \citep{yirage2026,patel2025llminference,taxbreak2026,memoryfloor2026,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,liu2024deepseekv3,amd2024aie,amd2024xdna,dlbook}. Fleet configs must pin bucket per deployment \citep{kwon2023vllm,taxbreak2026}.

\paragraph{Pattern 3: Warp-specialized TMA.} Dedicated warps issue TMA descriptors while MMA warps consume previous tiles \citep{semianalysis2024tma,nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,taxbreak2026,memoryfloor2026,patel2025llminference,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Misbalanced roles underutilize TMA or stall MMA \citep{patel2025llminference,semianalysis2024tma}.

\paragraph{Pattern 4: Softmax on SFU + shuffle.} Max-find via warp shuffle; \textbf{exp2} via SFU; carriers in FP32 even when matmul uses BF16 \citep{dao2022flashattention,patel2025llminference,nvidia2022hopper,dlbook,yirage2026,chen2020compilerbasedhardw,dao2023flashdecoding,taxbreak2026,memoryfloor2026,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,semianalysis2024tma,amd2024aie,amd2024xdna}. Chapter~10 invariants apply inside Hopper kernels \citep{dao2022flashattention,taxbreak2026}.

\paragraph{Pattern 5: Graph + fusion composition.} CUDA Graphs remove host overhead; fusion removes HBM bytes---compose both for H100 cells with low $R_{\mathrm{floor}}$ \citep{memoryfloor2026,nvidia2024cudagraphs,taxbreak2026,patel2025llminference,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. Report which effect dominated in each row \citep{memoryfloor2026,taxbreak2026}.

\paragraph{Pattern 6: Triplet parity tests.} Export identical IR to CUDA/XDNA/CPU; compare bytes/token normalized by memory hierarchy \citep{amd2024xdna,amd2024aie,yirage2026,chen2020compilerbasedhardw,patel2025llminference,taxbreak2026,memoryfloor2026,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dlbook,dao2023flashdecoding}. GPU ms/token alone misleads fleet decisions \citep{patel2025llminference,amd2024aie}.

\paragraph{Industrial narrative (Fregly-aligned).} DeepSeek-scale training stories emphasize co-design under hardware export limits \citep{liu2024deepseekv3,patel2025llminference}. Decode on Hopper repeats the lesson at chip scale: skillful residency beats brute-force silicon \citep{taxbreak2026,memoryfloor2026,dao2022flashattention,yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,nvidia2022hopper,kwon2023vllm,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. \section{Worked Example: One Decode Layer on H100}
\label{sec:ch04_worked_example}

This section walks through a single transformer decode layer the way a performance engineer would during a Hopper bring-up review. The goal is not to derive new math but to connect Chapter~1 counters to concrete CUDA tiers before Chapter~11 stitches the full MegaKernel \citep{taxbreak2026,memoryfloor2026}.

\paragraph{Setup.} Consider Llama-3.2-1B at batch size one, context length 2048, hidden dimension 2048, sixteen query heads, and grouped-query attention with four KV heads \citep{taxbreak2026,kwon2023vllm}. One new output token triggers one decode step. We compare an unfused PyTorch-style schedule against a fused Hopper plan that keeps activation-sized tensors in shared memory or registers for the entire layer \citep{dao2022flashattention,yirage2026}.

\paragraph{Unfused activation traffic (qualitative).} A conventional stack materializes at least: post-norm hidden state, Q projection output, K and V writes into the paged KV cache, attention output, post-attention residual buffer, MLP intermediate after gate/up projections, down projection output, and final residual update. Even when each tensor is only one token wide, each boundary that stores to HBM and reloads adds roughly $2d$ bytes of activation traffic per layer per token, where $d$ is the model width \citep{memoryfloor2026,patel2025llminference}. Six to eight such round trips per layer is typical on dense Llama-class stacks before counting KV cache reads \citep{taxbreak2026}. TaxBreak reports hundreds of kernel launches per token on H100 for unfused stacks; the launch tax is orchestration, but the byte tax is residency failure \citep{taxbreak2026,nvidia2024cudagraphs}.

\paragraph{Fused residency plan.} A Hopper MegaKernel target keeps the following live on-chip across stages: the incoming hidden vector fragment, QKV MMA tiles in shared memory, online softmax carriers $(m,\ell,o)$ per head in registers, partial $PV$ accumulators, SwiGLU gate/up tiles, and the down-projection epilogue buffer \citep{dao2022flashattention,semianalysis2024tma,yirage2026}. Weights and KV still stream from HBM---fusion does not remove parameter traffic---but activation-sized tensors never take a detour through global memory between attention and MLP \citep{memoryfloor2026,kwon2023vllm}. The engineering question becomes whether the plan fits Table~\ref{tab:cuda_memory} without spilling carriers \citep{nvidia2022hopper,chen2020compilerbasedhardw}.

\paragraph{Shared-memory arithmetic (sketch).} Hopper exposes up to 228 KB configurable shared memory per SM \citep{nvidia2022hopper,semianalysis2024tma}. A conservative fused tile for $d=2048$ might allocate: 8 KB for hidden state staging, 48 KB for QKV double buffers, 32 KB for softmax score tiles, 64 KB for SwiGLU intermediates, and 16 KB for epilogue handoff---roughly 168 KB before cluster pooling \citep{yirage2026,dao2022flashattention}. That leaves headroom for TMA descriptor scratch and barrier state, but not for a second concurrent layer replica \citep{semianalysis2024tma}. If planners add a duplicate score tile ``for simplicity,'' occupancy collapses or the compiler silently spills $(m,\ell,o)$ to local memory---both show up as bytes/token regressions in Memory-Floor style accounting \citep{memoryfloor2026,patel2025llminference}.

\paragraph{Register pressure for online softmax.} Each head maintains three scalars $(m,\ell)$ plus a partial output vector of length $d_{\mathrm{head}}=128$ in FP32 for numerical stability \citep{dao2022flashattention,dlbook}. Sixteen heads imply sixteen partial output vectors if not warp-shared---a register budget that fusion planners must co-locate with MMA fragment registers \citep{nvidia2022hopper,yirage2026}. Warp-specialized layouts assign one warp group per head cluster so shuffle reductions stay in-register \citep{dao2022flashattention,patel2025llminference}. Exceeding the register cap per thread lowers occupancy; the right trade-off is often \emph{lower occupancy, higher bytes/token win} when fusion removes HBM passes \citep{memoryfloor2026,semianalysis2024tma}.

\paragraph{TMA in the KV loop.} For each KV block of size $B_k$, TMA loads $K_b$ and $V_b$ tiles into shared memory while MMA units compute $QK_b^\top$ on the previous block \citep{semianalysis2024tma,dao2023flashdecoding}. Paged KV via vLLM block tables means TMA descriptors must be rebuilt or indexed per physical block id inside the kernel \citep{kwon2023vllm,patel2025llminference}. A common failure is issuing TMA for $M=1$ micro-tiles where descriptor setup dominates---profile both TMA and scalar load paths on the target shape before mandating TMA in YiRage lowering \citep{yirage2026,taxbreak2026,dlbook}.

\paragraph{Cluster decision point.} When score tiles exceed single-block shared memory, Hopper clusters pool DSM across cooperating blocks \citep{nvidia2022hopper,semianalysis2024tma}. Flash-Decoding splits KV across SMs and merges online softmax statistics in-kernel \citep{dao2023flashdecoding}. If cluster launch is illegal on the deployment SKU, the fallback is an extra merge kernel---which reintroduces launches/token and may erase fusion wins \citep{taxbreak2026,dao2022flashattention}. YiRage should reject cluster plans that fail compile rather than emit a silent multi-kernel fallback \citep{yirage2026,chen2020compilerbasedhardw}.

\paragraph{Profile interpretation.} After implementing the fused plan, an engineer runs Nsight Systems on the full model step and Nsight Compute on the MegaKernel \citep{patel2025llminference,semianalysis2024tma}. Systems timelines should show one or few launches per layer instead of dozens \citep{taxbreak2026,nvidia2024cudagraphs}. Compute counters should show DRAM throughput dropping with unchanged MMA instruction counts---evidence that bytes/token improved while tensor cores were never the first bottleneck \citep{memoryfloor2026,nvidia2022hopper}. If TC utilization rises but ms/token is flat, the team optimized the wrong metric \citep{patel2025llminference,taxbreak2026}.

\paragraph{Ablation table discipline.} Publish three rows for every Hopper PR: graph-only (launches fixed, bytes unchanged), fusion-only (bytes fixed, launches unchanged if possible), and composed graph plus fusion \citep{memoryfloor2026,nvidia2024cudagraphs,taxbreak2026}. Fregly-style goodput reporting requires labeling which lever moved \citep{patel2025llminference,dlbook}. Internal regression matrices in the appendix anchor TaxBreak launches/token and Memory-Floor $R_{\mathrm{floor}}$ cells so readers can map this worked example to measured numbers \citep{taxbreak2026,memoryfloor2026,yirage2026}.

\paragraph{Contrast with L4 and XDNA.} The same IR exports to L4-class GPUs and Ryzen AI XDNA NPU backends \citep{amd2024xdna,amd2024aie,yirage2026}. Memory-Floor shows Qwen-2.5-7B batch-1 $R_{\mathrm{floor}}$ much higher on L4 than H100---different roofline knee, same residency story \citep{memoryfloor2026,patel2025llminference}. XDNA cannot rely on Hopper TMA; it assigns static SRAM slots at compile time \citep{amd2024aie,chen2020compilerbasedhardw}. Triplet parity tests ensure CUDA-specific wins do not fork the compiler IR \citep{yirage2026,amd2024xdna}.

\begin{example}
\textbf{Review question.} A teammate proposes a Hopper kernel that uses TMA for every $1\times128$ score fragment but still stores attention output to global memory before MLP. Ask: what happens to bytes/token and to Chapter~10 softmax carrier residency? Correct answer: bytes/token stays high and MegaKernel legality fails even if TMA overlap looks excellent in microbench \citep{memoryfloor2026,dao2022flashattention,yirage2026}.
\end{example}

\section{Hopper Feature Reference for Compiler Authors}
\label{sec:ch04_compiler_reference}

Compiler authors lowering decode IR to CUDA need a checklist tighter than a hardware manual \citep{chen2020compilerbasedhardw,yirage2026,nvidia2022hopper}. This section states invariants the YiRage CUDA backend enforces; Chapter~5 mirrors them for XDNA static tiles \citep{amd2024xdna,amd2024aie}.

\paragraph{Invariant 1: No activation-sized global stores inside a fused region.} Any store of a vector length $\Theta(d)$ between attention, softmax, and MLP stages is a fusion bug unless proven dead by analysis \citep{memoryfloor2026,yirage2026,chen2020compilerbasedhardw}. Pass managers should mark such stores as hard errors in decode mode \citep{taxbreak2026,dao2022flashattention}.

\paragraph{Invariant 2: TMA legality is shape-dependent.} TMA requires aligned multidimensional tiles; batch-1 decode often presents thin $M$ dimensions \citep{semianalysis2024tma,dlbook,taxbreak2026}. Lowering must select TMA versus vector loads per sub-op based on a cost model, not a global flag \citep{yirage2026,patel2025llminference,nvidia2022hopper}.

\paragraph{Invariant 3: Cluster dims are a compile-time contract.} Cluster launch bounds must match DSM footprint \citep{nvidia2022hopper,semianalysis2024tma}. Illegal combinations fail at compile time with a residency report, not at runtime with silent corruption \citep{yirage2026,chen2020compilerbasedhardw,dao2022flashattention}.

\paragraph{Invariant 4: Softmax PTX is part of the ABI.} Online softmax lowering emits reviewed \textbf{exp2} sequences with documented FTZ behavior \citep{dao2022flashattention,liu2024deepseekv3,patel2025llminference,dlbook}. Changing PTX without FP64 reference diffs at long context breaks fleet determinism \citep{dao2023flashdecoding,yirage2026}.

\paragraph{Invariant 5: Triplet exports share bufferization.} The same bufferization graph feeds CUDA, CPU, and XDNA passes \citep{yirage2026,amd2024xdna,amd2024aie,chen2020compilerbasedhardw}. GPU-only schedules are incomplete for a multi-hardware book and product story \citep{patel2025llminference,amd2024aie}.

\paragraph{Lowering walkthrough (six passes).}
\begin{enumerate}
\item \textbf{Bufferization:} Assign each SSA tensor to register, shared, or global tier for its lifetime \citep{yirage2026,chen2020compilerbasedhardw,memoryfloor2026}.
\item \textbf{Fusion grouping:} Merge attention, softmax, and MLP into one fused region subject to shared-memory cap \citep{dao2022flashattention,taxbreak2026}.
\item \textbf{TMA selection:} For each global-to-shared copy, choose TMA or vector load \citep{semianalysis2024tma,nvidia2022hopper}.
\item \textbf{Cluster planning:} If fused tile exceeds block shared memory, compute cluster shape and DSM layout \citep{nvidia2022hopper,dao2023flashdecoding}.
\item \textbf{Sync insertion:} Place barriers only on shared-memory phase edges \citep{semianalysis2024tma,yirage2026,nvidia2024cudagraphs}.
\item \textbf{PTX lowering:} Emit softmax micro-ops and MMA epilogues with stability annotations \citep{dao2022flashattention,liu2024deepseekv3,patel2025llminference}.
\end{enumerate}

\paragraph{Testing gates.} Each merged CUDA backend change carries: numerical diff versus FP64 reference at $T\in\{512,2048,8192\}$; Nsight DRAM byte delta at batch one; TaxBreak launches/token cell; XDNA compile or documented deferral \citep{taxbreak2026,memoryfloor2026,amd2024xdna,yirage2026,patel2025llminference,dao2023flashdecoding}.

\paragraph{Relationship to CUDA Graphs.} Graph capture replays a fixed launch sequence \citep{nvidia2024cudagraphs,memoryfloor2026}. Bufferization must pin graph capture buckets to max context lengths used in production \citep{kwon2023vllm,taxbreak2026}. Dynamic shapes without re-capture remain an open research thread \citep{patel2025llminference,liu2024deepseekv3,yirage2026}.

\paragraph{Blackwell porting notes.} Successor silicon widens FP4 and FP8 tensor paths and extends TMA \citep{nvidia2022hopper,patel2025llminference,liu2024deepseekv3}. Compiler invariants survive: residency first, orchestration second, MMA tile tuning last \citep{memoryfloor2026,taxbreak2026,semianalysis2024tma,dao2022flashattention,yirage2026,chen2020compilerbasedhardw}.



\paragraph{Nsight Compute counter cheat sheet.} For Hopper decode MegaKernels, prioritize, in order: DRAM bytes per kernel invocation, L1TEX throughput, local memory loads from spill, achieved occupancy versus chosen shared-memory config, TMA busy cycles versus MMA busy cycles, and only then tensor-core utilization \citep{patel2025llminference,semianalysis2024tma,memoryfloor2026,taxbreak2026,nvidia2022hopper,yirage2026}. Teams that open with TC utilization often optimize a unit that was idle for good reason \citep{memoryfloor2026,patel2025llminference,dlbook}.

\paragraph{Systems timeline reading.} Nsight Systems exposes host launch storms, CUDA Graphs replay gaps, and CPU-side sampling overhead \citep{nvidia2024cudagraphs,taxbreak2026,patel2025llminference}. A fused kernel may look slower in isolation if the baseline hid launch latency behind queue overlap \citep{taxbreak2026,memoryfloor2026}. Always compare end-to-end ms/token for one decode step, not kernel-only timers \citep{patel2025llminference,dlbook,kwon2023vllm}.

\paragraph{Regression hygiene.} Pin model weights, tokenizer, context length bucket, driver, and CUDA version when comparing Hopper PRs \citep{taxbreak2026,memoryfloor2026,yirage2026,patel2025llminference}. Record launches/token and bytes/token alongside median and p99 ms/token \citep{taxbreak2026,memoryfloor2026,nvidia2024cudagraphs}. Ship a short residency diagram in the PR description so reviewers see tier assignments without reading all PTX \citep{yirage2026,chen2020compilerbasedhardw,semianalysis2024tma,dao2022flashattention}.

\paragraph{Handoff to Chapter~5.} CUDA legality is dynamic: tiers are chosen per kernel subject to occupancy and spill constraints \citep{nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,amd2024xdna}. XDNA legality is static: SRAM slots are fixed at compile time with multi-tile DMA schedules \citep{amd2024aie,amd2024xdna,patel2025llminference}. The same bufferization graph must explain both or the triplet story collapses \citep{yirage2026,amd2024aie,memoryfloor2026,taxbreak2026,dao2022flashattention,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,dlbook,dao2023flashdecoding}.


\paragraph{Synchronization case study.} Consider a fused block that double-buffers TMA into shared memory while MMA consumes the alternate buffer \citep{semianalysis2024tma,nvidia2022hopper,dao2022flashattention}. Only two block-level barriers per buffer swap are required: one after TMA completion, one before overwriting the consumed buffer \citep{yirage2026,chen2020compilerbasedhardw,patel2025llminference}. Inserting a barrier after every warp shuffle in softmax adds bubbles without safety benefit \citep{dao2022flashattention,dlbook,taxbreak2026}. Compiler sync insertion should be edge-driven on the fused dataflow graph \citep{yirage2026,semianalysis2024tma,memoryfloor2026,nvidia2024cudagraphs,kwon2023vllm,liu2024deepseekv3,amd2024aie,amd2024xdna,dao2023flashdecoding}.

\paragraph{Paged KV and TMA co-design.} vLLM-style block tables map logical token positions to non-contiguous physical blocks \citep{kwon2023vllm,patel2025llminference}. TMA descriptors assume regular layouts; PagedAttention kernels either gather into contiguous shared-memory staging or use per-block descriptors inside the KV loop \citep{dao2023flashdecoding,yirage2026,semianalysis2024tma}. This is why MegaKernel codegen special-cases KV: it is not a generic GEMM \citep{taxbreak2026,dao2022flashattention,memoryfloor2026,chen2020compilerbasedhardw,nvidia2022hopper}.


\paragraph{Closing the loop with Chapter~1.} Return to the TaxBreak and Memory-Floor cells for your target model after every Hopper change \citep{taxbreak2026,memoryfloor2026,patel2025llminference}. If launches/token and bytes/token are unchanged, ms/token gains are suspect or attributable to unrelated driver noise \citep{nvidia2024cudagraphs,yirage2026,taxbreak2026,dlbook,kwon2023vllm,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,liu2024deepseekv3,amd2024aie,amd2024xdna,dao2023flashdecoding,chen2020compilerbasedhardw}. Goodput engineering means all three counters move together in the expected direction \citep{memoryfloor2026,patel2025llminference,taxbreak2026}.


\paragraph{Practitioner mantra.} Profile end-to-end decode first. Cut activation bytes second. Collapse launches third. Tune tensor cores fourth. Re-profile on every driver and context bucket change. Teach teammates to read residency diagrams before PTX diffs. This ordering prevents weeks of wasted TC tile sweeps on bandwidth-bound decode pools. Keep the mantra visible in every Hopper optimization doc your team maintains. It aligns silicon features with measured decode goodput outcomes every single engineering sprint review.

\section{Key Takeaways}
\label{sec:ch04_key_takeaways}

\begin{itemize}
\item \textbf{Mechanical sympathy on Hopper.} TMA, clusters, and shared memory keep decode data on-chip \citep{nvidia2022hopper,semianalysis2024tma,dao2022flashattention}.
\item \textbf{Profile before TC tuning.} Low $R_{\mathrm{floor}}$ means fix launches and bytes first \citep{memoryfloor2026,taxbreak2026,patel2025llminference}.
\item \textbf{TMA overlap $\neq$ fusion.} Async copies hide latency; MegaKernels cut activation HBM traffic \citep{semianalysis2024tma,memoryfloor2026,yirage2026}.
\item \textbf{Sync sparingly.} Warp shuffles and planned barriers beat grid sync in token loops \citep{nvidia2022hopper,nvidia2024cudagraphs,dao2022flashattention}.
\item \textbf{PTX for softmax.} Stable online softmax ties to SFU/PTX choices \citep{dao2022flashattention,liu2024deepseekv3,dlbook}.
\item \textbf{YiRage legality.} CUDA backend rejects illegal residency \citep{yirage2026,chen2020compilerbasedhardw,amd2024aie}.
\item \textbf{Goodput metrics.} Always pair kernels/token and bytes/token with ms/token \citep{taxbreak2026,memoryfloor2026}.
\end{itemize}

\section{Conclusion}
\label{sec:ch04_conclusion}

Hopper/Blackwell supply async copy, cluster, and tensor-core machinery for decode MegaKernels when schedules respect Table~\ref{tab:cuda_memory} \citep{nvidia2022hopper,semianalysis2024tma,taxbreak2026}. TMA overlaps KV/weight movement; DSM widens fusion; sync and PTX discipline protect Chapter~10 softmax carriers \citep{dao2022flashattention,patel2025llminference,yirage2026,chen2020compilerbasedhardw,memoryfloor2026,nvidia2024cudagraphs,kwon2023vllm,liu2024deepseekv3,dlbook,amd2024xdna,amd2024aie,dao2023flashdecoding}.

Chapter~5 contrasts dynamic CUDA tiers with XDNA static tiles---triplet regression from one IR is mandatory \citep{amd2024xdna,amd2024aie}. Carry bytes/token and launches/token; faster silicon does not redefine goodput \citep{taxbreak2026,memoryfloor2026,patel2025llminference,semianalysis2024tma,nvidia2022hopper,yirage2026,chen2020compilerbasedhardw,dao2022flashattention,liu2024deepseekv3,kwon2023vllm,nvidia2024cudagraphs,dlbook,dao2023flashdecoding}. Reviewers should reject Hopper PRs that show faster microbenchmark GEMMs without DRAM byte deltas or TaxBreak launch counts \citep{taxbreak2026,memoryfloor2026,patel2025llminference,yirage2026,dao2022flashattention,semianalysis2024tma,nvidia2022hopper,chen2020compilerbasedhardw,kwon2023vllm,liu2024deepseekv3,nvidia2024cudagraphs,amd2024aie,amd2024xdna,dlbook,dao2023flashdecoding}. That discipline keeps the book's Fregly-aligned story honest across fleets \citep{dlbook,taxbreak2026,memoryfloor2026}.
